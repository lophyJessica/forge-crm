# Forge CRM 客户关系管理系统

> 项目类型：B端 客户关系管理 | 角色：产品经理 | 品牌：Forge
>
> 基于京东零售 CRM 实战经验（线索评分模型、转化率 35%→53%、2800 条线索 1500 份问卷），以 Forge 为演示品牌还原。
>
> **品牌口径**：强盛科技 = 演示公司名；Forge = 系统品牌。二者为公司—产品关系，并存不冲突。

## 项目简介

Forge CRM 客户关系管理系统，覆盖线索管理、商机管理、客户管理、跟进记录、AI 评分预测五大模块。向上对接 ERP 客户档案与销售订单，向下串联 WMS 发货履约。核心验证方向：**AI 串联 CRM 和 ERP**，实现线索评分→自动转客户→商机预测→触发销售的闭环。

## 核心规范

1. 所有文档用简体中文输出，字段名、组件名、枚举值保留英文
2. PRD 文档必须包含：业务背景、功能范围（In/Out Scope）、单据定位、业务场景、状态机、字段清单
3. 状态变更必须通过动作按钮触发，不允许在表单中直接修改状态字段
4. 主数据不提供删除，统一用"停用"操作替代
5. 数据冲突时以简历为准（京东 CRM：线索首次响应 1.5h→30min，转化率 35%→53%，2800 条线索 1500 份问卷）

## 技术栈（Demo 版）

- 前端：React + TypeScript + Tailwind CSS
- 构建：Vite

## 相关系统

| 系统 | 关系 | 说明 |
|------|------|------|
| ERP（含销售订单） | 下游 | 客户档案 SSOT 在 ERP；CRM 商机成交后下推销售订单 |
| WMS | 间接下游 | ERP 发货单下发 WMS 执行，CRM 不直接对 WMS |

## 系统边界（铁律，不可违反）

| 数据 | SSOT（唯一权威源） | CRM 可以做什么 |
|------|-------------------|---------------|
| 客户档案 | **ERP** | CRM 持有客户快照、跟进记录、商机关联；正式建档必须写入 ERP |
| 商品/价格 | **ERP** | CRM 商机阶段可引用 ERP 商品，不独立维护 |
| 线索 | **CRM** | 唯一权威源 |
| 商机 | **CRM** | 唯一权威源 |
| 订单 | **ERP** | CRM 商机成交后下推 ERP 生成销售订单草稿 |

## AI 串联层 — 核心验证

| AI 节点 | 输入 | 输出 | 执行动作 |
|---------|------|------|---------|
| 线索评分 | 历史跟进数据+转化结果 | 0-100 分 | 高分线索自动分配，低分线索进入培育池 |
| 商机预测 | 客户画像+历次订单+跟进频率 | 成交概率+推荐商品 | 预测成交→ERP 库存预占；预测流失→CRM 召回任务 |
| 流失预警 | 跟进间隔+订单停滞天数 | 流失风险等级 | 高风险→推送跟进提醒+ERP 信用冻结 |

串联方式：API 实时同步。商机成交通知 ERP 下推订单，ERP 客户档案变更为 CRM 客户快照同步更新。

## 约束（NIS：Not In Scope）

- 本项目不涉及真实后端 API 实现，AI 串联用 Mock 模拟
- 不涉及 ERP/WMS 侧的原型修改——只做 CRM 侧，对接走 Mock
- 不涉及移动端（App/小程序），仅 PC Web
- 不涉及市场营销模块（短信/邮件群发、广告归因）
