# 01 — 业务背景与项目范围

## 业务背景

Forge CRM 为企业提供从获客到成交的客户全生命周期管理。核心场景：市场投放获取线索 → 销售跟进转化为商机 → 商务谈判成交 → 客户持续运营 → 流失预警与召回。

本项目基于京东零售 CRM 实战经验设计——线索评分模型驱动分配、转化率从 35% 提升到 53%、2800 条线索数据分析发现"未及时跟进"是最大瓶颈（占 42%）。

## 一期范围（In Scope）

| 模块 | 核心能力 | 说明 |
|------|---------|------|
| 线索管理 | 线索录入/导入/去重/分配/回收/转客户 | 线索池 + 公海，超时未跟进自动回收 |
| AI 线索评分 | 基于跟进历史和转化结果自动评分（0-100） | 高分线索优先分配，低分进培育池 |
| 商机管理 | 商机创建/阶段推进/预测成交/丢单 | 阶段：初步接触→需求确认→报价→谈判→成交/丢单 |
| 客户管理 | 客户快照/关联商机/关联订单/跟进记录 | 客户档案 SSOT 在 ERP，CRM 持有快照 |
| 跟进记录 | 跟进方式/内容/下次计划 | 关联线索/商机/客户 |
| AI 商机预测 | 客户画像+历次订单+跟进频率 → 成交概率 | 高概率→ERP 库存预占通知 |
| AI 流失预警 | 跟进间隔+订单停滞天数 → 风险等级 | 高风险→推送跟进提醒 |
| 与 ERP 串联 | 商机成交→ERP 销售订单草稿 / 客户建档→同步 ERP | API 实时同步（Mock 模拟） |

## 一期不包含（Out Scope）

| 能力 | 为什么 Out | 计划 |
|------|-----------|------|
| 营销模块（短信/邮件/广告归因） | 优先验证 AI 串联 | 二期 |
| 客户分层与会员体系 | 依赖 ERP 订单数据积累 | 二期 |
| 移动端（App/小程序） | 先跑通 PC 端核心链路 | 二期 |
| ERP/WMS 侧的实际接口开发 | 非 CRM 项目范围 | Mock 示意即可 |
