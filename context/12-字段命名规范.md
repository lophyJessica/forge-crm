# 12-字段命名规范 (SSOT)
本项目所有 PRD 字段编码、枚举、Mock 统一遵守本规范；基础数据(库区/商品/货位)为已落地样板。
## 1. 基本约定
- 字段编码一律 camelCase（首字母小写）。
- 状态枚举一律 ENABLED / DISABLED（BaseDataStatus）。
- 时间字段 createdAt / updatedAt / xxxAt，格式 `YYYY-MM-DD HH:mm:ss`。
- 布尔字段 isXxx（isBarcodeBound）或 xxxManaged（batchManaged）。
## 2. 单号字段：语义前缀 + No
- 单据自身单号：rcvNo 收货 / putNo 上架 / waveNo 波次 / pickNo 拣货 / checkNo 复核 / pkgNo 包裹 / dshNo 交运 / ckNo 盘点 / transferNo 调拨 / blNo 报损 / flNo 库存流水。
- 跨单据引用上游单号：source + 语义 + No（sourceOrderNo / sourceCheckNo / sourceTransferNo）。
- 禁止用泛化 id 命名业务单号。
## 3. 商品字段：product 系
- productCode / productName / productBarcode / productSpec / unit。
- 禁止 sku_code / sku_name / sku_barcode。
## 4. 数量字段：保留业务粒度，只 camelCase
- 精细语义保留仅转写：actualReceiveQty / lineTransferOutQty / qtyOnHandAfter / qtyPicked 等。
- 禁止塌并为 Demo 简化字段；PRD 领先，前端二期补齐，允许文档字段暂无代码对应。
