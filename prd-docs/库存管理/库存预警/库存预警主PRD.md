# 库存预警主PRD

> 状态：V1.0 已补充 | 角色：主PRD | 类型：查询页 | Mock 日期：2026-07
> 权威层级：context/ > 本文件（改业务规则须回溯 context/）
> 参照套件：库存管理/库存查询（同为查询页 6 件）

## 1. 业务背景

Forge WMS 一期已覆盖入库、出库、盘点、调拨等库存变动链路，库存数量会随执行确认自动更新。仓库运营需要在 PC/PDA 上快速识别低库存商品，避免一线作业人员在销售出库、拣货或补货沟通时才发现库存不足。

库存预警用于展示低于商品档案阈值的库存清单：当前现存低于安全库存时标黄，当前现存低于最低库存时标红。该页面仅做只读查询和视觉提示，不生成任务、不推送消息、不发起补货。

## 2. 功能范围

### 2.1 In Scope

- PC/PDA 访问库存预警查询页。
- 按仓库、商品、预警等级筛选预警清单。
- 按仓库 + 商品聚合当前现存，并读取商品档案中的安全库存、最低库存。
- 根据预警规则展示标黄/标红标签。
- 空值按通用规范展示 `-`，数量字段按整数右对齐展示。

### 2.2 Out Scope

- 自动预警推送、站内信、PDA 震动提醒、短信/企微通知。
- 补货建议、采购建议、自动生成采购单/调拨单。
- 在库存预警页维护安全库存/最低库存。
- 新增、编辑、详情、确认、关闭、忽略、处理完成等动作。
- 手工调整库存、库存成本核算、库龄分析。

## 5. 状态机

不适用(只读预警清单,无语态流转)。

原因：库存预警不是业务单据，不存在草稿、待审核、已确认、已关闭等状态；页面只读取库存与商品档案数据并计算展示结果，不通过按钮触发状态变更。

## 6. 核心口径

### 6.1 术语三口径

| 术语 | 字段建议 | 口径 |
|:--|:--|:--|
| 当前现存 | `currentStock` | 按仓库 + 商品聚合库存现存；不包含在途 |
| 安全库存 | `safetyStock` | 商品档案字段，上游 SSOT |
| 最低库存 | `minimumStock` | 商品档案字段，上游 SSOT；当前商品档案字段清单仍为占位，字段编码待确认 |

### 6.2 预警等级

| 优先级 | 条件 | `warningLevel` | 展示 |
|:--|:--|:--|:--|
| 1 | `currentStock < minimumStock` | `MINIMUM_LOW` | 标红，文案“低于最低库存” |
| 2 | `currentStock < safetyStock` 且未命中最低库存 | `SAFETY_LOW` | 标黄，文案“低于安全库存” |
| - | 未命中以上条件 | 不进入预警清单 | 不展示 |

规则说明：
- 低于判断使用严格小于 `<`；等于安全库存或最低库存不触发对应预警。
- 当同时低于最低库存和安全库存时，以最低库存预警为准，展示红色。
- 安全库存/最低库存只从商品档案读取，本页不自建阈值字段、不提供维护入口。
- 当前前端 Demo 商品种子数据已有 `safetyStock`，但商品档案字段清单仍是占位；`minimumStock` 尚未在现有商品档案 PRD/前端类型中落地，本套件按“字段存在于商品档案但当前为占位”处理。

## 8. 验收标准

| 编号 | 验收项 | 预期 |
|:--|:--|:--|
| AC01 | 只读查询 | 页面不出现新增、编辑、详情、确认、补货入口 |
| AC02 | 仓库筛选 | 选择仓库后，仅展示该仓库命中的预警行 |
| AC03 | 商品筛选 | 输入商品编码/名称关键词后，仅展示匹配商品 |
| AC04 | 预警等级筛选 | 选择 `SAFETY_LOW` 或 `MINIMUM_LOW` 后，仅展示对应等级 |
| AC05 | 标黄规则 | `currentStock < safetyStock` 且未低于最低库存时，展示黄色预警 |
| AC06 | 标红规则 | `currentStock < minimumStock` 时，展示红色预警，优先级高于黄色 |
| AC07 | 边界值 | `currentStock = safetyStock` 或 `currentStock = minimumStock` 时，不触发对应预警 |
| AC08 | 阈值来源 | 安全库存/最低库存展示为商品档案来源；最低库存未落地时标注占位/待确认 |
