## 1. 基础规则

| 规则ID | 规则 | 说明 |
| :--- | :--- | :--- |
| BASE-R01 | 库存流水为只读审计日志 | 页面不提供新增、编辑、删除、作废、调整库存、状态变更能力 |
| BASE-R02 | FL 由系统自动生成 | 只有库存动作生效时写入，不允许前端直接调用写入接口造数 |
| BASE-R03 | 一次变动至少一条流水 | 按仓库、货位、商品、来源单据行拆分记录；多货位/多商品生成多条 |
| BASE-R04 | 写入后不可变更 | 如业务发生冲正，必须通过后续业务动作生成新的 FL，不修改历史 FL |
| BASE-R05 | 查询页不拥有状态机 | 状态机写作口径为“不适用(只读查询流水,无单据状态流转)” |

## 2. FL 单号规则

| 项目 | 规则 |
| :--- | :--- |
| 字段 | `flNo` |
| 格式 | `FL{YYYYMMDD}-{8位序号}` |
| 长度 | 日期体 `{YYYYMMDD}` 8 位 + 连字符 1 位 + 序号 8 位；加 `FL` 前缀后完整展示为 19 个字符 |
| 示例 | `FL20260705-00000001` |
| 生成方 | 系统自动生成 |
| 可编辑 | 否 |
| 回收 | 不回收 |

> 命名规则与 `context/09-单据命名规则.md` 保持一致。

## 3. 生成触发规则

| 规则ID | 变动类型 | 触发动作 | 库存影响 | 生成要求 |
| :--- | :--- | :--- | :--- | :--- |
| GEN-R01 | `INBOUND=入库+` | 入库链最终确认/上架确认 | 现存增加；可用或冻结口径按入库链确认结果同步 | 每个上架货位、商品行生成 FL |
| GEN-R02 | `OUTBOUND=出库-` | 包装完成 `PKG` 过账 | 现存 `-N`，占用 `-N`；可用不重复扣减 | 每个包裹商品行生成 FL |
| GEN-R03 | `TRANSFER_OUT=调拨-` | 调拨单 `TR` 调出库确认 | 调出仓现存 `-N`，在途增加 | 按调出货位、商品行生成 FL |
| GEN-R04 | `TRANSFER_IN=调拨+` | 调拨单 `TR` 调入库确认 | 调入仓现存 `+N`，在途减少 | 按调入货位、商品行生成 FL |
| GEN-R05 | `STOCK_GAIN=盘盈+` | 盘点差异确认调增 | 现存 `+N` | 按盘点货位、商品行生成 FL |
| GEN-R06 | `STOCK_LOSS=盘亏-` | 盘点差异确认调减 | 现存 `-N` | 按盘点货位、商品行生成 FL |
| GEN-R07 | `FREEZE=冻结` | 质检、盘点、报损等锁库 | 冻结 `+N`，可用 `-N`，现存不变 | 记录冻结后的现存/占用/冻结/可用快照 |
| GEN-R08 | `UNFREEZE=解冻` | 质检合格、盘点完成等解锁 | 冻结 `-N`，可用 `+N`，现存不变 | 记录解冻后的现存/占用/冻结/可用快照 |
| GEN-R09 | `ALLOCATE=占用` | SO 审核等库存锁定 | 占用 `+N`，可用 `-N`，现存不变 | 来源单号来自上游销售/订单动作 |
| GEN-R10 | `RELEASE=释放` | 订单取消或出库实扣后释放占用 | 占用 `-N`，可用 `+N`，现存不变 | 释放数量不得超过当前占用数量 |

## 4. 幂等与一致性规则

| 规则ID | 规则 | 验证点 |
| :--- | :--- | :--- |
| IDEM-R01 | 同一 `sourceOrderType + sourceLineId + changeType` 在同一库存动作下不得重复生成 FL | 重复提交、网络重试只保留一条有效流水 |
| IDEM-R02 | FL 写入与库存口径更新必须在同一事务内完成 | 库存已变但 FL 缺失、FL 存在但库存未变都不可接受 |
| IDEM-R03 | 每条 FL 写入时必须保存变动后快照 | 至少包含 `qtyOnHandAfter`，系统字段保存占用、冻结、可用快照 |
| IDEM-R04 | 历史 FL 不随商品/仓库主数据改名而回写 | 列表展示使用写入时快照，避免历史对账漂移 |

## 5. 查询规则

### 5.1 条件组合

所有查询条件采用 `AND` 组合逻辑。字段为空时不参与过滤。

| 条件 | 匹配规则 | 空值处理 | 校验规则 |
| :--- | :--- | :--- | :--- |
| `flNo` | 模糊匹配，系统转大写 | 不过滤 | 不允许输入危险脚本字符 |
| `warehouseCode` | 精确匹配 | 默认当前用户可见仓库 | 必须叠加权限仓库范围 |
| `zoneCode` | 精确匹配 | 不过滤 | 若已选仓库，仅展示该仓库下库区 |
| `locationCode` | 精确匹配优先，支持右模糊 | 不过滤 | PDA 扫码值按精确匹配 |
| `productKeyword` | 商品编码精确或名称模糊 | 不过滤 | 支持编码、名称、条码联想 |
| `occurredRange` | 闭区间 | 默认最近 90 天 | 跨度不得超过 365 天 |
| `changeType` | 精确匹配 | 不过滤 | 仅允许字段清单定义的枚举值 |
| `changeDirection` | `changeQty > 0` 或 `< 0` | `ALL` 不过滤 | 与 `changeType` 同时选择时继续 `AND` 过滤 |
| `sourceOrderNo` | 模糊匹配，系统转大写 | 不过滤 | 仅查询，不触发来源单据动作 |

### 5.2 默认排序与分页

| 项目 | 规则 |
| :--- | :--- |
| 默认排序 | `occurredAt DESC`，同一秒内按 `flNo DESC` |
| PC 分页 | 默认 20 条/页 |
| PDA 展示 | 默认展示最近数据，向下滚动加载；每次加载条数由前端实现控制 |
| 空值 | 统一展示 `-` |

## 6. 展示规则

| 规则ID | 规则 |
| :--- | :--- |
| VIEW-R01 | `changeQty` 必须带正负号；正数展示为 `+N`，负数展示为 `-N` |
| VIEW-R02 | `INBOUND`、`TRANSFER_IN`、`STOCK_GAIN` 为现存增加类；`OUTBOUND`、`TRANSFER_OUT`、`STOCK_LOSS` 为现存减少类 |
| VIEW-R03 | `FREEZE`、`UNFREEZE`、`ALLOCATE`、`RELEASE` 为状态口径变动，`qtyOnHandAfter` 可以不变 |
| VIEW-R04 | 变动后可用按系统字段快照复核：`qtyAvailableAfter = qtyOnHandAfter - qtyAllocatedAfter - qtyFrozenAfter` |
| VIEW-R05 | 现存倒挂时允许列表展示负数，但必须红色强调；倒挂处理不在本页面发起 |
| VIEW-R06 | 来源单号为追溯字段，本模块不定义来源单据详情页，不提供编辑动作 |

## 7. 权限规则

| 规则ID | 角色/范围 | 规则 |
| :--- | :--- | :--- |
| PERM-R01 | 仓管员 | 只能查询其拥有权限的仓库流水 |
| PERM-R02 | 仓库主管 | 可查询所辖仓库范围内的流水 |
| PERM-R03 | 产品/测试/管理员 | 可按配置查看多仓流水 |
| PERM-R04 | PDA 用户 | 扫码查询结果仍受仓库权限限制 |
| PERM-R05 | 导出 | 仅导出当前用户当前筛选结果，不绕过页面权限 |

## 8. 异常与边界

| 场景 | 处理规则 |
| :--- | :--- |
| 查询时间跨度超过 365 天 | 阻断查询，提示「查询时间跨度不可超过 365 天」 |
| 开始时间晚于结束时间 | 阻断查询，提示「开始时间不能晚于结束时间」 |
| 查询无结果 | 展示暂无数据，不展示新增或调整入口 |
| 来源单号为空 | 展示 `-`，不影响流水有效性；需在后续数据治理中追查来源动作 |
| 重复提交库存动作 | 不重复生成 FL；按幂等键校验 |
| 导出结果为空 | 不展示导出按钮，避免导出空文件 |

## 9. 禁止项

| 禁止项 | 说明 |
| :--- | :--- |
| 新增库存流水 | FL 必须由库存动作自动生成 |
| 编辑库存流水 | 不允许修改历史审计记录 |
| 删除库存流水 | 删除会破坏审计底账 |
| 反向调整库存 | 本页不提供库存调整入口 |
| 状态流转动作 | 本模块无状态机，不存在状态动作按钮 |
