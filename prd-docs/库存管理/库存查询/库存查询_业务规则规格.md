# 库存查询_业务规则规格

> 状态：已补全
> 角色：业务规则规格 | 类型：查询页
> 前置文档：《库存查询主PRD》《库存查询字段清单》
> 说明：本文件只写查询页权限、筛选逻辑、展示规则；不写状态流转表。

## 1. 权限规则

### 1.1 只读权限

| 规则ID | 规则 | 说明 |
|:--|:--|:--|
| AUTH-R01 | 查询页只读 | 用户只能查询、筛选、分页、刷新，不允许维护库存 |
| AUTH-R02 | 无库存调整入口 | 页面不展示“调整库存”“新增库存”“编辑库存”“删除库存”等入口 |
| AUTH-R03 | 无状态操作 | 页面不展示审核、确认、作废、关闭、停用等状态动作 |
| AUTH-R04 | 操作列不展示 | 列表不设置操作列，不跳转新增/编辑/详情页 |

### 1.2 数据可见范围

| 角色/范围 | 可见数据 | 说明 |
|:--|:--|:--|
| 仓管员 | 授权仓库、库区、货位范围内库存 | 具体授权模型 context 未明确，以系统权限配置为准 |
| 仓库主管 | 授权仓库范围内库存 | 可查看该仓全部库区和货位 |
| 管理员/产品/测试复核 | 按账号授权范围查看 | 测试环境可配置全仓只读权限 |

后端必须先应用 `permissionScope`，再叠加用户输入筛选条件，避免通过接口参数越权查询其他仓库库存。

## 2. 筛选逻辑

| 规则ID | 筛选项 | 规则 |
|:--|:--|:--|
| FILTER-R01 | 仓库 | 按 `warehouseCode` 精确匹配；仓库下拉仅展示授权仓库 |
| FILTER-R02 | 库区 | 按 `zoneCode` 精确匹配；选择仓库后仅展示该仓库下库区 |
| FILTER-R03 | 货位 | 支持 `locationCode` 或 `locationBarcode` 精确匹配；PDA 扫码优先 |
| FILTER-R04 | 商品 | 商品编码/条码精确匹配，商品名称支持模糊匹配 |
| FILTER-R05 | 批次 | `batchNo` 支持模糊匹配；无批次商品展示为 `-` |
| FILTER-R06 | 不显示零库存 | `hideZeroStock = true` 时过滤 `qtyOnHand <= 0` 的行 |
| FILTER-R07 | 分页 | PC 默认 `pageSize = 20`，支持 20/50/100；页码从 1 开始 |
| FILTER-R08 | 排序 | 默认按 `lastChangedAt DESC`，再按 `warehouseCode, locationCode, productCode` 升序 |

### 2.1 筛选执行顺序

1. 读取用户账号权限，生成授权仓库/库区/货位范围。
2. 应用用户输入的仓库、库区、货位、商品、批次条件。
3. 计算或读取库存余额五个数量：现存、占用、可用、冻结、在途。
4. 若开启不显示零库存，则过滤 `qtyOnHand <= 0`。
5. 按默认排序分页返回。

### 2.2 PC/PDA 差异

| 端 | 查询方式 | 规则 |
|:--|:--|:--|
| PC | 筛选区组合条件 | 支持仓库、库区、货位、商品、批次、零库存过滤、分页 |
| PDA | 扫码 + 精简筛选 | 以扫码货位、扫码商品为主，可保留仓库筛选；返回字段口径与 PC 一致 |

## 3. 展示规则

### 3.1 数量口径

| 规则ID | 字段 | 规则 |
|:--|:--|:--|
| QTY-R01 | 现存 | `现存 = 期初 + Σ入库 - Σ出库 + Σ盘盈 - Σ盘亏` |
| QTY-R02 | 占用 | 销售订单审核等锁库动作形成占用；占用不直接改变现存 |
| QTY-R03 | 冻结 | 质检、盘点、报损等触发冻结；冻结参与可用扣减 |
| QTY-R04 | 可用 | `可用 = 现存 - 占用 - 冻结` |
| QTY-R05 | 在途 | 调拨途中数量单独展示；不参与可用公式 |

### 3.2 聚合粒度

| 规则ID | 规则 | 说明 |
|:--|:--|:--|
| GROUP-R01 | 默认明细粒度 | 按 `warehouseCode + zoneCode + locationCode + productCode + batchNo` 聚合 |
| GROUP-R02 | 同 SKU 多批次 | 不同批次必须拆成多行 |
| GROUP-R03 | 同 SKU 多货位 | 不同货位必须拆成多行 |
| GROUP-R04 | 无批次商品 | `batchNo` 展示 `-`，但仍参与记录唯一键 |

### 3.3 样式与空态

| 规则ID | 场景 | 展示规则 |
|:--|:--|:--|
| VIEW-R01 | 空值 | 展示 `-` |
| VIEW-R02 | 数量 | 整数，右对齐 |
| VIEW-R03 | 可用为负 | 数值标红，仅提示异常，不提供调整入口 |
| VIEW-R04 | 空结果 | 展示“当前筛选条件下暂无库存记录” |
| VIEW-R05 | 按钮不可用 | 遵守通用规范：按钮不可用时隐藏，不展示灰色 disabled 态 |

## 4. 接口/数据一致性规则

| 规则ID | 规则 | 验证点 |
|:--|:--|:--|
| DATA-R01 | 查询接口不得写库存 | 查询接口只读，不生成 `FL`，不改变库存余额 |
| DATA-R02 | 最近流水只读展示 | `lastFlNo` 仅用于追溯最近一次库存变动 |
| DATA-R03 | 公式一致性 | 返回行必须满足 `qtyAvailable = qtyOnHand - qtyAllocated - qtyFrozen` |
| DATA-R04 | 单号格式 | 若展示库存流水号，必须符合 `FL{YYYYMMDD}-{8位序号}` |
| DATA-R05 | 时间格式 | 时间统一 `YYYY-MM-DD HH:mm:ss` |

## 5. 禁止项

- 禁止在本页面设计草稿、待审核、已确认等状态。
- 禁止设计状态 Tab。
- 禁止设计新增页、编辑页、详情页。
- 禁止设计库存调整、反审核、作废、删除等动作。
- 禁止通过前端手工修改 `qtyOnHand`、`qtyAllocated`、`qtyAvailable`、`qtyFrozen`、`qtyInTransit`。
