# 商机_Demo_详情页

> **版本**：V2.0 | 2026-07-18
> **页面类型**：PC Web 详情页
> **前置文档**：《商机主PRD》《商机字段清单》
> **字段约束**：字段定义与展示口径引用《商机字段清单》，本文只定义页面结构和交互。

---

## 1. 页面概述

| 项目 | 内容 |
|------|------|
| 页面名称 | 商机详情 |
| 页面路径 | `/opportunities/{id}` |
| 页面目标 | 汇总商机阶段、预测、商品、跟进、合同与 ERP 联动结果 |
| 进入方式 | 列表编号、看板卡片、客户关联商机、全局链接 |
| 数据权限 | 引用《商机主PRD》数据可见范围 |
| 页面骨架 | 页头状态区 → 阶段时间线 → 信息区 → 商品明细 → 跟进记录 → 关联结果 → 底部操作栏 |
| 刷新策略 | 首次加载 + 操作成功后局部刷新 |

详情页设计原则：

- 全部业务字段只读展示。
- 状态变化只通过底部动作或合法外部事件触发。
- 当前阶段、AI 结果与外部同步状态在首屏可见。
- 信息卡片采用 KV 布局，不放置可编辑控件。
- 商品、跟进和关联结果使用独立全宽区块。
- 页面最大内容宽度 1280px。

## 2. 加载与权限

### 2.1 初始化顺序

1. 校验商机查看权限。
2. 拉取商机主数据与当前版本。
3. 并行拉取阶段历史、商品明细、跟进记录。
4. 并行拉取合同和 ERP 订单 Mock 结果。
5. 拉取最新 AI 预测快照。
6. 根据状态和权限计算底部动作。

### 2.2 页面状态

| 场景 | 表现 | 操作 |
|------|------|------|
| 首次加载 | 固定尺寸骨架屏 | 等待 |
| 商机不存在 | 404 状态页 | 返回列表 |
| 无权限 | 403 状态页 | 返回列表 |
| 主数据失败 | 整页错误态 | 重试、返回列表 |
| 子区块失败 | 区块错误态 | 局部重试 |
| 外部同步中 | 状态提示 + 刷新入口 | 手动刷新 |

## 3. 页头状态区

### 3.1 左侧信息

- 返回按钮使用 ArrowLeft 图标。
- 主标题展示商机名称。
- 副标题展示商机编号。
- 商机名称最多两行。
- 编号支持一键复制。
- 返回时恢复列表或看板上下文。

### 3.2 右侧状态

| 元素 | 展示规则 | 交互 |
|------|----------|------|
| 商机阶段 | Badge，样式引用字段清单 | 不可点击修改 |
| AI 成交概率 | 环形进度 + 数字 | 悬浮展示预测更新时间 |
| AI 状态 | 最新、计算中、失败降级 | 失败时可重试 |
| ERP 同步 | 仅赢单后展示 | 可查看订单草稿结果 |

### 3.3 AI 摘要

- 展示最新成功预测的解释摘要。
- 展示预测生成时间。
- 重新计算时保留上次成功值。
- 无历史结果且失败时显示 `预测暂不可用`。
- 有权限时显示 RefreshCw 图标重试。
- 重试不改变商机阶段。
- 高概率场景展示库存预占通知结果。

## 4. 七节点阶段时间线

### 4.1 节点定义

时间线固定展示七个节点：

1. 初步接触。
2. 需求确认。
3. 方案报价。
4. 商务谈判。
5. 合同签订。
6. 赢单。
7. 输单。

> 节点业务编码和状态口径引用《商机字段清单》；本节只定义视觉和交互。

### 4.2 节点视觉状态

| 节点状态 | 视觉 | 时间展示 | 说明 |
|----------|------|----------|------|
| 已到达 | 实心圆 + 主色连线 | 展示到达时间 | 历史阶段 |
| 当前 | 描边加粗 + 当前状态色 | 展示最近变更时间 | 非终态当前节点 |
| 未到达 | 灰色空心圆 | `--` | 尚未推进 |
| 赢单终态 | 绿色 CheckCircle | 展示赢单时间 | 正常成交分支 |
| 输单终态 | 红色 XCircle | 展示输单时间 | 关闭分支 |
| 回退后历史合同 | 节点附历史标识 | 展示历史到达时间 | 合同作废后回退谈判 |

### 4.3 时间线布局

- 横向排列，固定高度 112px。
- 每个节点等宽。
- 节点名称允许两行但不改变总高度。
- 时间使用两行显示日期和时分。
- 宽度不足时容器横向滚动。
- 不使用随视口缩放的字体。
- 赢单和输单从合同前的主链路分叉展示。

### 4.4 分支规则

- 非终态时高亮当前主链路节点。
- 赢单时高亮主链路全部到达节点和赢单节点。
- 输单时高亮输单前实际到达节点与输单节点。
- 未实际到达的后续主链路节点保持灰色。
- 输单节点不因当前非终态而显示为可点击。
- 时间线仅展示，不触发推进。
- 合同作废回退后，合同节点保留“历史到达”标识。

### 4.5 时间线详情

- 点击已到达节点打开阶段变更 Popover。
- Popover 展示动作、操作者、时间和结果。
- 失败尝试不作为阶段节点，但可在审计日志查看。
- Popover 点击外部关闭。
- 无阶段日志时显示 `暂无变更记录`。

## 5. 商机信息卡片

### 5.1 卡片结构

- 标题为 `商机信息`。
- 使用 Description KV 两列布局。
- 每行两个 KV。
- 长文本独占整行。
- 标签宽度 120px。
- 值区域最小高度 24px。
- 空值统一显示 `--`。

### 5.2 KV 清单

| 字段名 | 展示形态 | 交互 |
|--------|----------|------|
| 商机编号 | 等宽文本 | 复制 |
| 商机名称 | 普通文本 | 无 |
| 关联客户 | 链接 | 新开当前 CRM 客户详情 |
| 商机阶段 | Tag | 无 |
| 预计金额 | 格式化金额 | 无 |
| 预计成交日期 | 日期 | 无 |
| AI 成交概率 | 进度 + 数字 | 查看预测摘要 |
| 需求描述 | 保留换行文本 | 展开/收起 |
| 合同编号 | 链接或 `--` | 存在时进入合同详情 |
| 输单原因 | 警示文本或 `--` | 仅输单后显示真实内容 |
| 创建人 | 用户名称 | 无 |
| 创建时间 | 完整时间 | 无 |
| 阶段变更时间 | 完整时间 | 无 |
| 赢单时间 | 完整时间或 `--` | 无 |

### 5.3 展示约束

- 不在详情页重复解释字段定义。
- 合同编号不存在时不展示链接样式。
- 输单原因仅在对应终态突出展示。
- 需求描述默认最多显示 6 行。
- 超过 6 行显示“展开全部”。
- 收起后滚动位置不跳动。

## 6. 商品明细表格

### 6.1 区块头

| 元素 | 内容 |
|------|------|
| 标题 | `商品明细` |
| 数量 | `共 N 项` |
| 数据来源 | `ERP 商品快照` |
| 刷新 | 只刷新商品可用性，不改历史价格快照 |

### 6.2 表格列

| 列名 | 宽度 | 固定 | 对齐 | 展示规则 |
|------|------|------|------|----------|
| 商品编码 | 168px | 左 | 左 | 等宽文本 |
| 商品名称 | 240px | 否 | 左 | 两行省略，Tooltip 全文 |
| 参考单价 | 140px | 否 | 右 | 格式化金额 |
| 数量 | 100px | 否 | 右 | 数字 |
| 小计 | 148px | 否 | 右 | 格式化金额 |
| ERP 可用性 | 120px | 否 | 中 | 当前可用或历史失效提示 |

### 6.3 表格行为

- 表格只读。
- 行高 52px。
- 商品编码左固定。
- 不展示操作列。
- 明细底部展示合计。
- 合计与预计金额不一致时展示弱提示。
- 商品已失效时保留历史快照，不隐藏整行。
- 商品查询失败不影响其他详情区块。
- 无商品时显示 `暂未关联商品`。

## 7. 跟进记录组件

### 7.1 区块结构

| 元素 | 规格 |
|------|------|
| 标题 | `跟进记录` |
| 计数 | 后台返回总条数 |
| 新增按钮 | `添加跟进`，有权限且非终态时展示 |
| 排序 | 跟进时间倒序 |
| 加载 | 首屏 10 条，支持加载更多 |

### 7.2 筛选控件

| 控件 | Placeholder | 行为 |
|------|-------------|------|
| 跟进方式 | `全部方式` | 选项引用跟进记录 SSOT |
| 跟进人 | `全部跟进人` | 主管可筛团队成员 |
| 时间范围 | `开始日期 - 结束日期` | 按跟进时间过滤 |

### 7.3 时间线单项

每条记录固定展示：

- 跟进方式图标。
- 跟进人。
- 跟进时间。
- 跟进内容。
- 下次计划（存在时）。
- 来源对象为当前商机。
- 由拜访完成自动生成时显示 `来自拜访` 标识。

时间线视觉：

- 左侧时间轴宽度固定 24px。
- 内容区不使用嵌套卡片。
- 内容最多展示 6 行，支持展开。
- 自动生成记录为只读。
- 手工记录是否可编辑引用跟进记录规则。

### 7.4 添加跟进抽屉

- 点击后打开右侧抽屉。
- 默认关联当前商机且不可更换。
- 字段约束引用跟进记录 SSOT。
- 保存成功后关闭抽屉。
- 新记录插入时间线顶部。
- 保存后触发 AI 商机预测刷新。
- 保存失败保留输入内容。
- 终态商机不展示添加入口。

### 7.5 空态与异常

- 无记录显示 `暂无跟进记录`。
- 有权限且非终态时空态提供“添加跟进”。
- 查询失败显示区块级“重新加载”。
- AI 预测刷新失败不影响跟进保存成功反馈。

## 8. 关联合同与 ERP 结果

### 8.1 合同摘要

| 场景 | 展示 |
|------|------|
| 未发起 | `尚未发起合同` |
| 草稿或待签署 | 合同编号、合同名称、合同状态、查看入口 |
| 已签署 | 合同摘要 + 签署完成提示 |
| 已归档 | 合同摘要 + 归档提示 |
| 已作废 | 历史合同摘要 + 作废提示 + 商机回退说明 |

合同区块规则：

- 合同数据只读。
- 不在商机详情内编辑合同。
- 查看合同在当前 CRM 新路由打开。
- 合同创建失败时显示“重试发起”。
- 合同作废后保留历史访问入口。

### 8.2 ERP 订单草稿摘要

| 同步状态 | 展示 | 操作 |
|----------|------|------|
| 未触发 | 不展示区块 | 无 |
| 处理中 | 加载图标 + 请求时间 | 刷新 |
| 成功 | ERP 订单编号、同步时间 | 查看 Mock 详情 |
| 失败 | 错误摘要 | 主管或管理员重试 |

ERP 规则：

- 仅赢单后展示订单同步区块。
- 重试使用同一业务幂等键。
- 成功后不再展示重试。
- 重复回调不生成第二个订单草稿。
- CRM 不展示 WMS 直接操作入口。

## 9. 底部操作矩阵

### 9.1 操作栏布局

- 固定在页面底部。
- 左侧为“返回列表”。
- 右侧按“次要动作→主要动作→危险动作”排列。
- 同时最多展示 4 个文本按钮。
- 超出动作放入 More 菜单。
- 无权限动作不渲染。

### 9.2 阶段动作矩阵

| 当前阶段 | 左侧 | 右侧主操作 | 右侧次操作 | More |
|----------|------|------------|------------|------|
| 初步接触 | 返回列表 | 推进 | 编辑、添加跟进 | 输单 |
| 需求确认 | 返回列表 | 推进 | 编辑、添加跟进 | 输单 |
| 方案报价 | 返回列表 | 推进 | 添加跟进 | 输单 |
| 商务谈判 | 返回列表 | 发起合同 | 添加跟进 | 输单 |
| 合同签订 | 返回列表 | 查看合同 | 添加跟进 | 输单 |
| 赢单 | 返回列表 | 查看 ERP 订单 | 无 | 无 |
| 输单 | 返回列表 | 无 | 无 | 无 |

### 9.3 动态条件

- “编辑”仅在主 PRD 动作矩阵允许时展示。
- “推进”只对应唯一下一阶段。
- “发起合同”仅在不存在有效合同时展示。
- 合同创建失败时主按钮变为“重试发起合同”。
- “查看合同”在存在有效或历史合同时展示。
- “查看 ERP 订单”仅在同步成功时可跳转。
- ERP 同步失败时主管或管理员展示“重试同步”。
- “输单”在任一非终态按权限展示。
- 终态不展示编辑、推进、跟进或输单。

## 10. 动作交互与弹窗

### 10.1 推进

| 项目 | 规格 |
|------|------|
| 标题 | `推进商机` |
| 内容 | `确认将商机推进至“{目标阶段}”吗？` |
| 取消 | `取消` |
| 确认 | `确认推进` |
| 成功 | 刷新时间线、信息区、AI 区和操作栏 |
| 失败 | 保持原状态并展示具体原因 |

### 10.2 发起合同

| 项目 | 规格 |
|------|------|
| 标题 | `发起合同` |
| 内容 | `系统将创建合同草稿并推进商机，是否继续？` |
| 取消 | `暂不发起` |
| 确认 | `创建合同` |
| 成功 | 进入合同阶段并展示合同摘要 |
| 失败 | 保持谈判阶段并显示重试入口 |

### 10.3 输单

| 项目 | 规格 |
|------|------|
| 标题 | `确认输单` |
| 内容 | `输单后商机将关闭且不可恢复，请填写原因。` |
| 字段 | 输单原因 |
| Placeholder | `请输入具体输单原因，便于后续复盘` |
| 取消 | `取消` |
| 确认 | `确认输单`，危险样式 |
| 校验 | 引用字段清单；失败时聚焦输入框 |

### 10.4 并发变化

- 动作提交携带详情版本号。
- 版本冲突时关闭提交加载态。
- Toast `商机状态已变化，已刷新最新数据`。
- 页面重新加载主数据和动作矩阵。
- 不重复执行原动作。

## 11. Mock 详情数据

### 11.1 主场景

- 使用一条处于商务谈判的商机。
- 关联一个有效客户。
