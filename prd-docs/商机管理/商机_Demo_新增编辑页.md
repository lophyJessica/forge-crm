# 商机_Demo_新增编辑页

> **版本**：V2.0 | 2026-07-18
> **页面类型**：PC Web 新增页 / 编辑页
> **前置文档**：《商机主PRD》《商机字段清单》
> **字段约束**：字段定义、必填性、格式、字符限制和取值范围以《商机字段清单》为唯一权威源。

---

## 1. 页面概述

| 项目 | 新增模式 | 编辑模式 |
|------|----------|----------|
| 页面名称 | 新建商机 | 编辑商机 |
| 页面路径 | `/opportunities/new` | `/opportunities/{id}/edit` |
| 进入方式 | 列表主按钮、客户详情快捷入口 | 列表或详情的“编辑”动作 |
| 页面目标 | 创建一条处于首阶段的商机 | 修改当前阶段允许维护的信息 |
| 提交结果 | 生成商机并触发首次 AI 预测 | 保存变更，不直接改变阶段 |
| 页面骨架 | 页头 + 表单内容 + 固定底部操作栏 | 页头 + 状态提示 + 表单内容 + 固定底部操作栏 |

页面设计原则：

- 状态变更不得通过表单字段完成。
- 商机阶段只在编辑页展示为只读信息。
- 客户和商品只引用权威数据，不提供现场新建主数据入口。
- 页面只保存商机信息，不执行“推进”或“输单”。
- 所有校验以字段清单 SSOT 和主 PRD 业务规则为准。
- 表单最大内容宽度 1040px，居中显示。
- 桌面端两列栅格，长文本字段独占整行。

## 2. 路由与初始化

### 2.1 新增模式初始化

- 进入页面后加载当前用户权限。
- 加载可见客户远程搜索能力。
- 加载 ERP 商品 Mock 查询能力。
- 从列表进入时表单为空。
- 从客户详情进入时预填关联客户。
- 快捷进入的关联客户默认锁定，提供“解除预填”动作。
- 不在前端预生成商机编号。
- 不展示商机阶段可编辑控件。
- 首屏加载失败时显示整页错误态。

### 2.2 编辑模式初始化

- 根据路由标识加载商机详情。
- 先校验查看权限，再校验编辑权限。
- 展示商机编号和商机阶段只读摘要。
- 仅允许主 PRD 动作矩阵规定的阶段进入编辑模式。
- 不可编辑阶段直接访问编辑 URL 时跳转详情页。
- 跳转后 Toast `当前阶段不可编辑`。
- 已失效客户或商品保留历史文本并显示警示。
- 加载失败时提供“重试”和“返回列表”。

### 2.3 路由参数

| 参数 | 来源 | 用途 | 异常处理 |
|------|------|------|----------|
| `id` | 编辑页路径 | 查询商机详情 | 不存在时进入 404 |
| `customerId` | 客户详情快捷入口 | 预填关联客户 | 无权或失效时忽略并提示 |
| `from` | 来源页面 | 返回时恢复上下文 | 非法值回到商机列表 |

## 5. 商品明细工具区

### 5.1 区域结构

| 元素 | 规格 |
|------|------|
| 标题 | `关联商品` |
| 辅助文案 | `商品与价格信息来自 ERP` |
| 添加按钮 | `添加商品`，Plus 图标 |
| 推荐入口 | `AI 推荐`，Sparkles 图标 |
| 空态 | `暂未关联商品` |

### 5.2 添加商品抽屉

- 抽屉宽度 720px。
- 支持按商品编码或名称搜索。
- 查询结果来自 ERP Mock。
- 已选择商品在结果中显示“已添加”。
- 已失效商品不允许选择。
- 支持多选后一次确认。
- 点击“确认添加”后写入前端表单态。
- 关闭抽屉不保存本次未确认选择。
- ERP 查询失败显示重试入口。

### 5.3 商品明细表格

| 列 | 宽度 | 编辑方式 | 规则 |
|----|------|----------|------|
| 商品编码 | 160px | 只读 | 来自 ERP |
| 商品名称 | 220px | 只读 | 超长省略 |
| 参考单价 | 140px | 只读 | 来自 ERP 快照 |
| 数量 | 120px | NumberInput | 校验引用关联明细约束 |
| 小计 | 140px | 系统计算 | 只读展示 |
| 操作 | 80px | Trash2 图标 | 移除当前行 |

商品联动：

- 修改数量后即时重算小计与明细合计。
- 明细合计只提供参考，不自动覆盖预计金额。
- 当明细合计与预计金额不一致时显示弱提示。
- 删除商品只移除表单态，不删除 ERP 商品。
- 删除最后一行后恢复空态。
- 推进前置所需的商品约束由主 PRD 校验。

### 5.4 AI 推荐商品

- 点击后基于当前客户和需求描述请求推荐。
- 未选择客户时阻断并聚焦关联客户。
- 请求中按钮显示加载态。
- 返回结果在抽屉顶部“推荐”分组展示。
- 推荐只作建议，用户勾选后才加入。
- 推荐失败显示 `推荐服务暂不可用，可手动选择商品`。
- 推荐失败不影响保存。

## 6. 字段联动规则

| 联动编号 | 触发字段/事件 | 联动目标 | 行为 |
|----------|---------------|----------|------|
| L01 | 从客户详情进入 | 关联客户 | 自动预填并标记来源 |
| L02 | 关联客户变化 | 关联商品 | 已选商品保留，但提示重新核对匹配度 |
| L03 | 关联客户变化 | AI 推荐 | 清空旧推荐缓存 |
| L04 | 需求描述变化 | AI 推荐 | 下次打开时重新请求 |
| L05 | 预计金额变化 | AI 预测 | 保存成功后触发新预测 |
| L06 | 预计成交日期变化 | AI 预测 | 保存成功后触发新预测 |
| L07 | 商品明细变化 | 预计金额 | 仅提示差异，不自动覆盖 |
| L08 | 保存成功 | AI 预测 | 异步触发，页面不等待结果完成 |

联动通用约束：

- 联动不得静默覆盖用户手工输入。
- 任何自动清空动作必须先提示。
- AI 结果不得自动修改业务字段。
- 外部数据失败不清空现有表单内容。

## 9. 保存交互规范

### 9.1 校验顺序

1. 当前页面权限。
2. 当前商机可编辑状态。
3. 字段清单必填与格式校验。
4. 客户快照有效性校验。
5. 商品引用有效性校验。
6. 版本号并发校验。
7. 提交保存。

### 9.2 校验失败

- 所有错误字段同时标红。
- 页面滚动并聚焦第一个错误字段。
- 顶部不重复堆叠同类错误。
- Toast 使用 `请检查并完善表单信息`。
- 商品错误定位到商品明细行。
- 远程对象失效时提供重新选择入口。

### 9.3 保存成功

- 新增成功 Toast `商机创建成功`。
- 新增成功跳转商机详情页。
- 编辑成功 Toast `商机信息已更新`。
- 编辑成功返回商机详情页。
- 详情页展示保存后的最新数据。
- 创建后 AI 预测异步运行，不阻塞跳转。

### 9.4 保存失败

- 网络失败保留全部表单内容。
- 服务端校验失败映射到具体字段。
- 并发冲突弹窗提示 `商机已被他人更新`。
- 并发弹窗按钮为 `重新加载` 和 `保留当前页面`。
- 重复提交返回首次成功结果。
- 外部 ERP 查询失败不清除已选历史商品。
