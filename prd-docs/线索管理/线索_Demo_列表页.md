# 线索_Demo_列表页

> **文档定位**：AI 生成线索列表页前端原型的输入文档，聚焦页面结构、控件类型、交互行为。
> **字段定义**：字段枚举值、必填性等见《线索字段清单》，本文件只写页面呈现规则，严禁重复定义。
> **版本**：V1.0 | 2026-07-17

---

## 1. 页面概述

| 项目 | 内容 |
|------|------|
| 页面名称 | 线索池 |
| 页面目标 | 帮助销售查看自己名下的线索，或从公海认领新线索 |
| 页面路径 | `/leads` |
| 骨架结构 | 顶部 Tab 栏 + 查询卡片 + 工具条 + 表格卡片 |

---

## 2. 状态 Tab 栏

| Tab 标签 | 过滤逻辑 |
|----------|---------|
| 全部 | 展示所有状态（默认选中） |
| 待分配 | PENDING_ASSIGN（待分配+标注分池） |
| 我的线索 | ASSIGNED + FOLLOWING（当前登录销售名下的） |
| 公海 | PENDING_ASSIGN + ABANDONED超7天（可认领的线索） |
| 已转客户 | CONVERTED |
| 已作废 | ABANDONED |

**Tab 交互规则**：
- 每个 Tab 右侧展示计数（格式：`Tab名 N`，超过 99 展示 `99+`）
- 切换 Tab 后列表刷新，页码重置，不清空查询条件
- 「我的线索」Tab 仅对销售角色可见

---

## 3. 查询区

### 布局

桌面端 6 列栅格。右侧操作组：`展开/收起` → `重置` → `搜索`。

### 查询条件

| 字段 | 控件类型 | 规格说明 |
|------|---------|---------|
| 关键词 | BatchSearchInput | 支持公司名称/联系人/手机号多值搜索，换行或逗号分隔 |
| 线索来源 | Select | ONLINE/ACTIVITY/EXHIBITION/REFERRAL/IMPORT/OTHER |
| 所属行业 | Select | MANUFACTURING/RETAIL/HEALTHCARE/FINANCE/IT/OTHER |
| AI评分 | Range | 最小值-最大值区间，0-100 |
| 归属人 | SelectWithSearch | 销售列表，搜索姓名 |
| 创建时间 | DateRange | 快捷选项：今天/本周/本月 |

## 5. 表格

### 表格行为

| 行为 | 规格 |
|------|------|
| 行勾选 | 支持（批量分配/批量作废时使用） |
| 列宽拖拽 | 支持 |
| 固定列 | 线索编号左固定；操作列右固定 |
| 分页 | 默认每页 20 条，支持切换（20/50/100） |
| 空状态 | 展示「暂无数据」插图 |

### 列定义

| 列名 | 内容说明 | 宽度 | 固定 |
|------|---------|------|------|
| 勾选列 | 复选框 | 48px | 左固定 |
| 线索编号 | 可点击进入详情页，链接色 | 180px | 左固定 |
| 公司名称 | 文本 | 180px | |
| 联系人 | 文本 | 100px | |
| 手机号 | 文本 | 140px | |
| 邮箱 | 文本 | 180px | |
| 线索来源 | 文本 Tag | 100px | |
| AI评分 | 数字 Tag（配色见下） | 80px | |
| 线索状态 | 状态 Tag（配色见下） | 120px | |
| 归属人 | 文本 | 100px | |
| 最近跟进时间 | `YYYY-MM-DD HH:mm` | 160px | |
| 创建时间 | `YYYY-MM-DD HH:mm` | 160px | |
| 操作 | 行内文字动作（按状态动态展示） | 120px | 右固定 |

### 状态 Tag 配色

| 状态 | 颜色语义 |
|------|---------|
| DRAFT 草稿 | 灰色 |
| PENDING_ASSIGN 待分配 | 蓝色 |
| ASSIGNED 已分配 | 橙色 |
| FOLLOWING 跟进中 | 绿色 |
| CONVERTED 已转客户 | 绿色 |
| ABANDONED 已作废 | 红色 |

### AI 评分配色

| 分值 | 颜色 |
|------|------|
| ≥80 | 绿色 |
| 50-79 | 黄色 |
| <50 | 红色 |

### 行内操作（按状态动态展示）

| 状态 | 展示的操作 |
|------|-----------|
| DRAFT 草稿 | 查看 / 编辑 / 删除 |
| PENDING_ASSIGN/公海 | 查看 / 认领（自己非放弃人时） |
| ASSIGNED 已分配 | 查看 / 添加跟进 / 放弃 |
| FOLLOWING 跟进中 | 查看 / 添加跟进 / 转客户 / 放弃 |
| CONVERTED 已转客户 | 查看 |
| ABANDONED 已作废 | 查看 |

## 7. Mock 数据规格

### 状态覆盖要求

| 状态 | 最少条数 | 备注 |
|------|:------:|------|
| PENDING_ASSIGN | 5 | 模拟待分配 |
| ASSIGNED | 3 | 模拟刚分配 |
| FOLLOWING | 8 | 模拟跟进中（主力数据） |
| CONVERTED | 2 | 模拟已转化 |
| ABANDONED | 2 | 模拟已作废 |

**总计**：≥ 20 条

### 数据要求
- 公司名使用虚构名称（如"强盛科技"、"龙腾实业"）
- 线索编号连续：LEAD20260717-0001 ~ LEAD20260717-0020
- AI评分差异化分布：2条≥80、10条50-79、8条<50
- 手机号格式：1380001XXXX
