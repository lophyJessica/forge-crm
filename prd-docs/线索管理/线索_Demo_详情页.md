# 线索_Demo_详情页

> **文档定位**：AI 生成线索详情页前端原型的输入文档
> **字段定义**：字段枚举值、必填性等见《线索字段清单》，本文件只写页面呈现规则，严禁重复定义
> **版本**：V1.0 | 2026-07-17

---

## 1. 页面概述

| 项目 | 内容 |
|------|------|
| 页面名称 | 线索详情 |
| 页面目标 | 查看线索完整信息、AI评分、跟进记录时间线，执行跟进/转客户/放弃等操作 |
| 页面路径 | `/leads/{id}` |
| 骨架结构 | 状态区 → AI评分卡片 → 基本信息卡片 → 跟进记录时间线 → 固定底部操作栏 |

---

## 2. 状态区

- 当前状态 Badge（颜色同列表页配色）
- 线索编号 `LEAD20260717-0001`
- 归属人（如有）
- 分配时间（如有）

---

## 3. AI 评分卡片

| 展示项 | 内容 |
|--------|------|
| 综合评分 | 大号数字（0-100），配色：≥80绿色、50-79黄色、<50红色 |
| 评分维度明细 | 展开查看：来源(15%)、行业匹配(20%)、响应速度(25%)、历史转化(25%)、活跃度(15%) |
| 评分时间 | 最近一次评分时间 |

---

## 4. 基本信息卡片

卡片标题：**基本信息**。Key-Value 布局，4 列桌面端，只读展示。

| 展示字段 |
|---------|
| 线索来源、公司名称、联系人、手机号 |
| 邮箱、职位、所属行业、所在地区 |
| 线索备注 |

---

## 5. 跟进记录时间线

卡片标题：**跟进记录**。

时间线格式：左侧时间轴 + 右侧卡片，按时间倒序。

每条跟进记录展示：

| 字段 | 说明 |
|------|------|
| 跟进时间 | `YYYY-MM-DD HH:mm` |
| 跟进人 | 销售姓名 |
| 跟进方式 | 电话 / 拜访 / 邮件 |
| 跟进内容 | 文本（折叠，超过 3 行展示"展开"） |
| 下次跟进计划 | 文本（如有） |

空状态：暂无跟进记录时展示"暂无跟进记录"占位图。

---

## 6. 底部操作栏（按状态动态展示）

| 状态 | 展示的操作按钮 |
|------|--------------|
| DRAFT 草稿 | `返回` `编辑` `提交` `删除` |
| PENDING_ASSIGN 待分配 | `返回` `认领`（公海时） |
| ASSIGNED 已分配 | `返回` `添加跟进` `放弃` |
| FOLLOWING 跟进中 | `返回` `添加跟进` `转客户` `放弃` |
| CONVERTED 已转客户 | `返回` `查看关联客户` |
| ABANDONED 已作废 | `返回` |

按钮顺序：`返回` 在最左（次要），主操作在右。

---

## 7. 交互规范

### 7.1 添加跟进
- 弹出跟进记录表单 Modal
- 必填：跟进方式、跟进内容
- 选填：下次跟进计划
- 提交后：刷新时间线 + 更新最近跟进时间

### 7.2 转客户
- 二次确认弹窗："确认将「{公司名称}」转为正式客户？转客户后将同步创建 ERP 客户档案"
- 确认后：状态变更 CONVERTED，通知 ERP 建档

### 7.3 放弃
- 二次确认弹窗 + 必填放弃原因 Textarea
- 确认后：状态变更 ABANDONED，记录放弃原因

### 7.4 认领
- 无需二次确认
- 认领后：状态变更 ASSIGNED，归属人变更
