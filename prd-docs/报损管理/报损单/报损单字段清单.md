# 报损单字段清单

> 角色：字段清单(SSOT) | 类型：业务单据类
> 本文件为报损单字段唯一事实来源；主 PRD、业务规则、流程推演和 Demo 页面引用本文件，不重复维护字段口径。

## 1. 字段设计原则

- BL 单号由系统按 `BL{YYYYMMDD}-{4位序号}` 生成，不可编辑。
- 状态字段只读，状态变更必须通过动作按钮触发。
- 来源为调拨差异时，关联 TR 单号、报损原因、商品和报损数量由系统带入，默认不可编辑。
- 商品、仓库、货位信息在明细中快照存储，主数据后续变更不影响历史 BL。
- 数量遵循 `11-字段校验规范`：正整数，`>0`，且不得超过可报损上限。
- 日期格式 `YYYY-MM-DD`，时间格式 `YYYY-MM-DD HH:mm:ss`，空值展示 `-`。
- 备注、驳回原因、其它原因说明上限 200 字符。

## 5. 枚举值

### 5.1 来源 `sourceType`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| MANUAL | 手动 | 仓库人员手动创建报损 |
| TRANSFER_DIFF | 调拨差异 | TR 到货差异自动生成 |

### 5.2 报损状态 `status`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| DRAFT | 草稿 | 手动创建后尚未提交 |
| PENDING_REVIEW | 待审核 | 等待主管/财务审核 |
| APPROVED | 已审核 | 已审核通过并完成核销过账 |
| REJECTED | 已驳回 | 审核不通过，未核销现存 |

### 5.3 报损类型 `damageType`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| GOODS_DAMAGE | 货物损坏 | 包装破损、商品损坏等现场报损 |
| EXPIRED | 过期 | 商品过期或不可销售 |
| OTHER | 其它 | 需填写原因说明 |
| TRANSFER_LOSS | 调拨损耗 | 调拨到货差异自动生成 |

### 5.4 报损原因 `damageReason`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| DAMAGED | 损坏 | 手动报损原因 |
| EXPIRED | 过期 | 手动报损原因 |
| OTHER | 其它 | 手动报损原因，需填写说明 |
| TRANSFER_LOSS | 调拨损耗 | 调拨差异自动生成，固定原因 |

### 5.5 冻结状态 `freezeStatus`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| NOT_FROZEN | 未冻结 | 草稿阶段 |
| FROZEN | 已冻结 | 待审核阶段，报损数量不可出库 |
| RELEASED | 已释放 | 已驳回，冻结解除 |
| WRITTEN_OFF | 已核销 | 已审核，现存已扣减 |

## 6. 计算字段

| 字段 | 公式 | 说明 |
|:--|:--|:--|
| 可用 | `现存 - 占用 - 冻结` | 沿用 `06-库存管理规则` |
| 报损总数 | `Σ damageQty` | 头部汇总 |
| 待审核冻结后可用 | `qtyOnHandBefore - qtyAllocatedBefore - (qtyFrozenBefore + damageQty)` | 草稿不计算本单冻结 |
| 核销后现存 | `qtyOnHandBefore - damageQty` | 仅已审核后写入 |
| 核销后冻结 | `qtyFrozenBefore` | 本单冻结随核销释放，不保留在冻结口径 |

## 7. 校验时机

| 时机 | 校验范围 |
|:--|:--|
| 保存草稿 | 报损类型、原因、仓库、货位、商品、数量基础校验 |
| 提交审核 | 必填字段、数量上限、可冻结数量、审核人权限 |
| 自动生成 BL | 来源 TR、差异数量、原因固定为调拨损耗、幂等键 |
| 审核通过并确认核销 | 状态为待审核、审核权限、冻结数量存在、未重复核销 |
| 审核驳回 | 状态为待审核、审核权限、驳回原因必填 |

## 8. 不确定性

| 项 | 当前字段口径 |
|:--|:--|
| 批次管理 | context 未展开批次规则，`batchNo` 作为可空快照字段保留 |
| 报损冻结 FL | 库存流水模块定义报损锁库可生成 `FREEZE`，本字段清单保留 `freezeFlowNos`，是否强制展示待复核 |
| 调拨差异货位 | context 未明确短少货物归属货位，字段保留来源货位或差异待核销货位快照 |
