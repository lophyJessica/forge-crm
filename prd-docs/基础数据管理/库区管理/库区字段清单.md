# 库区字段清单

> 版本：V1.0 | 更新时间：2026-07-07
> 文档定位：库区字段的 SSOT。主 PRD、前端 Demo PRD、测试用例和下游单据只引用本清单字段口径，不重复定义枚举、必填性和校验规则。
> 适用范围：WMS 基础数据管理 / 库区管理。

---

## 1. 字段总览

库区隶属于仓库。`warehouseCode`、`warehouseName` 必须来源于仓库档案，不在库区管理中自建仓库。

| 字段名称 | Field | 类型 | 来源 | 必填 | 枚举 / 取值 | 校验规则 | 页面使用 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 库区编码 | `code` | String | 用户填写 | 是 | - | 字母 + 数字；全局唯一；保存时转大写；不可重复；编辑态只读 | 新增、编辑只读、列表、详情、候选展示 | 库区在 WMS 内的唯一标识 |
| 库区名称 | `name` | String | 用户填写 | 是 | - | 非空；去首尾空格；建议 1-50 字符 | 新增、编辑、列表、详情、候选展示 | 库区业务名称 |
| 库区类型 | `type` | Enum | 用户选择 | 是 | 见“2. 枚举定义” | 必须命中 `ZoneType`；不可为空；不得扩展五类之外类型 | 新增、编辑、列表、详情、筛选 | 区分收货、存储、拣货、退货、发货 |
| 所属仓库编码 | `warehouseCode` | String | 仓库档案 | 是 | 仓库档案 `code` | 必须选择仓库档案中存在且启用的仓库；不能为空 | 新增、编辑、列表、详情、候选过滤 | 保存库区与仓库的隶属关系 |
| 所属仓库名称 | `warehouseName` | String | 仓库档案 | 是 | 仓库档案 `name` | 由 `warehouseCode` 带出，不允许手工录入 | 列表、详情、候选展示 | 用于页面展示；仓库改名同步策略见不确定性 |
| 备注 | `remark` | Text | 用户填写 | 否 | - | 遵通用备注规则，上限 200 字符 | 新增、编辑、详情 | 补充说明，不作为系统判断条件 |
| 状态 | `status` | Enum | 系统生成 / 动作按钮变更 | 是 | 见“2. 枚举定义” | 新增默认 `ENABLED`；仅允许通过启用 / 停用按钮变更 | 列表、详情、筛选、候选过滤 | 控制库区是否可进入新业务候选范围 |
| 创建时间 | `createdAt` | DateTime | 系统生成 | 是 | `YYYY-MM-DD HH:mm:ss` | 新增保存时写入；不允许手工编辑；Demo Mock 使用 2026 年时间 | 详情、Mock 数据 | 首次建档时间 |
| 最后修改时间 | `updatedAt` | DateTime | 系统生成 | 是 | `YYYY-MM-DD HH:mm:ss` | 编辑或状态变更后写入；不允许手工编辑；Demo Mock 使用 2026 年时间 | 详情、Mock 数据 | 最近一次修改时间 |

## 2. 枚举定义

### 2.1 库区类型 `ZoneType`

`ZoneType` 严格按 `context/03-主要功能模块和清单.md` 五类定义，不增加其它类型。

| 枚举值 | 展示文案 | 说明 |
| :--- | :--- | :--- |
| `RECEIVING` | 收货区 | 对应收货、待验、待上架前的接收区域 |
| `STORAGE` | 存储区 | 对应常规库存存放区域 |
| `PICKING` | 拣货区 | 对应出库拣货区域 |
| `RETURN` | 退货区 | 对应退货、异常退回处理区域 |
| `SHIPPING` | 发货区 | 对应复核、包装后待交运区域 |

### 2.2 状态 `BaseDataStatus`

| 枚举值 | 展示文案 | 是否可被新业务引用 | 说明 |
| :--- | :--- | :---: | :--- |
| `ENABLED` | 启用 | 是 | 可被货位、新建业务单据、查询筛选选择 |
| `DISABLED` | 停用 | 否 | 不可被新业务选择，但历史数据仍可展示 |

## 3. 校验规则

| 校验对象 | 规则 | 触发时机 | 失败提示 |
| :--- | :--- | :--- | :--- |
| 必填字段 | 库区编码、库区名称、库区类型、所属仓库不能为空 | 提交保存 | 按字段提示“不能为空” |
| 库区编码格式 | 仅允许字母 + 数字，保存时统一转大写 | 失焦、提交保存 | 库区编码仅支持字母和数字 |
| 库区编码唯一 | `code` 在库区档案内全局唯一 | 提交保存 | 库区编码已存在 |
| 库区编码编辑 | 编辑已有库区时 `code` 只读，不进入保存变更 | 页面渲染 | 不提供可编辑入口 |
| 库区类型 | 必须命中 `ZoneType` 五类之一 | 提交保存 | 请选择有效的库区类型 |
| 所属仓库 | `warehouseCode` 必须来自仓库档案，且仓库状态为启用 | 选择、提交保存、启用 | 请选择启用状态的仓库 |
| 备注长度 | `remark` 不超过 200 字符 | 输入、提交保存 | 备注不能超过 200 字符 |
| 状态变更 | `status` 不允许表单直接修改，只允许动作按钮触发 | 页面交互 | 不展示状态下拉 |
| 日期时间 | 系统字段使用 `YYYY-MM-DD HH:mm:ss` | 写入、展示 | 格式异常时展示 `-` |

校验时机遵循《11-字段校验规范》：格式 / 范围可失焦校验，提交保存做全量严格校验；空值展示遵循通用规范使用 `-`。

## 4. 查询与展示规则

| 场景 | 字段 | 规则 |
| :--- | :--- | :--- |
| 综合搜索 | `code`、`name` | 一个 Input，支持按库区编码、库区名称模糊查询 |
| 所属仓库筛选 | `warehouseCode` | Select，选项来源于仓库档案 |
| 类型筛选 | `type` | Select，选项来源于本清单 `ZoneType` |
| 状态筛选 | `status` | Select，包含全部、启用、停用 |
| 列表默认列 | `code`、`name`、`type`、`warehouseName`、`status`、`updatedAt` | PC 表格默认分页 20 条 |
| 候选展示 | `code`、`name` | 下拉候选建议展示为 `库区编码 库区名称` |
| 空值展示 | `remark`、`updatedAt` | 无值展示 `-` |

## 5. 引用规则

| 引用方 | 引用字段 | 规则 |
| :--- | :--- | :--- |
| 仓库档案 | `warehouseCode`、`warehouseName` | 所属仓库来源于仓库档案；新增 / 编辑库区只允许选择启用仓库 |
| 货位管理 | `zoneCode`、`zoneName` | 货位所属库区来源于库区档案；新增货位只允许选择启用库区 |
| 入库单据 | `zoneCode`、`zoneName` | 收货 / 上架作业引用库区，历史单据保留原库区信息 |
| 出库单据 | `zoneCode`、`zoneName` | 拣货 / 发货相关作业可按库区组织和追溯 |
| 库存查询 / 库存流水 | `zoneCode`、`zoneName` | 支持按库区筛选和展示库存、流水 |
| 盘点 / 调拨 | `zoneCode`、`zoneName` | 新建作业单只允许选择启用库区 |

---

## 6. 不确定性说明

| 事项 | 说明 |
| :--- | :--- |
| `warehouseName` 是否快照存储或实时联查 | context 未定义同步策略；Demo 可存储 `warehouseName` 用于展示，但研发实现需统一 |
| `code` 长度上限 | context 未定义具体长度，本字段清单不设置长度上限 |
| 编码连字符 | `context/11` 要求字母 + 数字，本字段清单不允许连字符 |
| 停用前业务校验 | context 未明确是否需校验已有货位、库存余额或未完结任务；本字段清单只约束停用后不进入新业务候选 |
