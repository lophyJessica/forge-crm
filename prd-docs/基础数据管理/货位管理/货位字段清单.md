# 货位字段清单

> 角色：字段清单(SSOT) | 类型：档案主数据
> 本文件为货位字段唯一事实来源；主 PRD 和前端 Demo 版 PRD 引用本文件，不重复维护字段口径。

## 1. 字段设计原则

- 货位必须隶属库区，所属库区来源于库区档案；所属仓库随库区带出，不在货位档案中手工录入。
- 货位由货架编号、层、列定位；正式 `location_code` 的前缀、补零、分隔符规则未在 context 明确，本文不自创格式。
- 编码字段按 `context/11-字段校验规范`：字母+数字、不可重复。
- 条码字段按 `context/11-字段校验规范`：数字为主，支持 EAN-13。
- 状态变更只能通过动作按钮触发，不允许直接编辑 `status`。
- 主数据不提供物理删除；停用货位保留历史引用。
- 日期格式 `YYYY-MM-DD`，时间格式 `YYYY-MM-DD HH:mm:ss`，空值展示 `-`，备注 ≤200 字符。

## 2. 货位基础字段

| 字段编码 | 字段名称 | 类型 | 来源 | 必填 | 可编辑 | 枚举/校验/说明 |
|:--|:--|:--|:--|:--:|:--:|:--|
| location_id | 货位ID | string | 系统生成 | 是 | 否 | 页面隐藏主键 |
| location_code | 货位编码 | string | 用户维护/系统生成 | 是 | 仅新增 | 字母+数字，不可重复；正式生成规则未在 context 明确 |
| location_name | 货位名称 | string | 系统生成/用户维护 | 是 | 条件 | 建议由货架编号、层、列生成展示名；如 `A-01-02` 仅为 Demo 展示口径 |
| rack_no | 货架编号 | string | 用户输入 | 是 | 是 | context 明确的货位组成部分；具体格式未明确 |
| level_no | 层 | integer | 用户输入 | 是 | 是 | 正整数，`>0` |
| column_no | 列 | integer | 用户输入 | 是 | 是 | 正整数，`>0` |
| remark | 备注 | string | 用户输入 | 否 | 是 | ≤200 字符 |

## 4. 条码与作业字段

| location_barcode | 货位条码 | string | 用户绑定/系统生成 | 是 | 通过绑定动作 | 数字为主，支持 EAN-13；不可重复；条码生成来源未明确 |

| is_available | 是否可用 | boolean | 系统计算 | 是 | 否 | `true` 可用于新作业、`false` 不可用于新作业；计算规则见第 6 节 |

| barcode_bound_at | 条码绑定时间 | datetime | 系统 | 否 | 否 | `YYYY-MM-DD HH:mm:ss` |

## 6. 枚举值与计算字段

### 6.1 状态 `status`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| active | 启用 | 可作为新 PUT/PICK 作业的候选货位，仍需通过条码和所属库区校验 |
| inactive | 停用 | 不允许被新 PUT/PICK 作业使用，历史单据和库存流水仍可查询 |

### 6.2 是否已绑定条码 `is_barcode_bound`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| true | 已绑定 | `location_barcode` 有值且通过格式与唯一性校验 |
| false | 未绑定 | `location_barcode` 为空或绑定未完成 |

### 6.3 是否可用 `is_available`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| true | 可用 | `status=active`，所属库区有效，且已绑定条码 |
| false | 不可用 | 货位停用、所属库区无效或未绑定条码 |

### 6.4 计算字段

| 字段 | 公式/规则 | 说明 |
|:--|:--|:--|
| 货位名称 | `rack_no + '-' + level_no + '-' + column_no` | 仅为 Demo 展示建议，不代表正式编码规则 |
| 是否已绑定条码 | `location_barcode` 是否为空 | 绑定动作成功后为 `true` |
| 是否可用 | `status=active && 所属库区有效 && is_barcode_bound=true` | 不替代库存状态和库存可用数量 |

## 7. 校验规则

| 校验项 | 规则 | 触发时机 |
|:--|:--|:--|
| 必填字段 | 货位编码、货架编号、层、列、所属库区、货位条码不能为空 | 保存、启用 |
| 货位编码 | 字母+数字，不可重复 | 失焦、保存 |
| 货架/层/列组合 | 同一所属库区下，货架编号 + 层 + 列不可重复 | 保存 |
| 层/列 | 正整数，`>0` | 失焦、保存 |
| 所属库区 | 必须来源于库区档案，且库区有效 | 选择、保存、启用 |
| 所属仓库 | 随所属库区带出，不允许手填 | 选择库区 |
| 货位条码 | 数字为主，支持 EAN-13，不可重复 | 失焦、绑定、保存 |
| 状态 | 只能由「启用」「停用」动作改变 | 点击动作按钮 |
| 备注 | ≤200 字符 | 失焦、保存 |
| PUT 扫货位 | 货位存在、启用、已绑定条码、属于当前作业仓库 | PDA 扫码 |
| PICK 扫货位 | 货位条码必须匹配当前推荐货位 | PDA 扫码 |

## 8. 不确定项

| 项目 | 说明 |
|:--|:--|
| 货位编码格式 | context 未明确前缀、补零、分隔符、是否包含仓库/库区编码 |
| 条码生成来源 | context 未明确条码由系统生成、外部导入还是人工录入 |
| 条码唯一范围 | 本文按 WMS 内不可重复处理；若后续改为仓库内唯一，需回溯更新 |
| 停用库存校验 | context 未明确有现存、占用、冻结库存的货位是否允许停用 |
