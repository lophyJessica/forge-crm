# 供应商档案字段清单

> 版本：V1.0 | 更新时间：2026-07-07
> 文档定位：供应商档案字段的 SSOT。主PRD、前端 Demo PRD、测试用例和下游单据只引用本清单字段口径，不重复定义枚举、必填性和校验规则。
> 适用范围：WMS 基础数据管理 / 供应商档案。

---

## 1. 字段总览

| 字段名称 | Field | 类型 | 来源 | 必填 | 枚举 / 取值 | 校验规则 | 页面使用 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 供应商编码 | `code` | String | ERP 同步 | 是 | - | 字母、数字、连字符；全局唯一；建议兼容 `VEND001`、`VEND-0001`；WMS 只读 | 列表、详情、同步新增预览、编辑只读、候选展示 | 供应商唯一标识；WMS 同步匹配键 |
| 供应商名称 | `name` | String | ERP 同步 | 是 | - | 非空；建议 1-50 字符；WMS 只读 | 列表、详情、同步新增预览、编辑只读、候选展示 | 供应商正式名称 |
| 供应商简称 | `shortName` | String | ERP 同步 | 否 | - | 建议 1-30 字符；WMS 只读；无值展示 `-` | 列表可选列、详情、综合搜索 | 供应商简称；如 ERP 未提供则为空 |
| 联系人 | `contactName` | String | ERP 同步 | 是 | - | 非空；建议 1-20 字符；WMS 只读 | 列表、详情、同步新增预览、编辑只读、综合搜索 | 默认业务联系人 |
| 联系电话 | `contactPhone` | String | ERP 同步 | 是 | - | 手机号或座机号；建议不超过 20 字符；WMS 只读 | 列表、详情、同步新增预览、编辑只读 | 到货异常、退货沟通的主要联系方式 |
| 联系地址 | `address` | String | ERP 同步 | 否 | - | 建议 1-100 字符；WMS 只读；无值展示 `-` | 详情、同步新增预览、编辑只读 | 常用联系地址，不强制等同开票地址 |
| 结算方式 | `settlementMethod` | Enum | ERP 同步 | 是 | 见“2. 枚举定义” | 必须命中枚举值；WMS 只读 | 列表、详情、筛选、编辑只读 | ERP 采购结算口径；WMS 展示不维护 |
| 账期天数 | `paymentTermDays` | Integer | ERP 同步 | 条件必填 | `0-365` | `settlementMethod=ACCOUNT` 时必填且为 0-365 整数；`CASH` 时为 `0`；WMS 只读 | 列表、详情、编辑只读 | ERP 账期长度 |
| 税率 | `taxRate` | String | ERP 同步 | 否 | 如 `13%`、`3%`、`未税` | 建议不超过 10 字符；WMS 只读；无值展示 `-` | 列表可选列、详情、编辑只读 | ERP 采购税率口径 |
| 状态 | `status` | Enum | WMS 维护 | 系统自动 | 见“2. 枚举定义” | 新同步默认 `ENABLED`；仅允许通过启用 / 停用按钮变更；不允许表单直接编辑 | 列表、详情、筛选、候选过滤 | 控制供应商是否可进入 WMS 新业务候选范围 |
| 停用原因 | `disabledReason` | Text | WMS 维护 | 条件必填 | - | 停用动作触发时填写；建议 1-100 字符；启用状态展示 `-` | 停用弹窗、详情 | 记录 WMS 本地停用原因 |
| 来源系统 | `sourceSystem` | String | 系统生成 | 系统自动 | 固定 `ERP` | 同步写入；不允许手工编辑 | 详情、Mock 数据 | 说明供应商资料来源 |
| 最后同步时间 | `lastSyncAt` | DateTime | 系统生成 | 系统自动 | `YYYY-MM-DD HH:mm:ss` | 每次 ERP 同步成功后写入；不允许手工编辑 | 列表、详情、Mock 数据 | 最近一次同步时间 |
| 创建时间 | `createdAt` | DateTime | 系统生成 | 系统自动 | `YYYY-MM-DD HH:mm:ss` | 首次同步入 WMS 时写入；不允许手工编辑 | 详情、Mock 数据 | WMS 首次建档时间 |
| 最后修改时间 | `updatedAt` | DateTime | 系统生成 | 系统自动 | `YYYY-MM-DD HH:mm:ss` | ERP 同步更新或 WMS 状态变更后写入；不允许手工编辑 | 详情、Mock 数据 | WMS 最近一次变更时间 |

## 2. 枚举定义

### 2.1 结算方式 `SettlementMethod`

| 枚举值 | 展示文案 | 说明 |
| :--- | :--- | :--- |
| `CASH` | 现付 | ERP 供应商结算方式为现付，账期天数应为 `0` |
| `ACCOUNT` | 账期结算 | ERP 供应商结算方式为账期结算，账期天数必填 |

### 2.2 状态 `BaseDataStatus`

| 枚举值 | 展示文案 | 是否可被 WMS 新业务引用 | 说明 |
| :--- | :--- | :---: | :--- |
| `ENABLED` | 启用 | 是 | 可被新建收货、退货等业务候选引用 |
| `DISABLED` | 停用 | 否 | 不可被新业务选择，但历史单据和查询仍可展示 |

## 3. 校验规则

| 校验对象 | 规则 | 触发时机 | 失败提示 |
| :--- | :--- | :--- | :--- |
| ERP 同步字段只读 | `code`、`name`、`shortName`、`contactName`、`contactPhone`、`address`、`settlementMethod`、`paymentTermDays`、`taxRate` 在 WMS 不提供编辑入口 | 页面渲染、保存 | 不展示可编辑控件 |
| 供应商编码唯一 | `code` 在 WMS 供应商档案内全局唯一；同步时同编码更新，不新增重复记录 | ERP 同步写入 | 供应商编码已存在，执行更新 |
| 供应商编码格式 | 支持字母、数字、连字符；不允许空值 | ERP 同步写入、Mock 同步预览 | 供应商编码格式不正确 |
| 供应商名称 | 非空，建议 1-50 字符 | ERP 同步写入、Mock 同步预览 | 供应商名称不能为空 |
| 联系人 | 非空，建议 1-20 字符 | ERP 同步写入、Mock 同步预览 | 联系人不能为空 |
| 联系电话 | 非空，手机号或座机号，建议不超过 20 字符 | ERP 同步写入、Mock 同步预览 | 联系电话格式不正确 |
| 结算方式 | 必须命中 `SettlementMethod` 枚举 | ERP 同步写入、筛选 | 结算方式不正确 |
| 账期天数 | `ACCOUNT` 时为 0-365 整数；`CASH` 时为 `0` | ERP 同步写入、Mock 同步预览 | 账期天数不正确 |
| 停用原因 | 点击停用并确认时填写，建议 1-100 字符 | 停用弹窗确认 | 停用原因不能为空 |
| 状态变更 | `status` 不允许表单直接修改，只允许动作按钮触发 | 页面交互 | 不展示状态下拉 |
| 日期时间 | 系统字段使用 `YYYY-MM-DD HH:mm:ss`；Mock 数据使用 2026 年 | 写入、展示 | 格式异常时展示 `-` |

校验时机遵循《11-字段校验规范》：格式 / 范围可失焦校验，提交或同步确认做全量严格校验；空值展示遵循通用规范使用 `-`。

## 4. 查询与展示规则

| 场景 | 字段 | 规则 |
| :--- | :--- | :--- |
| 综合搜索 | `code`、`name`、`shortName`、`contactName` | 一个 Input，支持供应商编码、名称、简称、联系人模糊查询 |
| 结算方式筛选 | `settlementMethod` | Select，选项来源于本清单 `SettlementMethod` |
| 状态筛选 | `status` | Select，包含全部、启用、停用 |
| 列表默认列 | `code`、`name`、`shortName`、`contactName`、`contactPhone`、`settlementMethod`、`paymentTermDays`、`status`、`lastSyncAt` | 地址、税率可进入详情查看；列表过长文本截断 |
| 候选展示 | `code`、`name` | 下拉候选建议展示为 `供应商编码 供应商名称` |
| 只读展示 | ERP 同步字段 | 使用纯文本或只读输入框，不展示保存入口 |
| 空值展示 | `shortName`、`address`、`taxRate`、`disabledReason` | 无值展示 `-` |

---

## 5. 引用规则

| 引用方 | 引用字段 | 规则 |
| :--- | :--- | :--- |
| ERP PO | `code`、`name` | PO 下发时携带供应商快照，WMS 不在 RCV 中手工选择无来源供应商 |
| 收货单 RCV | `supplierCode`、`supplierName` | 继承 PO 并快照存储；供应商档案后续变更不影响既有 RCV |
| 退货处理 | `code`、`name`、`contactName`、`contactPhone` | 用于退回供应商联系和追溯；退货单据字段在退货模块单独定义 |
| 历史查询 | `code`、`name`、`status` | 启用和停用供应商均可展示，停用状态需明确标识 |
| 新业务候选 | `code`、`name` | 只允许选择 `status=ENABLED` 的供应商 |

---

## 6. 同步与变更规则

| 场景 | 处理规则 |
| :--- | :--- |
| ERP 新增供应商 | WMS 按 `code` 新增供应商档案，写入 ERP 同步字段，默认 `status=ENABLED` |
| ERP 修改供应商资料 | WMS 按 `code` 更新 ERP 同步字段，刷新 `lastSyncAt`、`updatedAt` |
| WMS 停用供应商 | 仅变更 WMS `status` 与 `disabledReason`，不回写 ERP |
| WMS 启用供应商 | 仅变更 WMS `status`，启用后新业务可引用 |
| 历史单据 | 已固化供应商快照不因同步或启停用而变化 |

## 7. 不确定性说明

| 事项 | 说明 |
| :--- | :--- |
| ERP 状态联动 | context 未明确 ERP 停用供应商是否会强制同步为 WMS 停用；本版将 WMS `status` 作为本地新业务候选控制字段。 |
| 供应商简称 | 用户要求包含简称，但 ERP 参照物未展示该字段；本版定义 `shortName` 为 ERP 同步只读，若 ERP 无值则展示 `-`。 |
| 停用原因 | ERP 参照物要求停用原因必填；context 未单独说明 WMS 是否必填。本版为便于审计，将 WMS 停用原因定义为停用时条件必填。 |
| 当前前端数据模型 | 现有 Demo 若仅有 `code`、`name` 简化模型，实现时需按本清单扩展 Dexie `suppliers` 字段；字段口径以本清单为准。 |
