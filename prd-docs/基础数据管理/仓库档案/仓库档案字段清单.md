# 仓库档案字段清单

> 版本：V1.0 | 更新时间：2026-07-07
> 文档定位：仓库档案字段的 SSOT。主PRD、前端 Demo PRD、测试用例和下游单据只引用本清单字段口径，不重复定义枚举、必填性和校验规则。
> 适用范围：WMS 基础数据管理 / 仓库档案。

---

## 1. 字段总览

| 字段名称 | Field | 类型 | 来源 | 必填 | 枚举 / 取值 | 校验规则 | 页面使用 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 仓库编码 | `code` | String | 用户填写 | 是 | - | 字母 + 数字；全局唯一；保存时转大写；不可重复；编辑态只读 | 新增、编辑只读、列表、详情、候选展示 | 仓库在 WMS 内的唯一标识 |
| 仓库名称 | `name` | String | 用户填写 | 是 | - | 非空；去首尾空格；建议 1-50 字符 | 新增、编辑、列表、详情、候选展示 | 仓库业务名称 |
| 仓库类型 | `type` | Enum | 用户选择 | 是 | 见“2. 枚举定义” | 必须命中枚举值；不可为空 | 新增、编辑、列表、详情、筛选 | 区分仓库业务角色 |
| 负责人 | `manager` | String | 用户填写 | 是 | - | 非空；去首尾空格；建议 1-20 字符 | 新增、编辑、列表、详情、综合搜索 | 仓库责任人 |
| 仓库地址 | `address` | String | 用户填写 | 是 | - | 非空；去首尾空格；建议 1-100 字符 | 新增、编辑、列表、详情 | 仓库实际地址，列表过长时截断展示 |
| 备注 | `remark` | Text | 用户填写 | 否 | - | 遵通用备注规则，上限 200 字符 | 新增、编辑、详情 | 补充说明，不作为系统判断条件 |
| 状态 | `status` | Enum | 系统生成 / 动作按钮变更 | 系统自动 | 见“2. 枚举定义” | 新增默认 `ENABLED`；仅允许通过启用 / 停用按钮变更 | 列表、详情、筛选、候选过滤 | 控制仓库是否可进入新业务候选范围 |
| 创建时间 | `createdAt` | DateTime | 系统生成 | 系统自动 | `YYYY-MM-DD HH:mm:ss` | 新增保存时写入；不允许手工编辑 | 详情、Mock 数据 | 首次建档时间 |
| 最后修改时间 | `updatedAt` | DateTime | 系统生成 | 系统自动 | `YYYY-MM-DD HH:mm:ss` | 编辑或状态变更后写入；不允许手工编辑 | 详情、Mock 数据 | 最近一次修改时间 |

## 2. 枚举定义

### 2.1 仓库类型 `WarehouseType`

| 枚举值 | 展示文案 | 说明 |
| :--- | :--- | :--- |
| `MAIN` | 主仓 | 全国或区域核心仓，承担主要收发和库存承载 |
| `BRANCH` | 分仓 | 区域履约、备货或中转仓 |
| `STORE` | 门店 | 门店后仓、自提或临时备货仓 |

### 2.2 状态 `BaseDataStatus`

| 枚举值 | 展示文案 | 是否可被新业务引用 | 说明 |
| :--- | :--- | :---: | :--- |
| `ENABLED` | 启用 | 是 | 可被新建业务单据、库区、货位选择 |
| `DISABLED` | 停用 | 否 | 不可被新业务选择，但历史数据仍可展示 |

## 3. 校验规则

| 校验对象 | 规则 | 触发时机 | 失败提示 |
| :--- | :--- | :--- | :--- |
| 必填字段 | 仓库编码、仓库名称、仓库类型、负责人、仓库地址不能为空 | 提交保存 | 按字段提示“不能为空” |
| 仓库编码格式 | 仅允许字母 + 数字，保存时统一转大写 | 失焦、提交保存 | 仓库编码仅支持字母和数字 |
| 仓库编码唯一 | `code` 在仓库档案内全局唯一 | 提交保存 | 仓库编码已存在 |
| 仓库编码编辑 | 编辑已有仓库时 `code` 只读，不进入保存变更 | 页面渲染 | 不提供可编辑入口 |
| 备注长度 | `remark` 不超过 200 字符 | 输入、提交保存 | 备注不能超过 200 字符 |
| 状态变更 | `status` 不允许表单直接修改，只允许动作按钮触发 | 页面交互 | 不展示状态下拉 |
| 日期时间 | 系统字段使用 `YYYY-MM-DD HH:mm:ss` | 写入、展示 | 格式异常时展示 `-` |

校验时机遵循《11-字段校验规范》：格式 / 范围可失焦校验，提交保存做全量严格校验；空值展示遵循通用规范使用 `-`。

## 4. 查询与展示规则

| 场景 | 字段 | 规则 |
| :--- | :--- | :--- |
| 综合搜索 | `code`、`name`、`manager` | 一个 Input，支持按仓库编码、仓库名称、负责人模糊查询 |
| 类型筛选 | `type` | Select，选项来源于本清单 `WarehouseType` |
| 状态筛选 | `status` | Select，包含全部、启用、停用 |
| 列表默认列 | `code`、`name`、`type`、`manager`、`address`、`status` | 地址过长时截断，悬浮可查看完整内容 |
| 候选展示 | `code`、`name` | 下拉候选建议展示为 `仓库编码 仓库名称` |
| 空值展示 | `remark`、`updatedAt` | 无值展示 `-` |

---

## 5. 引用规则

| 引用方 | 引用字段 | 规则 |
| :--- | :--- | :--- |
| 库区管理 | `warehouseCode`、`warehouseName` | 所属仓库来源于仓库档案；新增 / 编辑库区只允许选择启用仓库 |
| 货位管理 | `warehouseCode`、`warehouseName` | 货位所属仓库来源于仓库档案，并需与所属库区归属一致 |
| 入库单据 | `warehouseCode`、`warehouseName` | 新建入库作业只允许选择启用仓库；历史单据保留原仓库信息 |
| 出库单据 | `warehouseCode`、`warehouseName` | 波次、拣货、复核、包裹、交运链路引用仓库编码与名称 |
| 库存查询 / 库存流水 | `warehouseCode`、`warehouseName` | 允许查询启用和停用仓库，停用状态需明确展示 |
| 盘点 / 调拨 | `warehouseCode`、`warehouseName` | 新建作业单只允许选择启用仓库 |

## 6. 不确定性说明

| 事项 | 说明 |
| :--- | :--- |
| 仓库类型枚举来源 | context 未单独定义仓库类型枚举；本文采用当前前端 Demo 数据模型中的 `MAIN`、`BRANCH`、`STORE`。如真实项目口径不同，应优先修改本字段清单。 |
| 操作人字段 | 当前前端 Demo 数据模型包含 `createdAt`、`updatedAt`，未包含创建人 / 最后修改人；本文暂不将操作人定义为一期 SSOT 字段。 |
| 停用原因 | context 与当前 Demo 未要求停用原因；本文暂不定义 `disabledReason`。若后续需要审计，应补字段并调整停用弹窗。 |
