# 收货单_Demo_执行页

> 角色：Demo·执行页 | 类型：执行作业单
> 本页覆盖 PC 收货作业页与 PDA 快速收货作业页；两端共用同一套 RCV 字段、状态和超收校验规则。

## 1. 页面定位

执行页用于仓管基于 PO 下推的 RCV 草稿完成收货录入、保存草稿、确认收货和打印标签。PC 适合完整核对与批量录入；PDA 适合现场扫码、少量改数和快速确认。页面强调快速核对 PO、录入实收、发现超收并阻断。

## 2. 页面结构

### 2.1 PC 收货作业页

```text
PC 收货执行页
├─ 顶部信息条
│  ├─ 收货单号、状态、来源PO、供应商
│  └─ 返回列表、查看详情
├─ 基础信息区
│  ├─ 仓库、库区、收货日期
│  └─ 备注
├─ 商品明细区
│  ├─ PO商品快照
│  ├─ 采购数量、已收数量、PO未收数量
│  └─ 本次实收数量、行备注
├─ 校验提示区
│  └─ 超收/必填/格式错误字段级提示
└─ 底部动作栏
   ├─ 保存草稿
   ├─ 确认收货
   └─ 打印标签（已确认后展示）
```

### 2.2 PDA 快速收货作业页

```text
PDA 收货作业页
├─ 扫码区
│  ├─ 扫 PO/RCV 条码
│  └─ 扫商品条码
├─ 当前商品卡片
│  ├─ 商品名称、规格、单位
│  └─ PO未收数量、本次已录数量
├─ 数量录入区
│  ├─ 本次实收数量
│  └─ 加减步进、手动输入
├─ 反馈区
│  └─ 超收提示、语音+震动反馈
└─ 底部大按钮
   ├─ 保存
   └─ 确认收货
```

## 3. 核心交互

| 交互 | 说明 |
|:--|:--|
| 仓库选择 | 默认带出常用仓库，可切换启用仓库 |
| 库区选择 | 默认收货区，列表受所选仓库过滤 |
| 实收录入 | 失焦校验正整数和 PO 未收上限 |
| 保存草稿 | 保存当前内容，不触发后续质检 |
| 确认收货 | 全量严格校验，通过后状态变为已确认 |
| 打印标签 | 已确认后展示，打印成功后更新打印次数 |

## 4. 字段状态

| 页面状态 | 可编辑 | 只读 |
|:--|:--|:--|
| 草稿 | 仓库、库区、收货日期、本次实收数量、备注 | 收货单号、PO、供应商、商品快照、采购数量、已收数量、PO 未收数量、状态 |
| 已确认 | 无 | 全部业务字段只读；可打印标签 |

## 5. 校验提示

| 场景 | 提示位置 | 文案 |
|:--|:--|:--|
| 实收为空 | 明细行本次实收数量 | `本次实收数量不能为空` |
| 实收非正整数 | 明细行本次实收数量 | `本次实收数量必须为大于 0 的整数` |
| 超收 | 明细行本次实收数量 | `本次实收数量不能大于 PO 未收数量` |
| 备注超长 | 备注输入框 | `备注不能超过 200 字符` |
| 重复确认 | 页面顶部提示 | `已确认收货单不可重复确认` |

## 6. 操作后反馈

| 动作 | 成功反馈 | 后续 |
|:--|:--|:--|
| 保存草稿 | `草稿已保存` | 停留当前页 |
| 确认收货 | `收货已确认` | 页面切换只读，展示打印标签按钮 |
| 打印标签 | `标签已发送打印` | 更新打印次数和最后打印时间 |

## 8. PC/PDA 差异

| 项 | PC | PDA |
|:--|:--|:--|
| 使用场景 | 办公区核对 PO、批量录入、打印标签 | 收货现场扫码、快速录入、即时反馈 |
| 输入方式 | 表格录入为主 | 扫码优先，数量用大按钮/步进器 |
| 信息密度 | 展示完整单据头和多行明细 | 聚焦当前 RCV 和当前 SKU |
| 校验 | 失焦校验 + 确认全量校验 | 扫码/录数即时校验 + 确认全量校验 |
| 反馈 | 字段标红、顶部提示 | 字段高亮、语音+震动反馈 |
| 离线 | 不要求 | 可自动切换离线，网络恢复后自动同步；同步前仍不得绕过超收规则 |

PDA 与 PC 不能同时确认同一张草稿 RCV；确认时以后端最新状态为准，若另一端已确认，则当前端提示“已确认收货单不可重复确认”。
