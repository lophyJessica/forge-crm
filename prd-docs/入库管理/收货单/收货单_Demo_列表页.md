# 收货单_Demo_列表页

> 角色：Demo·列表页 | 类型：执行作业单
> 页面只描述结构、交互和数据来源；字段定义以 `收货单字段清单.md` 为准，不重复枚举。

## 1. 页面定位

收货单列表页用于仓管在 PC 端查看 PO 下推生成的 RCV，按状态和入库进度筛选待处理单据，并进入收货执行页、详情页或打印标签。

## 2. 页面入口

| 入口 | 说明 |
|:--|:--|
| 菜单 | 入库管理 / 收货单 |
| 关联跳转 | 采购订单下发记录、入库管理概览、质检/上架关联详情 |

## 3. 页面结构

```text
收货单列表
├─ 顶部筛选区
│  ├─ 关键词：收货单号/PO单号/供应商/SKU
│  ├─ 状态：全部/草稿/已确认
│  ├─ 入库进度：全部/待质检/待上架/已入库/异常
│  ├─ 仓库、收货日期范围
│  └─ 查询、重置
├─ 操作区
│  └─ 批量打印标签（仅已确认且有标签的单据）
├─ 表格区
│  ├─ 单号、PO、供应商、仓库/库区、状态、入库进度、实收合计、创建时间
│  └─ 行操作：执行收货、查看详情、打印标签
└─ 分页
   └─ 默认 20 条/页
```

## 4. 列表行为

| 场景 | 行操作 |
|:--|:--|
| 状态=草稿 | 展示“执行收货”“查看详情” |
| 状态=已确认 | 展示“查看详情”“打印标签” |
| 入库进度=异常 | 展示异常标识，可进入详情查看质检原因 |
| 按钮不可用 | 直接隐藏，不展示 disabled 态 |

## 5. 查询规则

- 关键词支持收货单号、来源采购单号、供应商名称、商品编码模糊查询。
- 日期默认不限；选择日期范围时使用收货日期。
- 状态筛选只使用收货单自身状态：草稿、已确认。
- 入库进度用于查看后续链路，不改变收货单状态。

## 6. 空状态与异常态

| 状态 | 展示 |
|:--|:--|
| 无数据 | 展示空列表和“暂无收货单” |
| 查询无结果 | 展示“未找到符合条件的收货单” |
| 接口失败 | 保留筛选条件，提示“收货单列表加载失败，请重试” |
