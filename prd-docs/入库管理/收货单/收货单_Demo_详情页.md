# 收货单_Demo_详情页

> 角色：Demo·详情页 | 类型：执行作业单
> 详情页用于查看收货单快照、收货结果、标签打印记录和后续质检/上架关联进度。

## 1. 页面定位

收货单详情页面向研发、测试、产品复核，用于确认一张 RCV 的完整只读信息：来源 PO、供应商、仓库/库区、实收明细、状态、标签、后续质检和上架关联结果。

## 2. 页面结构

```text
收货单详情
├─ 顶部摘要
│  ├─ 收货单号、状态、入库进度
│  └─ 返回列表、打印标签
├─ 来源信息
│  ├─ 来源采购单号、供应商
│  └─ 创建人/创建时间、确认人/确认时间
├─ 收货信息
│  ├─ 仓库、库区、收货日期、备注
│  └─ 标签条码、打印次数、最后打印时间
├─ 商品明细
│  └─ 商品快照、采购数量、已收数量、PO未收数量、本次实收数量
├─ 关联进度
│  ├─ 质检单号、质检结果、不合格原因摘要
│  └─ 上架单号、上架进度、库存流水入口
└─ 操作记录
   └─ 创建、保存、确认、打印标签
```

## 3. 展示规则

| 区域 | 展示规则 |
|:--|:--|
| 顶部摘要 | 状态展示收货单自身状态；入库进度展示后续链路 |
| 来源信息 | PO、供应商只读，展示 PO 下发快照 |
| 收货信息 | 已确认后展示确认人和确认时间；空值展示 `-` |
| 商品明细 | 数量字段右对齐；异常数量标红展示 |
| 关联进度 | 未生成质检/上架单时展示 `-` |
| 操作记录 | 按时间倒序展示 |

## 4. 动作规则

| 动作 | 展示条件 | 说明 |
|:--|:--|:--|
| 打印标签 | 状态=已确认 | 打印收货标签并记录次数 |
| 进入执行页 | 状态=草稿 | 从详情进入收货执行页继续维护 |
| 查看质检单 | 已生成质检记录 | 跳转质检详情 |
| 查看上架单 | 已生成 PUT | 跳转上架详情 |

## 5. 关联进度口径

| 入库进度 | 判断逻辑 | 页面说明 |
|:--|:--|:--|
| 待质检 | RCV 已确认但未登记质检 | 等待质检处理 |
| 待上架 | 质检有合格品但未完成上架 | 可查看关联 PUT |
| 已入库 | 上架完成并生成库存流水 | 库存已转可用 |
| 异常 | 质检全部不合格或后续阻断 | 不允许上架，等待退货处理 |

## 7. 测试关注点

- 草稿详情可进入执行页，已确认详情不可修改收货数量。
- 已确认详情展示打印标签动作，草稿详情不展示正式打印标签动作。
- 质检全部不合格时，详情页展示异常进度且不展示上架执行入口。
- 空值统一展示 `-`，时间格式为 `YYYY-MM-DD HH:mm:ss`。
