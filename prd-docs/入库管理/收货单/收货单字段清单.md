# 收货单字段清单

> 角色：字段清单(SSOT) | 类型：执行作业单
> 本文件为收货单字段唯一事实来源；主 PRD、业务规则、Demo 页面引用本文件，不重复维护字段口径。

## 1. 字段设计原则

- 单号、状态、来源 PO、供应商、商品快照字段不可手工编辑。
- 数量字段按 `11-字段校验规范`：正整数、`>0`、`≤上限值`。
- 备注字段上限 200 字符。
- 日期格式 `YYYY-MM-DD`，时间格式 `YYYY-MM-DD HH:mm:ss`。
- 空值展示 `-`。
- 商品信息在单据中快照存储，后续主数据变化不影响历史收货单。

## 2. 单据头字段

| 字段编码 | 字段名称 | 类型 | 来源 | 必填 | 可编辑 | 校验/说明 |
|:--|:--|:--|:--|:--:|:--:|:--|
| rcv_no | 收货单号 | string | 系统生成 | 是 | 否 | `RCV{YYYYMMDD}-{4位序号}`，如 `RCV20260705-0001` |
| source_po_no | 来源采购单号 | string | ERP 下发 PO | 是 | 否 | PO 单号只读，用于追溯上游采购订单 |
| supplier_code | 供应商编码 | string | 继承 PO | 是 | 否 | 供应商快照字段 |
| supplier_name | 供应商名称 | string | 继承 PO | 是 | 否 | 供应商快照字段 |
| warehouse_code | 仓库编码 | string | 仓管选择/默认仓库 | 是 | 是 | 必须为启用状态仓库 |
| warehouse_name | 仓库名称 | string | 仓库档案 | 是 | 否 | 随仓库编码带出 |
| zone_code | 库区编码 | string | 仓管选择 | 是 | 是 | 默认收货区；必须属于所选仓库 |
| zone_name | 库区名称 | string | 库区档案 | 是 | 否 | 随库区编码带出 |
| status | 单据状态 | enum | 系统 | 是 | 否 | `草稿`、`已确认`；状态变更只能通过动作按钮触发 |
| receipt_date | 收货日期 | date | 仓管选择/系统默认 | 是 | 是 | 默认当天；不超过当前日期，补录场景除外 |
| label_barcode | 收货标签条码 | string | 系统生成 | 否 | 否 | 已确认后生成；数字为主，支持 EAN-13 兼容展示 |
| print_count | 打印次数 | integer | 系统累计 | 否 | 否 | 默认 0，每次打印成功后 +1 |
| last_print_time | 最后打印时间 | datetime | 系统 | 否 | 否 | 未打印展示 `-` |
| remark | 备注 | string | 仓管录入 | 否 | 是 | ≤200 字符 |
| created_by | 创建人 | string | 系统 | 是 | 否 | 当前操作人或 PO 下推任务 |
| created_at | 创建时间 | datetime | 系统 | 是 | 否 | `YYYY-MM-DD HH:mm:ss` |
| confirmed_by | 确认人 | string | 系统 | 否 | 否 | 点击确认收货的操作人 |
| confirmed_at | 确认时间 | datetime | 系统 | 否 | 否 | 已确认后写入 |

## 3. 明细行字段

| 字段编码 | 字段名称 | 类型 | 来源 | 必填 | 可编辑 | 校验/说明 |
|:--|:--|:--|:--|:--:|:--:|:--|
| line_no | 行号 | integer | 系统 | 是 | 否 | 从 1 递增 |
| po_line_no | PO 行号 | integer | 继承 PO | 是 | 否 | 用于回写 PO 收货进度 |
| sku_code | 商品编码 | string | 继承 PO | 是 | 否 | 商品快照，字母+数字 |
| sku_name | 商品名称 | string | 继承 PO | 是 | 否 | 商品快照 |
| spec | 规格 | string | 继承 PO | 否 | 否 | 无值展示 `-` |
| unit | 单位 | string | 继承 PO | 是 | 否 | 如 件、箱 |
| purchase_qty | 采购数量 | integer | PO 行 | 是 | 否 | 正整数，只读 |
| received_qty_before | 已收数量 | integer | 系统累计 | 是 | 否 | 同一 PO 行历史已确认收货数量 |
| po_unreceived_qty | PO 未收数量 | integer | 系统计算 | 是 | 否 | `采购数量 - 已收数量` |
| actual_receive_qty | 本次实收数量 | integer | 仓管录入 | 是 | 是 | 正整数 `>0`，且 `≤ PO未收数量`；确认收货时强校验 |
| line_remark | 行备注 | string | 仓管录入 | 否 | 是 | ≤200 字符 |

## 4. 关联展示字段

关联展示字段不参与收货单状态变更，用于列表和详情页追踪后续进度。

| 字段编码 | 字段名称 | 类型 | 来源 | 说明 |
|:--|:--|:--|:--|:--|
| qc_no | 关联质检单号 | string | 质检环节 | 未生成展示 `-` |
| qc_result | 质检结果 | enum | 质检环节 | `未质检`、`部分合格`、`全部合格`、`全部不合格` |
| put_no | 关联上架单号 | string | 上架环节 | 质检有合格品后生成；未生成展示 `-` |
| inbound_progress | 入库进度 | enum | 系统汇总 | `待质检`、`待上架`、`已入库`、`异常`；不是收货单状态 |

## 5. 枚举值

### 5.1 单据状态 `status`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| draft | 草稿 | 可编辑实收数量，可保存，可确认收货 |
| confirmed | 已确认 | 收货已确认，不可编辑数量，可打印标签 |

### 5.2 入库进度 `inbound_progress`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| pending_qc | 待质检 | 收货已确认，等待质检 |
| pending_putaway | 待上架 | 质检有合格品，等待上架 |
| completed | 已入库 | 上架完成，库存转可用 |
| exception | 异常 | 质检全部不合格或后续流程阻断 |

## 6. 计算字段

| 字段 | 公式 | 说明 |
|:--|:--|:--|
| PO 未收数量 | `purchase_qty - received_qty_before` | 按 PO 行实时计算 |
| 本单实收合计 | `Σ actual_receive_qty` | 仅统计本 RCV 明细行 |
| 打印次数 | `print_count + 1` | 每次打印成功后累加 |

## 7. 校验时机

| 时机 | 校验范围 |
|:--|:--|
| 实时失焦 | 数量格式、数量范围、备注长度 |
| 保存草稿 | 仓库/库区存在性、严重格式错误；允许暂未填满全部实收数量 |
| 确认收货 | 必填、正整数、`本次实收数量 ≤ PO未收数量`、状态仍为草稿 |
