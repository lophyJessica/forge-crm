## 3. 用例 1：正常收货并打印标签

### 输入

| 字段 | 值 |
|:--|:--|
| 收货单号 | RCV20260705-0001 |
| 来源采购单号 | PO20260705-0008 |
| 仓库/库区 | 上海一仓 / 收货区 |
| SKU10086 本次实收 | 60 件 |
| SKU20011 本次实收 | 50 卷 |
| 收货日期 | 2026-07-05 |

### 推演

| 步骤 | 动作 | 系统判断 | 结果 |
|:--:|:--|:--|:--|
| 1 | PO 下推生成 RCV | PO 有效 | RCV 状态为草稿 |
| 2 | 仓管录入实收 | 60≤60，50≤50 | 实时校验通过 |
| 3 | 点击确认收货 | 必填、数量、超收校验通过 | RCV 状态变为已确认 |
| 4 | 打印标签 | 已确认且条码存在 | 打印次数 0→1 |

### 期望结果

- RCV20260705-0001 状态为已确认。
- 标签条码生成，示例：`RCV20260705-0001-01`、`RCV20260705-0001-02`。
- PO20260705-0008 两行未收数量变为 0。
- 后续可进入质检；质检未完成前不可直接上架。

## 4. 用例 2：部分收货

### 输入

| 字段 | 值 |
|:--|:--|
| 收货单号 | RCV20260706-0001 |
| 来源采购单号 | PO20260706-0003 |
| SKU30021 采购数量 | 30 箱 |
| SKU30021 历史已收 | 0 箱 |
| SKU30021 本次实收 | 18 箱 |

### 推演

| 步骤 | 动作 | 系统判断 | 结果 |
|:--:|:--|:--|:--|
| 1 | 录入本次实收 18 | 18≤30 | 通过 |
| 2 | 点击确认收货 | 数量为正整数 | RCV 已确认 |
| 3 | 更新 PO 收货进度 | 0+18 | 历史已收变为 18，未收保留 12 |

### 期望结果

- 部分收货允许确认。
- PO 未收余量 12 箱保留，后续到货可继续下推新的收货记录或在同一业务链路中继续收货，具体实现以系统设计为准。

## 5. 用例 3：超收阻断

### 输入

| 字段 | 值 |
|:--|:--|
| 收货单号 | RCV20260705-0002 |
| 来源采购单号 | PO20260705-0008 |
| SKU10086 PO 未收 | 60 件 |
| SKU10086 本次实收 | 65 件 |

### 推演

| 步骤 | 动作 | 系统判断 | 结果 |
|:--:|:--|:--|:--|
| 1 | 录入本次实收 65 | 65>60 | 字段标红 |
| 2 | 点击确认收货 | 超收校验失败 | 阻断确认 |
| 3 | 仓管改为 60 | 60≤60 | 可重新确认 |

### 期望结果

- 系统提示：`本次实收数量不能大于 PO 未收数量`。
- RCV 保持草稿。
- 不生成正式收货标签，不进入质检。

## 6. 用例 4：质检未过不可上架

### 输入

| 字段 | 值 |
|:--|:--|
| 收货单号 | RCV20260705-0003 |
| SKU | SKU20011 仓储标签纸 |
| 本次实收 | 20 卷 |
| 质检结果 | 全部不合格 |
| 不合格原因 | 外箱受潮 |

### 推演

| 步骤 | 动作 | 系统判断 | 结果 |
|:--:|:--|:--|:--|
| 1 | 确认收货 | 收货校验通过 | RCV 已确认 |
| 2 | 质检登记全部不合格 | 合格数量=0 | 不生成上架单 |
| 3 | 尝试流转上架 | 质检未过 | 阻断 |

### 期望结果

- 收货单详情显示关联进度为异常。
- 不允许生成或执行 PUT。
- 等待退货处理；一期只登记不合格数量和原因，完整退货为二期。
