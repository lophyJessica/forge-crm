# 上架单_Demo_列表页

> 角色：Demo·列表页 | 类型：执行作业单
> 页面只描述结构、控件、交互和 Mock；字段定义以 `上架单字段清单.md` 为准，不重复维护字段枚举。

## 1. 页面定位

上架单列表页用于 PC 端查看质检合格后生成的 PUT，跟踪待上架、上架中、已完成任务和库存过账结果。列表页不提供新增入口，避免绕过质检合格前置。

## 2. 页面入口

| 入口 | 说明 |
|:--|:--|
| 菜单 | 入库管理 / 上架单 |
| 关联跳转 | 收货单详情、库存流水 |

## 3. 页面结构

```text
上架单列表
├─ 顶部筛选区
│  ├─ 关键词：上架单号/收货单号/SKU/上架人
│  ├─ 状态、仓库、是否已生成FL
│  ├─ 创建时间范围
│  └─ 查询、重置
├─ 表格区
│  ├─ 上架单号、来源收货单、仓库、状态、上架人
│  ├─ 合格总数、已上架总数、待上架总数、进度、创建时间
│  └─ 行操作：查看详情
└─ 分页
   └─ 默认 20 条/页
```

## 4. 行操作规则

| 场景 | 行操作 |
|:--|:--|
| 状态=待上架 | 查看详情 |
| 状态=上架中 | 查看详情 |
| 状态=已完成 | 查看详情、查看库存流水 |
| 按钮不可用 | 直接隐藏，不展示 disabled 态 |

> PDA 上架作业从 PDA 任务入口进入；PC 列表仅做查询和追溯。

## 5. 查询规则

- 关键词支持 PUT 单号、来源 RCV、商品编码、上架人模糊查询。
- 状态筛选只使用 PUT 自身状态。
- 是否已生成 FL 用于核对库存过账结果。

| 状态 | 展示 |
|:--|:--|
| 无数据 | `暂无上架单` |
| 查询无结果 | `未找到符合条件的上架单` |
| 接口失败 | 保留筛选条件，提示 `上架单列表加载失败，请重试` |
