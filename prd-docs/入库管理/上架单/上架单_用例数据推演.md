## 3. 用例 1：一次上架完成

### 输入

| 字段 | 值 |
|:--|:--|
| 上架单号 | PUT20260705-0001 |
| SKU | SKU10086 |
| 合格数量 | 100 |
| 实扫货位 | A-01-02 |
| 本次上架 | 100 |

### 状态快照

| 时点 | PUT 状态 | 冻结数量 | 现存可用增量 | FL |
|:--|:--|--:|--:|:--|
| 上架前 | 待上架 | 100 | 0 | 未生成 |
| 确认后 | 已完成 | 0 | +100 | 生成 |

### 期望结果

- PUT 状态为已完成。
- 库存从冻结转现存可用 100 件。
- 生成库存流水 `FL20260705-00000001`。

## 4. 用例 2：分批多货位上架

### 输入与公式

| 步骤 | 历史已上架 | 本次货位 | 本次上架 | 公式 | 状态 |
|:--:|--:|:--|--:|:--|:--|
| 1 | 0 | A-01-02 | 60 | `0+60<100` | 上架中 |
| 2 | 60 | A-01-03 | 40 | `60+40=100` | 已完成 |

### 期望结果

- 第一次确认后 PUT 为上架中，A-01-02 记录 60 件。
- 第二次确认后 PUT 为已完成，A-01-03 记录 40 件。
- 累计 100 件转现存可用，按实际确认记录生成 FL。

## 8. 走查清单

| 检查项 | 预期 |
|:--|:--|
| PUT 前缀 | 已在 `09-单据命名规则` 定义 |
| 来源限制 | 只能由质检合格结果生成 |
| 状态 | 待上架、上架中、已完成，无审核流 |
| 货位推荐 | 空闲货位优先，可为空 |
| 实际货位 | PDA 扫描，必须有效 |
| 数量上限 | 累计上架≤合格数量 |
| 过账时点 | PDA 确认上架后冻结转现存可用，生成 FL |
| 质检未过 | 不生成 PUT，不允许上架 |
