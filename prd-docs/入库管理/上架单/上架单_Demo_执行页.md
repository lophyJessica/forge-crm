# 上架单_Demo_执行页

> 角色：Demo·执行页 | 类型：执行作业单
> 本页为 PDA 上架作业页，重点描述扫码货位、数量确认和库存过账反馈；字段定义以 `上架单字段清单.md` 为准。

## 1. 页面定位

PDA 上架执行页用于仓管领取质检合格后生成的 PUT，依据推荐货位或现场人工指定货位完成上架。页面需要适合手套操作，大字体、大按钮，扫码优先。

## 2. 页面结构

```text
PDA 上架作业页
├─ 任务头
│  ├─ 上架单号、来源RCV、进度
│  └─ 仓库、状态、上架人
├─ 当前商品卡片
│  ├─ 商品名称、规格、单位
│  ├─ 合格数量、已上架数量、待上架数量
│  └─ 推荐货位
├─ 扫码区
│  └─ 扫实际货位条码
├─ 数量区
│  ├─ 本次上架数量
│  ├─ 加减步进
│  └─ 全部上架快捷按钮
├─ 反馈区
│  └─ 货位/数量/过账结果提示
└─ 底部大按钮
   ├─ 确认本次上架
   └─ 完成上架（满足条件后展示）
```

## 3. 核心交互

| 交互 | 说明 |
|:--|:--|
| 领取任务 | 查看待上架任务，进入 PDA 作业页 |
| 查看推荐货位 | 推荐货位突出展示，可为空 |
| 扫实际货位 | 扫描后校验货位存在、启用、属于当前仓库 |
| 本次数量 | 默认可填待上架数量，支持步进调整 |
| 全部上架 | 将本次数量填为当前剩余待上架数量 |
| 确认本次上架 | 校验通过后记录货位和数量 |
| 完成上架反馈 | 完成后展示库存转可用和 FL 生成结果 |

## 4. 扫码与数量反馈

| 场景 | 反馈 |
|:--|:--|
| 扫货位正确 | 高亮货位，短提示音 |
| 货位不存在 | 红色提示、震动、语音提示“货位无效” |
| 货位停用 | 阻断，提示“货位已停用，不可上架” |
| 数量为 0 | 阻断，提示“本次上架数量必须大于 0” |
| 累计超量 | 阻断，提示“累计上架数量不能大于合格数量” |
| 上架完成 | 提示“上架完成，库存已转可用” |

## 5. 页面状态

| PUT 状态 | 页面表现 |
|:--|:--|
| 待上架 | 展示当前商品、推荐货位和确认按钮 |
| 上架中 | 展示剩余待上架数量，允许继续上架 |
| 已完成 | 只读展示完成摘要和库存流水入口 |

## 7. 离线与同步

- PDA 支持离线缓存扫描和数量确认记录，网络恢复后自动同步。
- 离线期间仍需执行本地基础校验：数量不能超过待上架数量、货位条码不能为空。
- 同步时以后端最新状态为准；若 PUT 已完成或待上架数量变化，提示重新同步。
- 离线记录不得绕过累计数量上限。

## 8. 不展示内容

- 不展示新增上架单入口。
- 不展示审核按钮。
- 不展示质检登记、退货处理控件。
