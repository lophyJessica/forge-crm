# 拜访计划_Demo_详情页

> **版本**：V2.0 | 2026-07-18
> **页面类型**：PC Web 详情页
> **前置文档**：《拜访计划主PRD》《拜访计划字段清单》
> **字段约束**：字段定义、状态语义和 Tag 配色引用《拜访计划字段清单》，本文只定义展示与交互。

---

## 1. 页面概述

| 项目 | 内容 |
|------|------|
| 页面名称 | 拜访详情 |
| 页面路径 | `/visits/{id}` |
| 页面目标 | 查看计划、签到、结果、关联对象与自动跟进记录 |
| 进入方式 | 拜访列表、线索/商机/客户关联拜访 |
| 数据权限 | 引用《拜访计划主PRD》 |
| 页面骨架 | 页头状态区 → 状态时间线 → 计划信息 → 关联对象 → 签到与结果 → 跟进记录 → 审计日志 → 底部操作 |
| 页面宽度 | 最大 1120px，居中布局 |

详情页原则：

- 全部字段只读展示。
- 计划地址与签到地址分开展示。
- 状态仅由动作触发。
- 未签到不展示完成入口。
- 完成状态必须有唯一自动跟进记录。
- 取消状态不生成跟进记录。
- 不提供删除。
- 各子区块独立加载和失败。

## 2. 初始化与权限

### 2.1 加载顺序

1. 校验拜访查看权限。
2. 拉取拜访主数据与版本。
3. 拉取状态时间线。
4. 拉取关联对象摘要。
5. 完成状态下拉取自动跟进记录。
6. 拉取审计日志。
7. 根据状态与权限计算底部动作。

### 2.2 页面状态

| 场景 | 页面表现 | 操作 |
|------|----------|------|
| 首次加载 | 固定骨架屏 | 等待 |
| 拜访不存在 | 404 状态页 | 返回列表 |
| 无权限 | 403 状态页 | 返回列表 |
| 主数据失败 | 整页错误态 | 重试、返回列表 |
| 关联对象失败 | 区块错误态 | 局部重试 |
| 跟进记录失败 | 结果区块错误态 | 局部重试 |
| 并发状态变化 | 刷新主数据和底栏 | 查看最新状态 |

## 4. 状态时间线

### 4.1 节点

固定展示：

1. 创建计划。
2. 完成签到。
3. 完成拜访。
4. 取消拜访。

> 完成与取消为互斥分支；状态编码引用字段清单。

### 4.2 视觉状态

| 节点状态 | 视觉 | 时间展示 | 说明 |
|----------|------|----------|------|
| 已完成 | 实心圆 + 主色连线 | 完整时间 | 真实发生动作 |
| 当前待办 | 加粗描边 | 进入状态时间 | 当前可执行节点 |
| 未发生 | 灰色空心圆 | `--` | 尚未发生 |
| 完成终态 | 绿色 CheckCircle | 完成时间 | 跟进同步成功 |
| 取消终态 | 红色 Ban | 取消时间 | 只读终态 |

### 4.3 分支规则

- 已计划：只高亮创建节点。
- 已签到：高亮创建和签到节点。
- 已完成：高亮创建、签到和完成节点。
- 已取消且未签到：高亮创建与取消节点。
- 已取消且已签到：高亮创建、签到和取消节点。
- 未发生分支保持灰色。
- 时间线只读。
- 失败签到或失败完成进入审计日志，不高亮节点。

### 4.4 节点详情

- 点击已发生节点打开 Popover。
- 展示动作、操作者、时间和结果。
- 签到节点展示 Mock 定位摘要。
- 完成节点展示跟进记录标识。
- 取消节点展示动作原因摘要。
- Popover 可键盘关闭。

## 5. 计划信息卡片

### 5.1 布局

- 标题 `拜访信息`。
- 使用 Description KV 两列。
- 标签宽度 112px。
- 长文本独占整行。
- 空值显示 `--`。
- 不展示编辑图标。

### 5.2 KV 清单

| 字段名 | 展示形态 | 交互 |
|--------|----------|------|
| 拜访编号 | 等宽文本 | 复制 |
| 拜访标题 | 普通文本 | 无 |
| 关联对象类型 | 图标 + 文本 | 无 |
| 关联对象ID | 链接 | 进入对应对象详情 |
| 拜访方式 | 图标 + 文本 | 无 |
| 计划拜访时间 | 完整时间 | 无 |
| 拜访地址 | 普通文本 | 复制 |
| 拜访状态 | Tag | 无 |
| 签到时间 | 完整时间或 `--` | 无 |
| 签到地址 | 普通文本或 `--` | 复制 |
| 拜访内容 | 保留换行文本或 `--` | 展开/收起 |

### 5.3 地址区分

- `拜访地址`是计划时填写的目标地点或会议链接。
- `签到地址`是签到动作返回的 Mock 定位结果。
- 两者永不自动互相覆盖。
- 不一致时不判定失败，只在详情并列展示。
- 定位失败时签到不会成功，因此签到地址为空。
- 地址超长时两行省略并提供 Tooltip。

### 5.4 拜访内容

- 只有完成状态应存在有效拜访内容。
- 保留换行与段落。
- 默认最多显示 8 行。
- 超出时显示“展开全部”。
- 自动跟进记录内容与拜访内容保持来源关联。
- 详情页不允许修改拜访内容。

## 6. 关联对象摘要

### 6.1 通用结构

| 内容 | 展示 |
|------|------|
| 对象类型 | 图标 + 文案 |
| 对象编号 | 等宽文本 |
| 对象名称 | 两行省略 |
| 当前状态摘要 | 引用对应对象字段清单 |
| 最近跟进时间 | 完整时间或 `--` |
| 操作 | 查看关联对象 |

### 6.2 线索关联

- 展示线索编号和公司名称。
- 展示线索当前状态摘要。
- 若线索已转客户，保留历史线索链接。
- 不在拜访详情执行转客户。
- 完成后跟进记录挂在线索来源下。

### 6.3 商机关联

- 展示商机编号和名称。
- 展示当前阶段。
- 展示最新 AI 成交概率摘要。
- 完成后跟进记录挂在商机下。
- 商机终态不影响历史拜访查看。
- 拜访完成不直接推进商机。

### 6.4 客户关联

- 展示 ERP 客户编号和公司名称。
- 展示客户等级与同步标识。
- 客户基础信息只读。
- 完成后跟进记录挂在客户下。
- ERP停用不删除历史拜访。

### 6.5 关联对象异常

- 无权访问时显示 `关联对象不可访问`。
- 对象不存在时显示历史编号和异常提示。
- 区块失败不影响拜访主数据。
- 不提供重新关联入口。
- 关联错误只能通过取消并新建修正。

## 7. 签到结果区块

### 7.1 已计划

- 显示 `尚未签到`。
- 展示计划时间。
- 到达允许窗口时有权限用户显示签到动作。
- 未到时间显示可签到时间提示。
- 过期显示迟到提示。

### 7.2 已签到或已完成

展示：

- 签到成功图标。
- 签到时间。
- 签到地址。
- 定位 Mock 来源说明。
- 是否迟到摘要。
- 操作者。

### 7.3 签到失败历史

- 失败尝试不写签到字段。
- 审计日志展示失败时间和用户可见原因。
- 主区块仍显示 `尚未签到`。
- 可按当前状态重试。
- 不展示伪造地址。

## 8. 完成结果与自动跟进

### 8.1 完成一致性

完成状态下必须同时满足：

- 拜访状态为完成。
- 拜访内容存在并通过校验。
- 自动跟进记录存在。
- 跟进记录来源为当前拜访。
- 跟进记录关联对象与拜访一致。

任一不满足时显示一致性异常，并仅允许管理员处理，不允许用户重复完成生成第二条记录。

### 8.2 跟进记录摘要

| 内容 | 展示 |
|------|------|
| 跟进记录标识 | 等宽文本 |
| 来源 | `来自拜访` Tag |
| 关联对象 | 类型 + 编号链接 |
| 跟进时间 | 拜访完成时间 |
| 跟进内容 | 拜访内容摘要 |
| 操作 | 查看关联对象时间线 |

### 8.3 跟进记录展示规则

- 完成状态展示摘要。
- 非完成状态不展示伪记录。
- 取消状态显示 `取消拜访未生成跟进记录`。
- 跟进加载失败时显示局部重试。
- 跟进记录不可在拜访详情编辑。
- 重复完成返回同一条跟进记录。

### 8.4 AI 回流摘要

- 关联商机时显示 `已提交商机预测刷新` 或失败提示。
- 关联客户时显示 `将参与下一次流失风险计算`。
- 关联线索时显示 `最近跟进时间已更新`。
- AI 失败不改变拜访完成和跟进记录。
- AI 结果不在拜访详情直接修改。

## 9. 审计日志

### 9.1 记录范围

- 创建计划。
- 编辑计划。
- 签到成功或失败。
- 完成成功或失败。
- 自动跟进记录生成。
- 取消。
- 权限或状态阻断。
- AI 回流结果。

### 9.2 时间线项

每条展示：

- 动作名称。
- 操作者或系统。
- 操作时间。
- 原状态与目标状态。
- 结果摘要。
- 失败原因。
- 请求标识摘要。

### 9.3 展示规则

- 默认最近 10 条。
- 支持加载更多。
- 时间倒序。
- 普通销售只看业务摘要。
- 管理员可展开技术摘要。
- 不展示定位或接口认证凭证。
- 取消动作原因按权限展示。

## 10. 底部操作矩阵

### 10.1 布局

- 固定底栏高度 64px。
- 左侧为返回列表。
- 右侧按次要、主要、危险顺序排列。
- 最多 4 个文本按钮。
- 超出动作进入 More。
- 不可用动作不渲染。

### 10.2 状态矩阵

| 状态 | 左侧 | 主操作 | 次操作 | More |
|------|------|--------|--------|------|
| 已计划 | 返回列表 | 签到 | 编辑 | 取消 |
| 已签到 | 返回列表 | 完成 | 无 | 取消 |
| 已完成 | 返回列表 | 查看跟进 | 无 | 无 |
| 已取消 | 返回列表 | 无 | 无 | 无 |

### 10.3 权限叠加

- 销售操作自己负责的拜访。
- 主管操作团队授权拜访。
- 管理员可处理异常。
- 只读审计只返回和查看。
- 任一角色均无删除。
- 终态没有写操作。

## 12. 并发与异常反馈

| 场景 | 页面处理 |
|------|----------|
| 编辑期间被签到 | 编辑保存失败，跳转最新详情 |
| 签到与取消并发 | 首个成功；另一动作刷新状态 |
| 完成与取消并发 | 首个成功；禁止两个终态共存 |
| 跟进写入失败 | 完成事务回滚，保留内容 |
| 重复完成 | 返回首次跟进记录 |
| 关联对象不可访问 | 关联区块提示，无重新关联 |
| AI 回流失败 | 拜访和跟进保持成功，显示弱提示 |

## 13. Mock 详情场景

### 13.1 主场景

- 一条已签到的商机拜访。
- 展示计划地址和不同的签到地址。
- 时间线高亮创建与签到。
- 关联商机处于商务谈判。
- 主操作为“完成”。
- 审计日志包含创建、签到和一次 AI 预检查。

### 13.2 补充场景

- 已计划未到时间：签到入口提示时间。
- 已计划已过期：迟到警示。
- 定位失败：无签到时间与地址。
- 已完成：展示唯一自动跟进记录。
- 完成失败：状态保持已签到且内容保留。
- 已取消未签到：取消分支正确。
- 已取消已签到：保留签到历史但无跟进。
- 关联客户已停用：历史可查看。

## 15. 键盘与可访问性

- 状态时间线节点可键盘聚焦。
