# 拜访计划_Demo_列表页

> **版本**：V2.0 | 2026-07-18
> **页面类型**：PC Web 列表页
> **前置文档**：《拜访计划主PRD》《拜访计划字段清单》
> **字段约束**：字段定义、状态与 Tag 配色引用《拜访计划字段清单》，本文不重复定义。

---

## 1. 页面概述

| 项目 | 内容 |
|------|------|
| 页面名称 | 拜访管理 |
| 页面路径 | `/visits` |
| 页面目标 | 查询销售拜访计划并执行当前状态允许的动作 |
| 主要用户 | 销售、销售主管、系统管理员、只读审计角色 |
| 页面骨架 | 页头 → 状态 Tab → 查询区 → 工具条 → 表格 → 分页 |
| 默认排序 | 计划拜访时间升序，过期未完成优先 |
| 每页条数 | 默认 20 条，可切换 20 / 50 / 100 |
| 详情入口 | 拜访编号链接 |

页面原则：

- 状态不在表格内直接修改。
- 不提供删除拜访。
- 签到、完成、取消均由动作触发。
- 完成动作必须保证跟进记录同时生成。
- 关联对象链接按权限展示。
- 终态仅查看。
- 表格最小宽度 1540px。

## 2. 状态 Tab 与后台计数

### 2.1 Tab 规格

| Tab | 过滤语义 | 计数来源 | 点击行为 |
|-----|----------|----------|----------|
| 全部 | 当前权限范围内全部拜访 | 后台聚合 | 清除状态条件，保留其他查询 |
| 已计划 | 尚未签到的拜访 | 后台聚合 | 筛选并回到第一页 |
| 已签到 | 已签到待完成的拜访 | 后台聚合 | 筛选并回到第一页 |
| 已完成 | 已完成且跟进已生成 | 后台聚合 | 筛选并回到第一页 |
| 已取消 | 已取消拜访 | 后台聚合 | 筛选并回到第一页 |

### 2.2 计数规则

- Tab 格式为 `名称 (计数)`。
- 计数按数据权限聚合。
- 计数受查询区条件影响。
- 计数不受分页影响。
- 签到后已计划减一、已签到加一。
- 完成后已签到减一、已完成加一。
- 取消后来源状态减一、已取消加一。
- 新建后全部和已计划各加一。
- 计数失败显示 `--`。
- 从详情返回时恢复 Tab。

## 3. 查询区

### 3.1 布局

- 默认两行自适应。
- 控件标签位于上方。
- 第一行放编号、标题和关联对象。
- 第二行放方式、状态、时间和创建人。
- “查询”为主按钮。
- “重置”为次按钮。
- 查询后回第一页。
- 条件写入 URL。

### 3.2 控件详细规格

| 字段/条件 | 控件 | 宽度 | Placeholder | 规格 | 查询行为 |
|-----------|------|------|-------------|------|----------|
| 拜访编号 | Input | 200px | `请输入拜访编号` | 支持粘贴、清空 | 精确匹配优先 |
| 拜访标题 | Input | 220px | `请输入拜访标题` | 去除首尾空格 | 模糊匹配 |
| 关联对象类型 | Select | 180px | `请选择关联类型` | 选项引用字段清单 | 联动对象选择器 |
| 关联对象ID | SelectWithSearch | 220px | `请先选择关联类型` | 远程搜索；依赖类型 | 按对象标识过滤 |
| 拜访方式 | Select | 160px | `请选择拜访方式` | 选项引用字段清单 | 单选过滤 |
| 拜访状态 | Select | 160px | `请选择拜访状态` | 选项引用字段清单 | 与 Tab 联动 |
| 计划拜访时间 | DateTimeRangePicker | 300px | `开始时间 - 结束时间` | 含首尾时间 | 范围过滤 |
| 创建人 | SelectWithSearch | 180px | `请选择创建人` | 主管和管理员可见 | 按用户过滤 |

### 3.3 联动查询

- 未选关联对象类型时，关联对象 ID 控件禁用。
- 选择类型后清空旧对象值。
- 不同类型调用对应远程查询。
- 远程选项只返回用户可见对象。
- 清空类型时同步清空对象。
- 查询失败在下拉内提供重试。
- 查询条件不复制关联对象字段定义。

### 3.4 查询反馈

- 标题输入按回车触发查询。
- 查询中按钮显示 Loader。
- 查询失败保留旧结果。
- Toast `拜访查询失败，请重试`。
- 重置清空条件并回到“全部”。
- 无结果显示 `未找到符合条件的拜访计划`。
- 快速查询取消旧请求。

## 5. 表格规格

### 5.1 表格行为

- 表头吸顶。
- 拜访编号列左固定。
- 操作列右固定。
- 行高 56px。
- 不支持行选择。
- 不支持整行点击。
- 空值统一显示 `--`。
- 过期未完成行展示警示图标。
- 行内动作只锁定当前行。

### 5.2 列定义

| 列名 | 宽度 | 固定 | 对齐 | 展示规则 |
|------|------|------|------|----------|
| 拜访编号 | 168px | 左 | 左 | 链接，等宽文本 |
| 拜访标题 | 220px | 否 | 左 | 两行省略，Tooltip 全文 |
| 关联对象类型 | 128px | 否 | 中 | 文本/Tag，口径引用字段清单 |
| 关联对象ID | 180px | 否 | 左 | 链接到对应详情 |
| 拜访方式 | 112px | 否 | 中 | 图标 + 文案 |
| 计划拜访时间 | 176px | 否 | 中 | 完整时间；过期警示 |
| 拜访状态 | 120px | 否 | 中 | Tag 引用字段清单 |
| 签到时间 | 176px | 否 | 中 | 无值 `--` |
| 创建人 | 112px | 否 | 左 | 单行省略 |
| 最近操作时间 | 176px | 否 | 中 | 完整时间 |
| 操作 | 220px | 右 | 左 | 动态操作，超出 More |

### 5.3 过期提示

- 计划时间已过且仍为已计划时显示红色 ClockAlert。
- Tooltip：`计划时间已过，请尽快处理`。
- 过期不自动取消。
- 过期仍可按业务规则签到。
- 过期签到写入迟到审计摘要。
- 颜色不是唯一提示，必须有图标与文本。

## 6. 行内操作动态矩阵

| 拜访状态 | 主操作 | 次操作 | More 菜单 |
|----------|--------|--------|-----------|
| 已计划 | 签到 | 查看、编辑 | 取消 |
| 已签到 | 完成 | 查看 | 取消 |
| 已完成 | 查看 | 查看跟进 | 无 |
| 已取消 | 查看 | 无 | 无 |

动态规则：

- 动作按状态和权限共同决定。
- 无权动作不渲染。
- 不可用动作不显示 disabled。
- 完成动作会打开内容补充与确认弹窗。
- 已完成不展示编辑、签到、完成或取消。
- 已取消不展示任何写动作。
- 任一状态不展示删除。
- More 使用 Ellipsis 图标并有 Tooltip。

## 7. 签到交互

### 7.1 发起签到

1. 用户点击“签到”。
2. 前端校验当前行仍为已计划。
3. 打开签到确认弹窗。
4. 用户点击“开始定位并签到”。
5. 请求 PC Demo 定位 Mock。
6. Mock 成功后提交签到时间与地址。
7. 后端实时校验状态和版本。
8. 成功后刷新行与 Tab 计数。

### 7.2 签到弹窗

| 项目 | 规格 |
|------|------|
| 标题 | `拜访签到` |
| 内容 | `系统将获取 Mock 签到时间和地址。` |
| 计划摘要 | 拜访标题、关联对象、计划时间 |
| 取消 | `取消` |
| 确认 | `开始定位并签到` |
| 加载 | `定位中...` |

### 7.3 签到成功

- 状态更新为已签到。
- 展示签到时间。
- 当前行操作变为“完成”。
- Toast `签到成功`。
- Tab 计数刷新。
- 详情时间线新增签到节点。

### 7.4 签到失败

| 失败 | 行为 | 文案 |
|------|------|------|
| 定位 Mock 失败 | 保持已计划 | `定位失败，未完成签到` |
| 时间窗口不允许 | 保持已计划 | `当前尚未到可签到时间` |
| 状态已变化 | 刷新当前行 | `拜访状态已变化` |
| 无权限 | 保持原状态 | `无权执行签到` |
| 服务异常 | 保持原状态 | `签到失败，请重试` |

失败不得写入签到时间或地址。

## 8. 完成交互

### 8.1 完成弹窗

| 项目 | 规格 |
|------|------|
| 标题 | `完成拜访` |
| 内容 | `完成后将自动生成一条关联对象的跟进记录。` |
| 输入字段 | 拜访内容 |
| Placeholder | `请输入本次拜访的沟通内容和结果` |
| 约束 | 引用拜访字段清单 |
| 取消 | `继续记录` |
| 确认 | `确认完成` |

### 8.2 提交步骤

1. 校验拜访内容。
2. 携带当前版本提交完成请求。
3. 后端校验状态仍为已签到。
4. 后端校验关联对象仍可写入跟进。
5. 同一事务更新拜访并生成跟进记录。
6. 成功后关闭弹窗。
7. 刷新当前行和计数。
8. Toast `拜访已完成，跟进记录已生成`。

### 8.3 失败处理

- 拜访内容错误时就地标红。
- 跟进写入失败时整个事务回滚。
- 拜访保持已签到。
- 弹窗保留输入内容。
- Toast `完成失败，拜访状态与跟进记录均未变更`。
- 恢复后重试只生成一条跟进记录。
- 并发冲突刷新当前行。

## 9. 取消交互

### 9.1 取消弹窗

| 项目 | 规格 |
|------|------|
| 标题 | `取消拜访` |
| 内容 | `取消后拜访将进入只读终态且不会生成跟进记录。` |
| 输入 | 取消动作原因，写入审计日志 |
| Placeholder | `请输入取消原因` |
| 取消按钮 | `继续保留` |
| 确认按钮 | `确认取消`，危险样式 |

### 9.2 成功与失败

- 成功后状态变为已取消。
- 当前行只保留查看。
- Tab 计数刷新。
- Toast `拜访已取消`。
- 失败时保留原因。
- 并发完成后取消请求阻断。
- 取消不删除拜访。
- 取消不生成跟进记录。

## 10. 分页、刷新与空态

### 10.1 分页

- 默认 20 条。
- 支持 20、50、100 条。
- 显示总条数。
- 切页回表格顶部。
- 改变每页条数回第一页。
- URL 保存分页状态。

### 10.2 自动刷新

- 页面可见时每 60 秒刷新。
- 弹窗打开时不替换对应行。
- 自动刷新保留筛选和滚动位置。
- 到达计划时间不自动改变状态。
- 自动刷新失败使用弱提示。

### 10.3 空态

| 场景 | 文案 | 动作 |
|------|------|------|
| 无拜访 | `暂无拜访计划` | 有权限时新建拜访 |
| 今日为空 | `今天没有拜访计划` | 返回全部 |
| 查询无结果 | `未找到符合条件的拜访计划` | 重置筛选 |
| 服务异常 | `拜访数据暂不可用` | 重试 |
