# WMS 出库管理主PRD

> 版本 V1.0 | 2026-07-06 | 强盛科技 WMS | 日均 20,000+ 单

## 1. 业务背景

销售订单下发到 WMS → 波次聚合 → 拣货 → 播种(多订单分播) → 复核 → 包装 → 交运承运商。核心痛点：拣货路径混乱、漏拣错拣频发、包裹和订单对应不上。

## 2. 功能范围

**In Scope**：系统波次+手动波次 → PDA 拣货 → 播种 → 复核校验 → 称重包装 → 交运交接
**Out Scope**：智能波次引擎(三期)、物流轨迹实时跟踪、快递公司 API 对接

## 3. 单据体系

| 单据 | 前缀 | 来源 | 核心动作 |
|:--|:--|:--|:--|
| 波次单 WAVE | WAVE{日期}-{序号} | 系统/手动聚合订单 | 分配拣货员 |
| 拣货单 PICK | PICK{日期}-{序号} | 波次下推 | PDA扫描拣货 |
| 复核单 CHECK | CHECK{日期}-{序号} | 拣货完成 | 校验包裹商品 |
| 包裹 PKG | PKG{日期}-{6位序号} | 复核通过 | 称重贴面单 |
| 交运单 DSH | DSH{日期}-{序号} | 包裹聚合 | 交接承运商 |

## 4. 状态机

> 本状态机描述 WMS WAVE 波次单（亦作为出库主流程及出库需求单进度）的生命周期状态。

| 状态 | 含义 | 对应子单据状态 | 可用操作 |
|:--|:--|:--|:--|
| 草稿 DRAFT | 波次已聚合生成，等待下推开始 | WAVE: `草稿`；出库需求: `waved` | 分配人员、下推拣货、作废 |
| 拣货中 PICKING | 已下推拣货单，PDA 拣货作业中 | WAVE: `拣货中`；PICK: `待拣/拣货中` | PDA 拣货、异常关闭 |
| 已拣货 PICKED | 拣货分播完成，等待复核校验 | WAVE: `已拣货`；PICK: `已拣完`；CHECK: `待复核` | 复核校验 |
| 已复核 CHECKED | 复核完成且包装贴单完毕，等待交运 | WAVE: `已复核`；CHECK: `已完成`；PKG: `已完成` | 交运确认 |
| 已交运 SHIPPED | 包裹已交接承运商，出库任务完结 | WAVE: `已交运`；DSH: `已交运` | —（终态） |
| 已作废 VOIDED | 上游取消订单或波次异常关闭 | WAVE: `已作废`；PICK: `已作废` | —（终态） |

流转：草稿 → 拣货中 → 已拣货 → 已复核 → 已交运

## 5. 核心业务规则

- R01：波次按承运商+线路+优先级自动聚合，单波次上限 50 单
- R02：PDA 拣货——扫描货位条码→扫描商品条码→输入拣货量→确认
- R03：复核——扫描包裹号→逐件扫描商品，数量不匹配报警
- R04：播种——多订单同 SKU 集中拣货后，按订单分播到对应发货箱
- R05：拣货超量拦截——拣货量 > 订单量时 PDA 报错阻断
- R06：包装完成 = 库存扣减触发点（现存-N、释放占用-N，可用不变；生成 FL）
- R07：交运确认 = 订单状态完结，回传进销存

## 6. 验收重点

| 验收项 | 预期 |
|:--|:--|
| 波次聚合 | 同线路订单自动进同一波次 |
| 拣货超量 | PDA 扫描超量时报警阻断 |
| 复核不通过 | 退回拣货环节 |
| 包装完成 | 库存扣减（现存-N、释放占用-N，可用不变） + FL 生成 |
