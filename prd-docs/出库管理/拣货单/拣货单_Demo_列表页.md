# 拣货单_Demo_列表页

> 角色：Demo·列表页 | 类型：执行作业单
> 页面只描述结构、控件、交互和 Mock；字段定义以 `拣货单字段清单.md` 为准，不重复维护字段枚举。

## 1. 页面定位

拣货单列表页用于 PC 端查看由波次下推生成的 PICK，跟踪待拣、拣货中、已拣完任务和缺货异常。列表页不提供新增入口，避免绕过 WAVE。

## 2. 页面入口

| 入口 | 说明 |
|:--|:--|
| 菜单 | 出库管理 / 拣货单 |
| 关联跳转 | 波次详情、复核单详情、出库管理概览 |

## 3. 页面结构

```text
拣货单列表
├─ 顶部筛选区
│  ├─ 关键词：拣货单号/波次单号/拣货员/SKU
│  ├─ 状态、仓库、拣货模式、是否异常
│  ├─ 创建时间范围
│  └─ 查询、重置
├─ 表格区
│  ├─ 拣货单号、来源波次、状态、仓库、拣货员、拣货模式
│  ├─ 应拣总数、实拣总数、缺货总数、进度、创建时间
│  └─ 行操作：查看详情
└─ 分页
   └─ 默认 20 条/页
```

## 4. 行操作规则

| 场景 | 行操作 |
|:--|:--|
| 状态=待拣 | 查看详情 |
| 状态=拣货中 | 查看详情 |
| 状态=已拣完 | 查看详情、查看复核单（有 CHECK 时） |
| 状态=已作废 | 查看详情 |
| 异常标记=true | 展示异常标识，可进入详情查看缺货明细 |

> 列表页不展示“新增拣货单”。PDA 领取任务从 PDA 任务入口进入，不从 PC 列表直接执行扫码。

## 5. 查询规则

- 关键词支持拣货单号、来源波次单号、拣货员、商品编码模糊查询。
- 状态筛选只使用 PICK 自身状态。
- 是否异常根据缺货、扫错阻断记录等系统字段汇总。

| 状态 | 展示 |
|:--|:--|
| 无数据 | `暂无拣货单` |
| 查询无结果 | `未找到符合条件的拣货单` |
| 接口失败 | 保留筛选条件，提示 `拣货单列表加载失败，请重试` |
