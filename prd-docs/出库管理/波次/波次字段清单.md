# 波次字段清单

> 角色：字段清单(SSOT) | 类型：业务单据
> 本文件为波次字段唯一事实来源；主 PRD、业务规则、流程推演、Demo 页面引用本文件，不重复维护字段口径。

## 1. 字段设计原则

- 单号、状态、系统聚合结果、系统统计字段不可手工编辑。
- 数量字段按 `11-字段校验规范`：正整数、`>0`、`≤上限值`。
- 日期格式 `YYYY-MM-DD`，时间格式 `YYYY-MM-DD HH:mm:ss`。
- 备注上限 200 字符，空值展示 `-`。
- 商品、承运商、线路、优先级等信息在波次中快照存储，便于追溯当时的出库组织口径。

## 2. 波次头字段

|:--|:--|:--|:--|:--:|:--:|:--|

| wave_type | 波次类型 | enum | 系统聚合/用户选择 | 是 | 否 | `system` 系统波次、`manual` 手动波次 |
| status | 波次状态 | enum | 系统 | 是 | 否 | 见“枚举值”；状态只能通过动作按钮触发 |
| warehouse_code | 仓库编码 | string | 系统聚合/用户选择 | 是 | 条件可编辑 | 必须为启用仓库；系统波次按出库需求带入 |
| warehouse_name | 仓库名称 | string | 仓库档案 | 是 | 否 | 随仓库编码带出 |
| carrier_code | 承运商编码 | string | 系统聚合/用户选择 | 是 | 条件可编辑 | 系统波次聚合维度之一 |
| carrier_name | 承运商名称 | string | 系统聚合/用户选择 | 是 | 条件可编辑 | 如顺丰、京东物流；按出库需求快照 |
| route_code | 线路编码 | string | 系统聚合/用户选择 | 是 | 条件可编辑 | 系统波次聚合维度之一 |
| route_name | 线路名称 | string | 系统聚合/用户选择 | 是 | 条件可编辑 | 如上海-浦东线 |
| ship_priority | 发货优先级 | enum | 系统聚合/用户选择 | 是 | 条件可编辑 | `urgent` 加急、`normal` 普通、`low` 低 |
| pick_mode | 拣货模式 | enum | 系统推荐/用户选择 | 是 | 是 | `single_order` 按单拣货、`pick_sort` 边拣边分、`batch_then_sort` 先拣后播 |
| picker_id | 拣货员ID | string | 用户选择 | 否 | 是 | 待拣状态可分配；必须为启用账号 |
| picker_name | 拣货员 | string | 用户选择 | 否 | 否 | 随拣货员 ID 带出 |
| order_count | 订单数 | integer | 系统算 | 是 | 否 | `count(distinct demand_no)`，单波次 `≤50` |
| sku_count | SKU 数 | integer | 系统算 | 是 | 否 | `count(distinct sku_code)` |
| total_qty | 总件数 | integer | 系统算 | 是 | 否 | `Σ wave_qty` |
| planned_pick_date | 计划拣货日期 | date | 系统聚合/用户选择 | 否 | 是 | 不超过当前日期规则不适用；用于作业计划 |
| remark | 备注 | string | 用户输入 | 否 | 是 | ≤200 字符 |

## 3. 波次明细字段（关联的出库需求行）

| 字段编码 | 字段名称 | 类型 | 来源 | 必填 | 可编辑 | 枚举/校验/说明 |

| receiver_area | 收货区域 | string | ERP SO | 否 | 否 | 用于人工识别线路合理性 |
| sku_code | 商品编码 | string | ERP SO / 商品档案 | 是 | 否 | 字母+数字 |
| sku_name | 商品名称 | string | ERP SO / 商品档案 | 是 | 否 | 商品快照 |
| spec | 规格 | string | ERP SO / 商品档案 | 否 | 否 | 无值展示 `-` |
| unit | 单位 | string | ERP SO / 商品档案 | 是 | 否 | 如 件、箱 |
| order_qty | 订单数量 | integer | ERP SO | 是 | 否 | 正整数 `>0` |
| occupied_qty | 占用数量 | integer | 系统算 | 是 | 否 | SO 审核后占用数量；波次生成不新增占用 |
| wave_qty | 波次数量 | integer | 系统算/用户选择 | 是 | 条件可编辑 | 默认等于待出库数量；正整数 `>0`，`≤ order_qty` |

## 5. 枚举值

### 5.1 波次类型 `wave_type`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| system | 系统波次 | 系统按规则自动聚合 |
| manual | 手动波次 | 仓管手动圈选出库需求生成 |

### 5.2 波次状态 `status`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| pending_pick | 待拣 | 已生成，尚未下推或开始拣货 |
| picking | 拣货中 | 已下推 PICK，PDA 正在执行或等待执行 |
| completed | 已完成 | 关联下游出库链路完成 |
| closed | 已关闭 | 波次取消或异常终止，不物理删除 |

### 5.3 发货优先级 `ship_priority`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| urgent | 加急 | 优先生成和处理 |
| normal | 普通 | 默认优先级 |
| low | 低 | 低优先处理 |

### 5.4 拣货模式 `pick_mode`

| 值 | 展示 | 适用场景 |
|:--|:--|:--|
| single_order | 按单拣货 | 大件/异形商品 |
| pick_sort | 边拣边分 | 多订单同时拣，PDA 提示分播到发货箱 |
| batch_then_sort | 先拣后播 | 大批量同 SKU，集中拣货后统一播种 |

## 6. 计算字段

| 字段 | 公式 | 说明 |
|:--|:--|:--|
| 订单数 | `count(distinct demand_no)` | 单波次最多 50 单 |
| SKU 数 | `count(distinct sku_code)` | 用于判断拣货复杂度 |
| 总件数 | `Σ wave_qty` | 波次作业总数量 |
| 聚合键 | `warehouse + carrier + route + ship_priority` | context 明确承运商、线路、优先级；仓库用于避免跨仓作业 |

## 7. 校验时机

| 时机 | 校验范围 |
|:--|:--|
| 实时失焦 | 数量格式、备注长度 |
| 保存/生成 | 仓库、承运商、线路、优先级、明细数量、订单数上限 |
| 下推拣货 | 状态为待拣、明细未取消、未重复入其他未完成波次、拣货员有效 |
| 关闭波次 | 状态允许关闭、关闭原因必填、二次确认 |
