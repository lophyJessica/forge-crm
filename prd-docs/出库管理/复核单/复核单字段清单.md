# 复核单字段清单

> 角色：字段清单(SSOT) | 类型：执行作业单
> 本文件为复核单字段唯一事实来源；主 PRD、业务规则、流程推演和 Demo 页面引用本文件，不重复维护字段口径。

## 1. 字段设计原则

- CHECK 单号、来源 PICK、来源 WAVE、应复核数量、商品快照、发货箱/包厢不可手工编辑。
- 实复核数量优先由扫码累计；异常补录需权限，并记录操作原因。
- 未完成扫描只表示作业进度，不直接计入差异；复核员确认不通过后才固化少件/多件/错品/串箱差异。
- 复核数量允许记录现场差异；确认复核通过时必须满足 `实复核数=应复核数` 且明细状态均为已匹配。
- 日期格式 `YYYY-MM-DD`，时间格式 `YYYY-MM-DD HH:mm:ss`。
- 空值展示 `-`，备注/退回原因说明 ≤200 字符。
- CHECK 不维护库存过账字段；库存现存、占用、可用不因 CHECK 变化。

## 5. 枚举值

### 5.1 复核状态 `status`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| pending_check | 待复核 | PICK 已拣完，等待复核员开始 |
| checking | 复核中 | 已领取或已有扫码记录 |
| checked | 已复核 | 全部明细数量一致，复核通过 |
| returned_pick | 已退回拣货 | 复核不通过，已回退 PICK |
| voided | 已作废 | 上游波次/拣货任务异常关闭，任务取消 |

### 5.2 明细状态 `line_status`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| pending | 待复核 | 尚未开始该明细 |
| checking | 复核中 | 已扫箱/扫商品但未完成 |
| matched | 已匹配 | 实复核数量=应复核数量 |
| different | 有差异 | 少件、多件、错品或串箱 |
| returned | 已退回 | 随 CHECK 退回拣货 |

### 5.3 差异类型 `diff_type`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| none | 无差异 | 未形成确认差异，或实复核数量=应复核数量 |
| short | 少件 | 复核员确认不通过时，实复核数量<应复核数量 |
| over | 多件 | 实复核数量>应复核数量 |
| wrong_sku | 错品 | 实扫商品不属于当前 CHECK 明细 |
| wrong_box | 串箱 | 实扫发货箱/包厢不属于当前 CHECK |

### 5.4 退回原因 `return_reason`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| quantity_short | 少件 | 实复核数量少于应复核数量 |
| quantity_over | 多件 | 实复核数量多于应复核数量 |
| wrong_sku | 错品 | 扫到非当前订单/发货箱商品 |
| wrong_box | 串箱 | 扫到非当前复核单箱码 |
| scan_exception | 条码异常 | 商品或箱码条码无法识别 |
| other | 其他 | 需填写退回说明 |

## 6. 计算字段

| 字段 | 公式 | 说明 |
|:--|:--|:--|
| 应复核总数 | `Σ should_check_qty` | 复核任务总量 |
| 实复核总数 | `Σ actual_check_qty` | 已成功扫描或授权补录数量合计 |
| 待扫描数量 | `max(should_check_qty - actual_check_qty, 0)` | 用于区分“未扫完”和“少件差异” |
| 差异数量 | 已确认差异时 `actual_check_qty - should_check_qty`，否则为 0 | 明细级，可为负数/0/正数 |
| 差异总数 | `Σ 已确认差异明细.diff_qty` | 头部汇总，用于展示已确认差异方向 |
| 差异行数 | `COUNT(line_status in [different, returned])` | 少件、多件、错品、串箱均计入；待复核/复核中不计入 |
| 复核通过条件 | `所有明细 line_status=matched 且 diff_qty=0 且 diff_type=none` | 满足后可确认复核通过 |
| 复核进度 | `actual_check_qty / should_check_qty` | 可超过 100%，超出时标记差异 |

## 7. 校验时机

| 时机 | 校验范围 |
|:--|:--|
| 扫发货箱/包厢 | 箱码是否属于当前 CHECK |
| 扫商品 | 商品条码是否属于当前发货箱/订单明细 |
| 扫描累计 | 更新实复核数、待扫描数量；多件/错品/串箱即时标记差异 |
| 确认复核通过 | 所有明细 `line_status=matched` 且无错品/串箱异常 |
| 确认复核不通过 | 存在已确认差异，或有未匹配明细经复核员确认为少件；退回原因必填，原因=其他时退回说明必填 |
| 流转包装 | CHECK 状态必须为已复核 |
