# 包裹_Demo_列表页

> 角色：Demo·列表页 | 类型：执行作业单
> 页面只描述结构、控件、交互和 Mock；字段定义以 `包裹字段清单.md` 为准，不重复维护字段枚举。

## 1. 页面定位

包裹列表页用于 PC 端查看由复核通过下推生成的 PKG，跟踪待包装、包装中、已包装和库存过账结果。列表页不提供新增入口，避免绕过 CHECK。

## 2. 页面入口

| 入口 | 说明 |
|:--|:--|
| 菜单 | 出库管理 / 包裹 |
| 关联跳转 | 复核单详情、交运单详情、库存流水详情、出库管理概览 |
| 工作站入口 | 包装台登录后进入待包装任务列表 |

## 3. 页面结构

```text
包裹列表
├─ 顶部筛选区
│  ├─ 关键词：包裹号/复核单号/波次单号/面单号/SKU
│  ├─ 状态、仓库、承运商、包装员、过账状态
│  ├─ 创建时间范围、包装完成时间范围
│  └─ 查询、重置
├─ 表格区
│  ├─ 包裹号、来源复核单、状态、仓库、承运商、面单号
│  ├─ 重量、出库总数、过账状态、FL 单号、创建时间、包装完成时间
│  └─ 行操作：查看详情、开始包装、继续包装、查看交运、查看 FL
└─ 分页
   └─ 默认 20 条/页
```

## 4. 行操作规则

| 场景 | 行操作 |
|:--|:--|
| 状态=待包装 | 查看详情、开始包装 |
| 状态=包装中 | 查看详情、继续包装 |
| 状态=已包装 | 查看详情、查看交运（有 DSH 时）、查看 FL |
| 状态=已作废 | 查看详情 |
| 过账状态=过账失败 | 查看详情、继续包装 |

> 列表页不展示“新增包裹”。开始/继续包装只是进入包装执行页，不允许直接改状态。

## 5. 查询规则

- 关键词支持包裹号、来源复核单号、来源波次单号、面单号、商品编码模糊查询。
- 状态筛选只使用 PKG 自身状态。
- 过账状态用于定位未过账、已过账、过账失败任务。
- 已包装包裹默认可通过 FL 单号跳转库存流水。

## 7. 空状态与异常态

| 状态 | 展示 |
|:--|:--|
| 无数据 | `暂无包裹` |
| 查询无结果 | `未找到符合条件的包裹` |
| 接口失败 | 保留筛选条件，提示 `包裹列表加载失败，请重试` |
| 无包装权限 | 隐藏开始/继续包装，仅允许查看详情 |
