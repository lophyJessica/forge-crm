# 交运单_Demo_执行页

> 角色：Demo·执行页 | 类型：执行作业单
> 本页为交接承运商作业页，重点描述包裹扫描、交接信息确认、回传结果和库存不过账提示；字段定义以 `交运单字段清单.md` 为准。

## 1. 页面定位

交运执行页用于发货员将已包装包裹交接给承运商。页面执行“开始交运 → 扫描包裹/面单 → 核对数量 → 填写承运商交接信息 → 确认交运”。确认交运会完结订单并触发回传，但不触发库存过账。

## 2. 页面结构

```text
交运执行页
├─ 任务头
│  ├─ 交运单号、状态、仓库、承运商、线路
│  └─ 应交包裹数、实交包裹数、异常包裹数
├─ 扫码区
│  ├─ 包裹号/面单号扫码输入
│  └─ 最近扫描结果
├─ 包裹明细区
│  ├─ 包裹号、面单号、重量、扫描状态
│  └─ 交接结果、异常原因
├─ 交接信息区
│  ├─ 承运商交接人、手机号
│  ├─ 车牌号、交接地点
│  └─ 实际交接时间
├─ 回传结果区
│  ├─ TMS 回传状态
│  ├─ ERP 发货回执状态
│  └─ 财务凭证状态
└─ 底部动作区
   ├─ 保存交接信息
   └─ 确认交运
```

## 3. 核心交互

| 交互 | 说明 |
|:--|:--|
| 开始交运 | 待交运任务进入交运中，写入发货员和开始时间 |
| 扫描包裹/面单 | 校验包裹是否属于当前 DSH、是否已包装、承运商是否一致 |
| 重复扫描 | 提示已扫描，不重复计数 |
| 数量核对 | 实交包裹数必须等于应交包裹数 |
| 填写交接信息 | 承运商交接人、交接地点、实际交接时间必填 |
| 确认交运 | 状态变为已交运，订单完结，触发回传 |
| 重试回传 | 仅在已交运且回传失败时展示 |

## 4. 扫码反馈

| 场景 | 反馈 |
|:--|:--|
| 扫包裹正确 | 明细行高亮，实交数 +1 |
| 扫错包裹 | 红色提示、震动、语音提示“包裹不属于当前交运单” |
| 包裹未包装 | 阻断，提示“包裹未包装完成，不能交运” |
| 承运商不一致 | 阻断，提示“承运商不一致，请核对” |
| 重复扫描 | 提示“该包裹已扫描”，不重复计数 |
| 少交 | 确认交运时提示“存在未交接包裹” |

## 5. 页面状态

| DSH 状态 | 页面表现 |
|:--|:--|
| 待交运 | 展示开始交运按钮 |
| 交运中 | 展示扫码区、明细区、交接信息和确认交运 |
| 已交运 | 只读展示交接摘要和回传结果；回传失败时可重试 |
| 已作废 | 只读展示“任务已作废”，不允许交运 |

## 9. 离线与同步

- PDA 可离线缓存包裹扫描和交接信息草稿。
- 确认交运必须以后端在线校验为准，避免重复交运或包裹状态过期。
- 网络恢复后同步草稿，仍需重新点击确认交运。
- 已交运状态同步后只读，不允许再次确认。

## 10. 不展示内容

- 不展示新增交运单入口。
- 不展示审核按钮。
- 不展示拣货、复核、包装执行控件。
- 不展示库存扣减、释放占用或生成 FL 操作。
