# 交运单_Demo_列表页

> 角色：Demo·列表页 | 类型：执行作业单
> 页面只描述结构、控件、交互和 Mock；字段定义以 `交运单字段清单.md` 为准，不重复维护字段枚举。

## 1. 页面定位

交运单列表页用于 PC 端查看由已包装包裹聚合生成的 DSH，跟踪待交运、交运中、已交运任务，以及 TMS/ERP/财务回传结果。列表页不提供新增入口，避免绕过 PKG。

## 2. 页面入口

| 入口 | 说明 |
|:--|:--|
| 菜单 | 出库管理 / 交运单 |
| 关联跳转 | 包裹详情、波次详情、出库管理概览 |
| 工作站入口 | 发货区工作站进入待交运任务列表 |

## 3. 页面结构

```text
交运单列表
├─ 顶部筛选区
│  ├─ 关键词：交运单号/包裹号/面单号/承运商交接人
│  ├─ 状态、仓库、承运商、线路、回传状态
│  ├─ 创建时间范围、交运确认时间范围
│  └─ 查询、重置
├─ 表格区
│  ├─ 交运单号、状态、仓库、承运商、线路、交接人、交接时间
│  ├─ 应交包裹数、实交包裹数、异常数、TMS/ERP/财务回传状态
│  └─ 行操作：查看详情、开始交运、继续交运、重试回传
└─ 分页
   └─ 默认 20 条/页
```

## 4. 行操作规则

| 场景 | 行操作 |
|:--|:--|
| 状态=待交运 | 查看详情、开始交运 |
| 状态=交运中 | 查看详情、继续交运 |
| 状态=已交运 | 查看详情 |
| 状态=已交运且存在回传失败 | 查看详情、重试回传 |
| 状态=已作废 | 查看详情 |

> 列表页不展示“新增交运单”。开始/继续交运只是进入交运执行页，不允许直接改状态。

## 5. 查询规则

- 关键词支持交运单号、包裹号、面单号、承运商交接人模糊查询。
- 状态筛选只使用 DSH 自身状态。
- 回传状态可按 TMS/ERP/财务任一失败筛选。
- 已交运但回传失败的单据需要在列表突出异常标识。

## 7. 空状态与异常态

| 状态 | 展示 |
|:--|:--|
| 无数据 | `暂无交运单` |
| 查询无结果 | `未找到符合条件的交运单` |
| 接口失败 | 保留筛选条件，提示 `交运单列表加载失败，请重试` |
| 无交运权限 | 隐藏开始/继续交运，仅允许查看详情 |
