# 交运单_Demo_详情页

> 角色：Demo·详情页 | 类型：执行作业单
> 详情页用于查看交运单来源、包裹清单、承运商交接记录、回传状态、订单完结和库存不过账说明。

## 1. 页面定位

交运单详情页面向研发、测试、产品复核，用于核对 DSH 是否由已包装 PKG 聚合生成、包裹是否完整交接给承运商、TMS/ERP/财务回传结果是否可追溯，并确认 DSH 阶段没有发生库存过账。

## 2. 页面结构

```text
交运单详情
├─ 顶部摘要
│  ├─ 交运单号、状态、承运商、回传状态
│  └─ 返回列表、查看包裹、重试回传
├─ 交运头信息
│  ├─ 仓库、线路、交接批次、发货员
│  └─ 承运商交接人、交接地点、实际交接时间
├─ 包裹明细
│  ├─ 包裹号、面单号、重量、扫描状态
│  └─ 交接结果、异常原因
├─ 回传记录
│  ├─ TMS 回传状态和时间
│  ├─ ERP 发货回执状态和时间
│  └─ 财务凭证状态和时间
├─ 订单完结记录
│  └─ 关联订单/出库需求状态
└─ 操作记录
   └─ 生成、开始、扫码、确认交运、回传、重试
```

## 3. 展示规则

| 区域 | 展示规则 |
|:--|:--|
| 顶部摘要 | 状态只展示 DSH 自身状态：待交运、交运中、已交运、已作废 |
| 交运头信息 | 承运商交接人和交接时间在已交运后只读 |
| 包裹明细 | 包裹号可跳转 PKG 详情；数量右对齐 |
| 回传记录 | 回传失败展示失败原因和重试入口 |
| 订单完结记录 | 展示 DSH 确认后的订单已交运/完结状态 |
| 库存说明 | 展示“DSH 不做库存过账；库存实扣在上游 PKG 包装完成” |

## 4. 动作规则

| 动作 | 展示条件 | 说明 |
|:--|:--|:--|
| 查看包裹 | 有 pkg_no | 跳转 PKG 详情 |
| 进入交运执行 | 状态=待交运/交运中且有权限 | 跳转执行页 |
| 重试回传 | 状态=已交运且任一回传状态=失败 | 只重试回传，不改变交运结果 |
| 返回列表 | 全部状态 | 返回交运单列表 |

> 详情页不展示新增、审核、包装完成、库存过账、生成 FL 操作。

## 7. 测试关注点

- DSH 详情能追溯包裹 PKG。
- 状态不出现审核流状态。
- 扫错包裹、未包装包裹不能确认交运。
- 已交运后订单状态完结，回传记录可查看。
- 回传失败可重试，但不改变交运状态。
- DSH 阶段不生成 FL；库存变化只能追溯到上游包裹完成。
