# 合同_Demo_详情页

> **版本**：V2.0 | 2026-07-18
> **页面类型**：PC Web 详情页
> **前置文档**：《合同主PRD》《合同字段清单》
> **字段约束**：字段定义、状态语义和 Tag 配色引用《合同字段清单》，本文只定义详情展示与交互。

---

## 1. 页面概述

| 项目 | 内容 |
|------|------|
| 页面名称 | 合同详情 |
| 页面路径 | `/contracts/{id}` |
| 页面目标 | 查看合同状态、关联对象、签署事实和跨系统联动结果 |
| 进入方式 | 合同列表、商机详情合同链接、客户关联数据 |
| 数据权限 | 引用《合同主PRD》 |
| 页面骨架 | 页头状态区 → 合同时间线 → 信息卡片 → 关联对象 → 联动结果 → 审计日志 → 底部操作 |
| 页面宽度 | 最大 1200px，居中布局 |

详情页原则：

- 全部合同字段只读展示。
- 状态变化只通过底部动作或签署事件。
- 签署、商机赢单与 ERP 订单结果分别展示。
- 作废合同保留历史商机链接和动作原因审计。
- 不提供删除。
- 子区块独立加载和失败。
- 终态合同不展示编辑动作。

## 2. 初始化与权限

### 2.1 加载顺序

1. 校验合同查看权限。
2. 拉取合同主数据和版本。
3. 拉取状态时间线。
4. 并行拉取关联商机、客户快照和签署事件。
5. 拉取商机联动与 ERP 订单结果。
6. 拉取审计日志。
7. 根据合同状态、联动结果和权限计算动作栏。

### 2.2 页面状态

| 场景 | 页面表现 | 动作 |
|------|----------|------|
| 首次加载 | 固定骨架屏 | 等待 |
| 合同不存在 | 404 状态页 | 返回列表 |
| 无权限 | 403 状态页 | 返回列表 |
| 主数据失败 | 整页错误态 | 重试、返回列表 |
| 子区块失败 | 区块错误态 | 局部重试 |
| 状态并发变化 | 刷新主数据与动作栏 | 查看最新状态 |

## 4. 五节点合同时间线

### 4.1 节点

时间线固定展示：

1. 合同创建。
2. 提交签署。
3. 签署完成。
4. 归档。
5. 作废。

> 归档与作废是互斥分支；状态编码引用《合同字段清单》。

### 4.2 节点视觉

| 节点状态 | 视觉 | 时间 | 说明 |
|----------|------|------|------|
| 已完成 | 实心圆 + 主色连线 | 完整时间 | 真实发生的动作 |
| 当前待办 | 加粗描边 | 进入状态时间 | 当前可处理节点 |
| 未发生 | 灰色空心圆 | `--` | 尚未发生 |
| 归档终态 | 绿色 Archive 图标 | 归档时间 | 只读终态 |
| 作废终态 | 红色 Ban 图标 | 作废时间 | 只读终态 |

### 4.3 分支规则

- 草稿：只高亮创建节点。
- 待签署：高亮创建和提交签署。
- 已签署：高亮前三节点。
- 已归档：高亮签署主链路和归档分支。
- 已作废：高亮实际到达节点和作废分支。
- 草稿作废时不伪造提交签署节点。
- 待签署作废时保留提交签署节点。
- 时间线只读，不通过点击改变状态。

### 4.4 节点详情

- 点击已完成节点展示 Popover。
- Popover 展示动作、操作者、时间和结果。
- 签署节点展示外部事件标识摘要。
- 作废节点展示动作原因摘要。
- 失败动作不计入完成节点，进入审计日志。
- Popover 支持 Esc 关闭。

## 5. 合同信息卡片

### 5.1 布局

- 标题 `合同信息`。
- 使用 Description KV 两列。
- 标签宽度 112px。
- 长值独占整行。
- 空值统一显示 `--`。
- 不展示编辑图标。

### 5.2 KV 清单

| 字段名 | 展示形态 | 交互 |
|--------|----------|------|
| 合同编号 | 等宽文本 | 复制 |
| 合同名称 | 普通文本 | 无 |
| 关联客户 | 链接 | 进入客户详情 |
| 关联商机 | 链接 | 进入商机详情 |
| 合同金额 | 格式化金额 | 无 |

| 签署日期 | 日期或 `--` | 无 |

- 已作废合同仍展示历史关联。
- 签署日期没有签署事实时显示 `--`。
- 所有字段口径引用字段清单。

## 6. 关联客户与商机

### 6.1 客户摘要

| 内容 | 展示 |
|------|------|
| 公司名称 | 两行省略 |
| 客户编号 | 等宽文本 |
| 客户等级 | Tag |
| ERP同步标识 | 技术元数据提示 |
| 操作 | 查看客户 |

客户摘要规则：

- 数据来自 CRM 客户快照。
- 基础档案只读。
- 客户加载失败不影响合同主数据。
- ERP停用客户仍保留历史显示。

### 6.2 商机摘要

| 内容 | 展示 |
|------|------|
| 商机名称 | 两行省略 |
| 商机编号 | 等宽文本 |
| 商机阶段 | Tag |
| 预计金额 | 格式化金额 |
| AI 成交概率 | 最新成功结果 |
| 操作 | 查看商机 |

商机摘要规则：

- 待签署时提示商机被合同互锁。

- 联动异常时显示最新已知阶段和刷新入口。
- 不在合同详情直接推进商机。

## 7. 签署结果区块

### 7.1 未签署

- 草稿显示 `尚未提交签署`。
- 待签署显示 `等待签署完成`。
- 展示最近提交时间。
- 主管或管理员按权限看到“模拟签署完成”。

### 7.2 已签署

展示内容：

- 签署结果图标。
- 签署日期。
- 签署事件标识摘要。
- 事件接收时间。
- 幂等处理结果。
- 商机联动状态。
- ERP 下推状态。

### 7.3 签署异常

| 异常 | 表现 | 操作 |
|------|------|------|
| 重复事件 | 显示已忽略重复回调 | 无 |
| 事件验签失败 | 显示异常摘要 | 管理员查看日志 |
| 合同绑定不一致 | 红色错误提示 | 进入人工处理 |
| 商机联动失败 | 保留签署成功 | 主管或管理员重试 |

签署异常不允许用户手工把合同退回待签署。

## 8. 商机与 ERP 联动结果

### 8.1 联动步骤

| 顺序 | 节点 | 成功展示 | 失败展示 |
|:---:|------|----------|----------|
| 1 | 合同签署事实 | `已签署` | `签署结果异常` |
| 2 | 商机赢单 | `商机已赢单` | `商机联动失败` |
| 3 | ERP 订单草稿 | `ERP订单已生成` | `ERP下推失败` |
| 4 | 业绩累计 | `业绩已更新` | `业绩更新待重试` |

### 8.2 展示规则

- 使用纵向步骤条，不使用嵌套卡片。
- 每一步展示完成时间。
- 失败节点保留前序成功结果。
- 后序未执行显示 `等待前置节点完成`。
- 重试只作用于失败节点。
- 重试成功后继续执行后续节点。

### 8.3 ERP 订单摘要

| 内容 | 展示 |
|------|------|
| ERP 订单编号 | 成功后展示，链接到 Mock 详情 |
| 请求时间 | 完整时间 |
| 完成时间 | 完整时间或 `--` |
| 幂等键摘要 | 仅管理员 Tooltip 可见 |
| 操作 | 查看订单 / 重试 |

ERP 约束：

- CRM 不编辑订单。
- CRM 不取消订单。
- CRM 不直接操作 WMS。
- 重试不得生成第二份订单草稿。
- ERP 失败不回滚合同签署和商机赢单。

### 8.4 作废回退摘要

已作废合同展示：

- 作废时间。
- 动作原因摘要。
- 有效绑定解除结果。
- 商机回退目标阶段。
- 商机回退完成时间。
- 回退失败时的重试状态。

作废、解绑和商机回退应事务一致；正常情况下不出现部分成功。

## 9. 审计日志

### 9.1 记录范围

- 创建合同。
- 编辑草稿。
- 提交签署。
- 签署事件接收。
- 商机赢单联动。
- ERP 订单下推与重试。
- 归档。
- 作废与商机回退。
- 权限或状态阻断。

### 9.2 时间线项

每条日志展示：

- 动作名称。
- 操作者或系统来源。
- 操作时间。
- 原状态与目标状态。
- 结果摘要。
- 请求标识摘要。
- 失败时的用户可见原因。

### 9.3 展示规则

- 默认最近 10 条。
- 支持加载更多。
- 按时间倒序。
- 普通销售只看业务摘要。
- 管理员可展开技术摘要。
- 不展示密钥或认证凭证。
- 作废动作原因仅对有合同查看权限者可见。

## 10. 底部操作矩阵

### 10.1 布局

- 固定底栏高度 64px。
- 左侧为返回列表。
- 右侧按次要、主要、危险顺序排列。
- 同时最多 4 个文本按钮。
- 超出动作进入 More 菜单。
- 不可用动作不渲染。

### 10.2 状态矩阵

| 状态 | 左侧 | 主操作 | 次操作 | More |
|------|------|--------|--------|------|
| 草稿 | 返回列表 | 提交签署 | 编辑 | 作废 |
| 待签署 | 返回列表 | 模拟签署完成 | 无 | 作废 |
| 已签署 | 返回列表 | 归档 | 查看商机 | 重试联动（失败时） |
| 已归档 | 返回列表 | 无 | 查看商机 | 查看 ERP 订单 |
| 已作废 | 返回列表 | 无 | 查看历史商机 | 无 |

### 10.3 权限叠加

- 销售可编辑自己负责的草稿。
- 销售可提交签署和作废未签署合同。
- 主管或管理员可模拟签署完成。
- 主管或管理员可归档。
- 主管或管理员可重试联动。
- 只读审计仅返回和查看链接。
- 所有角色均无删除动作。

## 12. Mock 详情场景

### 12.1 主场景

- 一份待签署合同。
- 关联强盛科技客户。
- 关联商务谈判后进入合同阶段的商机。
- 合同金额与商机预计金额不同，展示弱提示。
- 时间线完成创建和提交签署节点。
- 主操作为“模拟签署完成”。
- 审计日志 5 条。

### 12.2 补充场景

- 草稿：可编辑、提交签署、作废。
- 已签署：商机赢单成功，ERP 下推处理中。
- 已签署联动失败：展示重试。
- 已归档：全部只读。
- 草稿作废：商机已回退。
- 待签署作废：保留提交节点并回退商机。
- 重复签署事件：显示忽略重复。
- ERP 重试成功：只产生一份订单草稿。

## 14. 键盘与可访问性

- 时间线节点可键盘聚焦。
- Badge 同时包含文字。
- 颜色不是状态的唯一表达。
- 图标按钮有 Tooltip 和 aria-label。
- 弹窗焦点锁定在内部。
- Esc 仅在未提交时关闭弹窗。
- 加载与结果通过 aria-live 播报。
- 固定底栏不遮挡审计日志末尾。
