# 公海_Demo_列表页

> **版本**：V2.0 | 2026-07-18
> **页面类型**：PC Web 列表页
> **前置文档**：《公海主PRD》《公海字段清单》《线索字段清单》
> **对象声明**：公海是线索共享视图，不创建独立业务字段或状态。
> **字段约束**：字段定义与 Tag 口径引用字段清单，本文只定义页面和交互。

---

## 1. 页面概述

| 项目 | 内容 |
|------|------|
| 页面名称 | 线索公海 |
| 页面路径 | `/lead-pool` |
| 页面目标 | 让销售快速发现并认领可用共享线索 |
| 主要用户 | 销售、销售主管、系统管理员、只读审计角色 |
| 页面骨架 | 页头 → 入池 Tab → 查询区 → 表格 → 分页 |
| 默认排序 | 引用《公海字段清单》排序规则 |
| 每页条数 | 默认 20 条，可切换 20 / 50 / 100 |
| 主要动作 | 单条认领 |

页面原则：

- 只展示当前查询时满足公海条件的线索。
- 不提供新增、编辑、删除、跟进或转客户操作。
- 认领成功后当前行立即移除。
- 认领提交时必须重新校验，不能相信列表快照。
- 原负责人保护中可查看，但不可认领。
- 联系信息按角色权限脱敏。
- 主内容最小宽度 1240px。

## 2. 入池 Tab 与后台计数

### 2.1 Tab 规格

| Tab | 过滤语义 | 计数来源 | 点击行为 |
|-----|----------|----------|----------|
| 全部 | 当前权限范围内全部公海线索 | 后台聚合 | 保留查询条件，清除入池类型条件 |
| 新线索 | 入池类型为新线索的视图记录 | 后台聚合 | 应用入池类型并回到第一页 |
| 已释放 | 入池类型为已释放的视图记录 | 后台聚合 | 应用入池类型并回到第一页 |
| 我的保护中 | 当前用户为原负责人且保护期未到 | 后台聚合 | 展示可查看但不可认领的记录 |

### 2.2 计数规则

- Tab 格式为 `名称 (计数)`。
- 后台按当前用户权限聚合。
- 计数受查询条件影响。
- 计数不受分页影响。
- 认领成功后刷新全部 Tab 计数。
- 其他人认领导致竞争失败时也刷新计数。
- 保护到期后“我的保护中”计数自动减少。
- 计数接口失败显示 `--`。
- 计数失败不阻断表格查询。
- 从线索详情返回时恢复当前 Tab。

## 3. 查询区

### 3.1 布局

- 查询区默认两行自适应。
- 控件标签位于上方。
- “查询”为主按钮。
- “重置”为次按钮。
- 查询后回到第一页。
- 条件同步到 URL。
- 回车仅在关键词输入框触发查询。

### 3.2 控件详细规格

| 条件/字段 | 控件 | 宽度 | Placeholder | 输入规则 | 查询行为 |
|-----------|------|------|-------------|----------|----------|
| 关键词 | Input | 240px | `搜索线索编号或公司名称` | 去除首尾空格；支持清空 | 编号精确优先、名称模糊 |
| 线索来源 | Select | 180px | `请选择线索来源` | 选项引用线索字段清单 | 单选过滤 |
| 所属行业 | Select | 180px | `请选择所属行业` | 选项引用线索字段清单 | 单选过滤 |
| AI评分 | RangeInput | 200px | `最小值 - 最大值` | 范围引用字段清单；最小不得大于最大 | 闭区间过滤 |
| 入池类型 | Select | 160px | `请选择入池类型` | 选项引用公海字段清单 | 与 Tab 条件联动 |
| 入池时间 | DateRangePicker | 260px | `开始日期 - 结束日期` | 含首尾日期 | 范围过滤 |
| 可认领性 | Select | 180px | `请选择可认领性` | 可认领 / 我的保护中 | 按当前用户动态过滤 |

### 3.3 查询交互

- AI评分最小值大于最大值时就地报错。
- 查询中禁用查询按钮并显示 Loader 图标。
- 查询失败保留旧数据。
- 查询失败 Toast `公海查询失败，请重试`。
- 重置清空条件并回到“全部”。
- 重置后保持每页条数。
- 查询无结果显示条件型空态。
- 快速连续查询取消上一请求。

## 7. 原负责人例外规则

### 7.1 角色判定

- 后台根据最近一次主动放弃事件判定原负责人。
- 前端不通过显示姓名自行推断。
- 超时回收不产生主动放弃原负责人保护。
- 多次放弃以最近一次主动放弃时间为准。
- 保护时间统一按服务端时钟计算。

### 7.2 展示行为

| 当前用户 | 距主动放弃时间 | 列表可见 | 认领入口 | 提示 |
|----------|----------------|:--------:|:--------:|------|
| 原负责人 | 小于 7×24 小时 | 是 | 不展示 | `可于{时间}后认领` |
| 原负责人 | 等于 7×24 小时 | 是 | 展示 | `可认领` |
| 原负责人 | 大于 7×24 小时 | 是 | 展示 | `可认领` |
| 其他销售 | 任意时间 | 按权限 | 展示 | `可认领` |

### 7.3 边界显示

- 未满 24 小时显示剩余小时和分钟。
- 大于等于 24 小时显示剩余天数和小时。
- Tooltip 展示精确可认领时间。
- 到期时自动刷新当前行。
- 浏览器时间不参与业务判定。
- 即使入口已刷新，提交时仍以服务端实时校验为准。

### 7.4 权威口径

- 保护期只限制原负责人重新认领。
- 保护期不等于线索对全体销售隐藏。
- 其他销售仍需通过权限、归属、持有上限和并发校验。
- 销售主管与管理员作为原负责人时同样受限制。
- 不提供人工绕过保护期的按钮。

## 8. 认领交互

### 8.1 发起认领

1. 用户点击“认领”。
2. 前端记录当前行版本。
3. 打开二次确认弹窗。
4. 用户确认后锁定按钮。
5. 请求携带线索标识、行版本和客户端请求键。
6. 后端重新校验权限、归属、保护期和持有上限。
7. 后端原子写入归属与分配时间。
8. 成功后关闭弹窗并移除当前行。
9. 刷新 Tab 计数。
10. Toast 提示及时跟进。

### 8.2 二次确认弹窗

| 项目 | 规格 |
|------|------|
| 标题 | `认领线索` |
| 内容 | `认领后该线索将归属你，并重新开始首次跟进计时。是否继续？` |
| 线索摘要 | 展示线索编号与公司名称 |
| 取消按钮 | `取消` |
| 确认按钮 | `确认认领` |
| 提交中 | `认领中...`，按钮禁用 |

### 8.3 成功行为

- 弹窗关闭。
- 当前行淡出移除。
- 保持当前页其他数据位置。
- 若当前页变空且不是第一页，自动回到上一页。
- 后台计数刷新。
- Toast `线索认领成功，请及时跟进`。
- Toast 提供“查看线索”快捷动作。

### 8.4 失败行为

| 失败类型 | 页面处理 | 文案 |
|----------|----------|------|
| 已被他人认领 | 关闭弹窗并移除行 | `该线索已被其他销售认领` |
| 原负责人保护 | 保留行，刷新到期时间 | `你在保护期内不可再次认领该线索` |
| 持有上限 | 保留行 | `已达到线索持有上限，请先处理现有线索` |
| 无权限 | 移除或刷新行 | `无权认领该线索` |
| 服务异常 | 保留行和弹窗上下文 | `认领失败，请重试` |
| 结果不确定 | 查询最终归属 | 根据查询结果展示成功或失败 |

### 8.5 防重复

- 弹窗提交中禁止重复点击。
- 键盘 Enter 只触发一次。
- 后端使用请求键幂等。
- 网络重试先查询最终归属。
- 当前用户已认领成功时返回首次结果。

## 10. 分页、刷新与空态

### 10.1 分页

- 默认每页 20 条。
- 支持 20、50、100 条。
- 显示总条数。
- 切页滚动到表格顶部。
- 改变每页条数回到第一页。
- 页码写入 URL。

### 10.2 自动刷新

- 页面可见时每 60 秒刷新。
- 认领弹窗打开时暂停自动刷新当前行替换。
- 到达保护期边界时单行即时刷新。
- 自动刷新保留筛选、页码和滚动位置。
- 自动刷新失败不弹重复 Toast，只展示弱提示。

### 10.3 空态

| 场景 | 文案 | 动作 |
|------|------|------|
| 公海为空 | `当前暂无可共享线索` | 刷新 |
| 查询无结果 | `未找到符合条件的公海线索` | 重置筛选 |
| 我的保护中为空 | `你当前没有处于保护期的线索` | 返回全部 |
| 服务异常 | `公海数据暂不可用` | 重试 |

空态不展示新增线索按钮。
