# 客户_Demo_详情页

> **版本**：V2.0 | 2026-07-18
> **页面类型**：PC Web 360° 详情页
> **前置文档**：《客户主PRD》《客户字段清单》
> **SSOT 声明**：基础档案来自 ERP 且只读；CRM 只展示快照、关联数据和 AI 结果。
> **字段约束**：字段定义与展示口径引用《客户字段清单》，本文不重复定义。

---

## 1. 页面概述

| 项目 | 内容 |
|------|------|
| 页面名称 | 客户 360° 详情 |
| 页面路径 | `/accounts/{id}` |
| 页面目标 | 在一个页面查看客户快照、风险、商机、订单、跟进与拜访 |
| 进入方式 | 客户列表、商机关联客户、全局深链接 |
| 数据权限 | 引用《客户主PRD》数据可见范围 |
| 页面骨架 | 页头 → 流失风险 Banner → 快照摘要 → 关系概览 → 关联商机 → ERP 订单 → 跟进聚合 → 拜访 → 底部操作 |
| 页面宽度 | 最大 1280px，居中布局 |

360° 视图原则：

- ERP 基础快照与 CRM 运营聚合明确分区。
- 基础快照不提供编辑控件。
- 每个关联区块独立加载、独立失败。
- 高流失风险在首屏形成明确行动提示。
- 新建商机、跟进和拜访属于关联动作，不改变客户档案。
- 停用客户保留历史视图并隐藏新业务动作。
- 不提供删除客户能力。

## 2. 初始化与容错

### 2.1 加载顺序

1. 校验客户查看权限。
2. 拉取客户快照与同步元数据。
3. 拉取最新成功流失风险。
4. 并行拉取商机、ERP 订单和跟进聚合。
5. 并行拉取拜访摘要与统计数据。
6. 根据客户可用性与权限计算操作入口。

### 2.2 页面状态

| 场景 | 页面表现 | 可执行动作 |
|------|----------|------------|
| 首次加载 | 固定尺寸骨架屏 | 等待 |
| 客户不存在 | 404 状态页 | 返回客户列表 |
| 无权限 | 403 状态页 | 返回客户列表 |
| 主快照加载失败 | 整页错误态 | 重试、返回列表 |
| 单个关联区块失败 | 区块错误态 | 局部重试 |
| 同步异常但有旧快照 | 显示旧快照 + 同步异常提示 | 管理员重试 |
| ERP已停用 | 顶部停用提示 + 历史只读 | 返回、查看历史 |

### 2.3 独立容错

- 商机加载失败不影响 ERP 快照。
- 订单加载失败不影响商机和跟进。
- 跟进聚合失败不影响风险 Banner 展示。
- AI 失败不影响客户基础信息。
- 任一区块重试只刷新该区块。
- 主快照不存在时不展示伪造客户信息。

## 3. 页头与同步状态

### 3.1 页头左侧

- 返回按钮使用 ArrowLeft 图标。
- 标题展示公司名称。
- 副标题展示客户编号。
- 客户编号支持复制。
- 公司名称超长时两行省略。
- 返回时恢复客户列表筛选与页码。

### 3.2 页头右侧

| 元素 | 展示 | 交互 |
|------|------|------|
| 客户等级 | Tag，口径引用字段清单 | 无 |
| ERP同步标识 | 技术元数据 | Tooltip 查看同步时间 |
| 刷新 | RefreshCw 图标 | 局部刷新全部区块 |

### 3.3 同步异常提示

- 提示文案：`ERP 快照同步异常，当前展示最近一次成功数据。`。
- 展示最后成功同步时间。
- 销售只可查看。
- 管理员显示“重试同步”。
- 重试前弹二次确认。
- 重试不清空当前快照。
- 同步成功后提示条消失。

### 3.4 ERP 停用提示

- 使用灰色全宽提示条。
- 文案：`该客户已在 ERP 停用，历史数据仅供查看。`。
- 隐藏新建商机、添加跟进和创建拜访。
- 不隐藏历史商机、订单或跟进。
- 不提供 CRM 恢复按钮。

## 4. AI 流失风险 Banner

### 4.1 Banner 位置

- 位于页头下方、快照卡片上方。
- 宽度与主内容一致。
- 高度自适应，最小 72px。
- 不使用浮动卡片覆盖内容。
- 风险文案与行动按钮同一水平区。

### 4.2 风险状态

| 风险场景 | 视觉 | 主文案 | 操作 |
|----------|------|--------|------|
| 高风险 | 红色左边框 + AlertTriangle | `该客户存在较高流失风险，建议尽快跟进。` | 添加跟进 |
| 中风险 | 黄色左边框 | `该客户需要持续关注。` | 查看风险依据 |
| 低风险 | 绿色左边框 | `当前客户关系活跃。` | 查看风险依据 |
| 待计算 | 灰色 | `流失风险正在计算。` | 刷新 |
| 失败且有历史结果 | 历史风险色 + Clock | `风险计算失败，当前展示上次结果。` | 主管重试 |
| 失败且无历史结果 | 灰色 + AlertCircle | `流失风险暂不可用。` | 主管重试 |

### 4.3 风险依据

- 展示命中的业务条件摘要。
- 展示最近跟进时间。
- 展示进行中商机数量。
- 展示最长阶段停留摘要。
- 展示最近成功计算时间。
- 不展示模型内部权重和技术参数。
- 风险依据使用 Popover 展开。

### 4.4 高风险后置结果

| 动作 | 展示 |
|------|------|
| CRM 跟进提醒 | 已创建、处理中、失败 |
| ERP 信用冻结通知 Mock | 已发送、处理中、失败 |

展示约束：

- 风险高不等于 ERP 已冻结信用。
- ERP 通知失败时显示管理员重试入口。
- 重复重试使用同一幂等键。
- 后置动作失败不改变风险 Tag。
- 停用客户不生成新的自动跟进提醒。

## 9. 跟进聚合组件

### 9.1 聚合范围

时间线聚合以下来源：

- 客户直接跟进。
- 转化前线索跟进。
- 客户名下商机跟进。
- 拜访完成自动生成的跟进记录。

聚合约束：

- 每条记录保留原来源类型。
- 每条记录保留原来源标识。
- 不复制成新的无来源记录。
- 按跟进时间统一倒序。
- 无权来源对象不进入结果。

### 9.2 区块头

| 元素 | 规格 |
|------|------|
| 标题 | `跟进记录` |
| 计数 | 聚合总条数 |
| 主按钮 | `添加跟进`，客户可用且有权限时展示 |
| 导出 | 主管或管理员可用 |

### 9.3 筛选控件

| 条件 | 控件 | Placeholder | 行为 |
|------|------|-------------|------|
| 来源类型 | MultiSelect | `全部来源` | 按来源过滤 |
| 跟进方式 | Select | `全部方式` | 选项引用跟进记录 SSOT |
| 跟进人 | SelectWithSearch | `全部跟进人` | 按数据权限加载 |
| 跟进时间 | DateRangePicker | `开始日期 - 结束日期` | 范围过滤 |

### 9.4 时间线项

每条记录展示：

- 跟进方式图标。
- 跟进人。
- 跟进时间。
- 跟进内容。
- 下次计划（存在时）。
- 来源 Tag。
- 来源编号链接。
- “来自拜访”自动生成标识。

时间线交互：

- 首屏 10 条。
- 点击“加载更多”追加 10 条。
- 新增成功插入顶部。
- 内容超过 6 行支持展开。
- 来源链接按权限跳转。
- 来源已失效时保留只读文本。

### 9.5 添加客户跟进

- 点击打开右侧抽屉。
- 关联对象固定为当前客户。
- 字段规格引用跟进记录 SSOT。
- 保存成功关闭抽屉并刷新风险提示的“数据已变化”状态。
- 新跟进不立即伪造低风险结果。
- 下一次定时计算或人工重算后更新风险。
- 保存失败保留用户输入。
- ERP已停用时不展示入口。
