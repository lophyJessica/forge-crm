# 盘点单字段清单

> 角色：字段清单(SSOT) | 类型：执行作业单
> 本文件为盘点单字段唯一事实来源；主 PRD、业务规则、流程推演和 Demo 页面引用本文件，不重复维护字段口径。

## 1. 字段设计原则

- CK 单号、状态、系统数、差异数、差异率、关联 FL 由系统生成或计算，不允许手工编辑。
- 盘点类型、盘点范围、盘点人、审核人在草稿阶段维护；进入盘点中后只读。
- 商品信息在盘点明细中快照存储，主数据后续变更不影响历史 CK。
- 日期格式 `YYYY-MM-DD`，时间格式 `YYYY-MM-DD HH:mm:ss`。
- 数量字段为整数，实盘数允许为 0；备注、原因说明、驳回原因 ≤200 字符。
- 空值展示 `-`。

## 2. 盘点头字段

| 字段编码 | 字段名称 | 类型 | 来源 | 必填 | 可编辑 | 枚举/校验/说明 |
|:--|:--|:--|:--|:--:|:--:|:--|
| ckNo | 盘点单号 | string | 系统生成 | 是 | 否 | `CK{YYYYMMDD}-{4位序号}`，如 `CK20260706-0001` |
| countType | 盘点类型 | enum | 用户选择 | 是 | 草稿可编辑 | `OPEN` 明盘、`BLIND` 盲盘、`DYNAMIC` 动盘、`FULL` 全盘 |
| scopeType | 盘点范围类型 | enum | 用户选择 | 是 | 草稿可编辑 | `WAREHOUSE` 仓库、`ZONE` 库区、`LOCATION` 货位、`SKU` 商品 |
| scopeDesc | 盘点范围描述 | string | 系统生成 | 是 | 否 | 根据仓库、库区、货位、商品组合生成摘要 |
| warehouseCode | 仓库编码 | string | 仓库档案 | 是 | 草稿可编辑 | 必须为启用仓库 |
| warehouseName | 仓库名称 | string | 仓库档案 | 是 | 否 | 随仓库编码带出 |
| zoneCodes | 库区编码集合 | array | 库区档案 | 否 | 草稿可编辑 | `scopeType=ZONE` 时至少 1 个 |
| locationCodes | 货位编码集合 | array | 货位档案 | 否 | 草稿可编辑 | `scopeType=LOCATION` 时至少 1 个；全盘可由系统展开 |
| productCodes | 商品编码集合 | array | 商品档案 | 否 | 草稿可编辑 | `scopeType=SKU` 时至少 1 个 |
| counterId | 盘点人ID | string | 用户档案 | 是 | 草稿可编辑 | 必须为启用账号且有 PDA 盘点权限 |
| counterName | 盘点人 | string | 用户档案 | 是 | 否 | 随盘点人 ID 带出 |
| reviewerId | 审核人ID | string | 用户档案 | 是 | 草稿可编辑 | 必须为主管或具备盘点审核权限账号 |
| reviewerName | 审核人 | string | 用户档案 | 是 | 否 | 随审核人 ID 带出 |
| status | 盘点状态 | enum | 系统 | 是 | 否 | `DRAFT` 草稿、`COUNTING` 盘点中、`PENDING_REVIEW` 待审核、`ADJUSTED` 已调整、`REJECTED` 已驳回 |
| plannedStartAt | 计划开始时间 | datetime | 用户输入 | 否 | 草稿可编辑 | 不早于当前日期；补录场景除外 |
| plannedEndAt | 计划结束时间 | datetime | 用户输入 | 否 | 草稿可编辑 | 不早于计划开始时间 |
| lockedAt | 锁定时间 | datetime | 系统 | 否 | 否 | 点击开始盘点并完成冻结后写入 |
| submittedAt | 提交审核时间 | datetime | 系统 | 否 | 否 | 生成差异报告后写入 |
| reviewedAt | 审核时间 | datetime | 系统 | 否 | 否 | 主管审核动作发生时间 |
| adjustedAt | 调整时间 | datetime | 系统 | 否 | 否 | 库存调整和 FL 生成成功后写入 |
| rejectedReason | 驳回原因 | string | 审核人输入 | 条件必填 | 审核时填写 | 审核驳回时必填，≤200 字符；不生成 FL，不调整库存 |
| totalLineCount | 明细行数 | integer | 系统计算 | 是 | 否 | `countDetails` 行数 |
| countedLineCount | 已盘行数 | integer | 系统计算 | 是 | 否 | 明细状态为已盘、重盘已盘或已调整的行数 |
| differenceLineCount | 差异行数 | integer | 系统计算 | 是 | 否 | `diffQty != 0` 的行数 |
| maxDiffRate | 最大差异率 | decimal | 系统计算 | 是 | 否 | 全部明细差异率最大值，展示为百分比 |
| recountTimes | 重盘次数 | integer | 系统计算 | 是 | 否 | 主管要求重盘一次加 1 |
| remark | 备注 | string | 用户输入 | 否 | 草稿可编辑 | ≤200 字符 |

## 3. 盘点明细字段

| 字段编码 | 字段名称 | 类型 | 来源 | 必填 | 可编辑 | 枚举/校验/说明 |
|:--|:--|:--|:--|:--:|:--:|:--|
| detailId | 明细ID | string | 系统生成 | 是 | 否 | 页面隐藏主键 |
| lineNo | 行号 | integer | 系统生成 | 是 | 否 | 按货位、商品排序 |
| locationCode | 货位编码 | string | 货位档案 | 是 | 否 | 盘点范围展开后生成 |
| locationBarcode | 货位条码 | string | 货位档案 | 是 | 否 | PDA 扫描校验 |
| locationName | 货位名称 | string | 货位档案 | 是 | 否 | 如 `A-01-02` |
| productCode | 商品编码 | string | 商品档案快照 | 是 | 否 | 字母+数字，不可手改 |
| productBarcode | 商品条码 | string | 商品档案快照 | 是 | 否 | 数字为主，支持 EAN-13 |
| productName | 商品名称 | string | 商品档案快照 | 是 | 否 | 商品快照 |
| productSpec | 规格型号 | string | 商品档案快照 | 否 | 否 | 空值展示 `-` |
| unit | 单位 | string | 商品档案快照 | 是 | 否 | 如 件、箱 |
| batchNo | 批次号 | string | 库存快照 | 否 | 否 | 当前 context 未展开批次规则，可为空 |
| systemQty | 系统数 | integer | 锁定时库存快照 | 是 | 否 | 锁定范围时取现存口径；盲盘 PDA 端不展示 |
| countedQty | 实盘数 | integer | PDA 输入 | 条件必填 | 盘点中可编辑 | 整数，`>=0`；确认明细时必填 |
| diffQty | 差异数 | integer | 系统计算 | 是 | 否 | `countedQty - systemQty` |
| diffRate | 差异率 | decimal | 系统计算 | 是 | 否 | `abs(diffQty) / systemQty`；系统数为 0 的处理见不确定性 |
| diffType | 差异类型 | enum | 系统计算 | 是 | 否 | `NONE` 无差异、`GAIN` 盘盈、`LOSS` 盘亏 |
| lineStatus | 明细状态 | enum | 系统 | 是 | 否 | `PENDING` 待盘、`COUNTED` 已盘、`RECOUNT_REQUIRED` 待重盘、`ADJUSTED` 已调整 |
| scannedLocationCode | 实扫货位 | string | PDA 扫描 | 条件必填 | 否 | 确认明细前必须等于当前货位 |
| scannedProductCode | 实扫商品 | string | PDA 扫描 | 条件必填 | 否 | 确认明细前必须等于当前商品 |
| countedBy | 实盘人 | string | 系统 | 否 | 否 | PDA 当前用户 |
| countedAt | 实盘时间 | datetime | 系统 | 否 | 否 | 确认明细时写入 |
| recountFlag | 是否重盘行 | boolean | 系统 | 是 | 否 | 主管要求重盘后相关行置为 true |
| lineRemark | 行备注 | string | PDA 输入 | 否 | 是 | ≤200 字符 |

## 5. 枚举值

### 5.1 盘点类型 `countType`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| OPEN | 明盘 | PDA 展示系统数 |
| BLIND | 盲盘 | PDA 不展示系统数 |
| DYNAMIC | 动盘 | 仅锁定当前扫描或任务覆盖货位 |
| FULL | 全盘 | 锁定全仓或指定全范围货位 |

### 5.2 盘点状态 `status`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| DRAFT | 草稿 | 未锁定库存 |
| COUNTING | 盘点中 | 已锁定范围，货位冻结 |
| PENDING_REVIEW | 待审核 | 差异报告已生成，等待主管审核 |
| ADJUSTED | 已调整 | 已确认调整，库存更新并生成 FL |
| REJECTED | 已驳回 | 审核不通过，已记录驳回原因，未调整库存 |

### 5.3 差异类型 `diffType`

| 值 | 展示 | 计算 |
|:--|:--|:--|
| NONE | 无差异 | `diffQty = 0` |
| GAIN | 盘盈 | `diffQty > 0` |
| LOSS | 盘亏 | `diffQty < 0` |

### 5.4 明细状态 `lineStatus`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| PENDING | 待盘 | 尚未确认实盘数 |
| COUNTED | 已盘 | 已确认实盘数并完成差异计算 |
| RECOUNT_REQUIRED | 待重盘 | 主管要求重盘 |
| ADJUSTED | 已调整 | 对应差异已纳入库存调整 |

## 6. 计算字段

| 字段 | 公式 | 说明 |
|:--|:--|:--|
| 差异数 | `countedQty - systemQty` | 正数盘盈，负数盘亏 |
| 差异率 | `abs(diffQty) / systemQty` | 系统数为 0 的口径见不确定性 |
| 差异行数 | `count(diffQty != 0)` | 头部汇总 |
| 最大差异率 | `max(diffRate)` | 判断是否触发主管重盘 |
| 已盘行数 | `count(lineStatus in COUNTED,RECOUNT_REQUIRED,ADJUSTED)` | 用于进度展示 |
| 完成条件 | `countedLineCount = totalLineCount` | 全部明细实盘后才可生成差异报告 |

## 7. 校验时机

| 时机 | 校验范围 |
|:--|:--|
| 保存草稿 | 盘点类型、仓库、盘点范围、盘点人、审核人基础校验 |
| 开始盘点 | 范围有效性、货位可冻结性、单号和状态校验 |
| PDA 扫货位 | 实扫货位是否在 CK 范围内，是否已冻结 |
| PDA 扫商品 | 实扫商品是否属于当前明细或盘点范围 |
| 确认明细 | 实盘数为整数且 `>=0` |
| 生成差异报告 | 全部明细已盘，差异数和差异率计算完成 |
| 主管审核 | 审核人权限、差异率阈值、重盘次数记录 |
| 确认调整 | 状态为待审核、无差异率 >5% 明细、幂等生成 FL |
| 审核驳回 | 状态为待审核、审核权限、驳回原因必填 |

## 8. 不确定性

| 项 | 当前字段口径 |
|:--|:--|
| 系统数为 0 的差异率 | `systemQty=0` 且 `countedQty=0` 记为 0%；`systemQty=0` 且 `countedQty>0` 记为 100%，触发重盘 |
| 批次管理 | context 未展开批次规则，`batchNo` 作为可空快照字段保留 |
