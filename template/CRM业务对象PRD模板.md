# CRM 业务对象 PRD 模板（v2.0 — 对齐 ERP 标准）

> 从 forge-erp/prd-docs/采购管理/采购订单/ 的 PRD 套件中提取,适配 CRM 场景。
> 标准:主PRD≥300行, Demo列表≥180行, Demo新增编辑≥190行, Demo详情≥170行

---

## 一、主PRD 标准章节

# [对象名称]主PRD

> **版本**：V1.0 | YYYY-MM-DD
> **读者**：研发工程师、测试工程师、产品复核

---

### 1. 业务背景

一段话讲清「这个对象解决什么问题」，然后列出 3-5 条没有统一管理时的具体痛点。
结尾:这个对象在整个系统链路中的定位(如:线索是 CRM 的入口控制对象,解决...)。

---

### 2. 功能范围

**In Scope**：
- 功能点 1（一句话描述）
- 功能点 2

**Out of Scope**：
- 功能点 X（原因说明，如"属于 XX 模块"或"一期不支持"）
- 每条 Out 必须写原因

---

### 3. 对象定位

#### 3.1 在系统中的位置

| 项目 | 内容 |
|------|------|
| 对象类型 | 线索（营销获客层）/ 商机（销售转化层）/ 客户（成交运营层） |
| 核心职责 | 一句话说明使命 |
| 来源 | 手动创建 / Excel导入 / 由XX对象转化 |
| 下游对象 | 线索→客户; 商机→ERP订单 |
| 关联关系 | 1:1 或 1:N |

#### 3.2 系统链路图

Mermaid flowchart: 上游数据来源 → 本对象 → 下游对象 → 最终影响

#### 3.3 实体关系说明

| 关系 | 说明 |
|------|------|
| 本对象 : 上游 | N:1 或 1:N，附解释 |
| 本对象 : 下游 | 同上 |

## 二、Demo PRD 标准章节

### 列表页 ≥180行

必备:Tab栏(含计数)/查询区(每控件详细规格)/工具条/表格列(含宽度+固定)/行内操作(按状态动态)/二次确认弹窗(标题+内容+按钮文案)/Mock(状态覆盖要求+数据要求)/反馈文案

### 新增编辑页 ≥190行

必备:页面概述(路径)/底部操作栏(按钮顺序+样式)/表单卡片(每字段:控件类型+placeholder+字符限制+联动规则)/编辑模式差异(草稿vs审核后)/字段联动规则/交互规范(保存/提交/返回的具体行为)

### 详情页 ≥170行

必备:页面概述(路径)/状态区(Badge+时间线)/信息卡片(KV只读)/明细/关联单据/底部操作(按状态动态)/跟进记录/交互规范(每操作的具体行为+弹窗文案)
